# 关于图

## 基本定义

**图（Graph）**：由**顶点（Vertex）**和**边（Edge）**组成的集合。

记作
$$
G=(V,E)
$$
其中

- $$V$$：顶点集合（Nodes）
- $$E$$：边集合（Edges）

## 图的分类

| 分类维度   | 类型         | 特点                                   |
| ---------- | ------------ | -------------------------------------- |
| **方向性** | **有向图**   | 边有方向，用箭头表示，如 A → B         |
|            | **无向图**   | 边无方向，如 A—B                       |
| **权值**   | **加权图**   | 每条边有“权值”（weight），如距离、代价 |
|            | **无权图**   | 边仅表示连接关系                       |
| **连通性** | **连通图**   | 任意两顶点间有路径相通（无向图）       |
|            | **强连通图** | 有向图中任意两顶点之间互相可达         |

## 基本术语

| 术语           | 含义                                                         |
| :------------- | ------------------------------------------------------------ |
| **度**         | 顶点连接的边数。无向图中：度 = 连接边条数；有向图中： 入度（In-degree）= 指向该顶点的边数；出度（Out-degree）= 从该顶点出发的边数 |
| **路径**       | 从一个顶点到另一个顶点经过的边序列。                         |
| **路径长度**   | 路径上边的数量（或权值和）。                                 |
| **简单路径**   | 不包含重复顶点的路径。                                       |
| **子图**       | 由原图部分顶点和边组成的新图。                               |
| **连通分量**   | 无向图中极大连通子图。                                       |
| **强连通分量** | 有向图中任意两点互相可达的极大子图。                         |
| **邻接**       | 若$$u,v\in E$$，则$$u$$和$$v$$邻接。                         |

# 存储结构

## 邻接矩阵

- 用一个**一维数组**存储图中**顶点**的信息，用一个**二维数组**（称为邻接矩阵）存储图中**各顶点之间的邻接关系**。

- 假设图$$G=(V,E)$$有$$n$$个顶点，则邻接矩阵是一个$$n\times n$$的方阵，定义为：
  $$
  edge[i][j]=\begin{cases}
  1,\ 若(i,j)\in E或<i,j>\in E \\
  0
  \end{cases}
  $$

- 适合用于**稠密图**。


## 邻接表

- 方便找任意顶点的所有邻接点；
- 适合用于**稀疏图**。

> 无向图的邻接表

- **顶点表**：存储所有顶点的信息和指向边表的指针；
- **边表**：对于每个顶点$$v_i$$，将所有与$$v_i$$​​相连的顶点连成的一个单链表。
- **存储空间需求**：$$O(n+2e)$$

- 边表则分为**出边表**和**入边表**。

# 遍历

- 从图中某一顶点出发，对图中所有顶点访问一次且仅访问一次；
- 如果从某顶点出发时无法到达图中的所有顶点，则**多次调用遍历算法**；
- 为避免顶点被重复访问，**附设访问标志数组**`visited[n]`，当第$$i$$个顶点被访问时，将`visited[i]`由`false`更改为`true`。

## 深度优先搜索

基于**栈**：

1. 设图$$G$$的初态是所有顶点都未被访问过，即`visited[]`设置为`false`；
2. 从$$G$$中任选一点作为出发点：
   1. 首先访问出发点$$v$$，且设置`visited[v]`为`true`；
   2. 然后从出发点$$v$$开始，考察它任意一个相邻的顶点$$w$$​，若`visited[w]`为`false`，访问它并将其设置为`true`；
   3. 以$$w$$为新的出发点，考察$$w$$的任意一个相邻的顶点，然后重复第2步，直到图中所有与$$v$$有路径相通的顶点均被访问为止；
   4. 若当前出发点的所有相邻顶点均被访问过，则回退到上一个点，以它作为出发点，重复第2步。
3. 到达新出发点时执行入栈操作，回退到上一个点时执行出栈操作。

以下图为例：

<img src="C:\Users\86133\Desktop\学习\dio图表\数据结构\图-深度优先搜索.drawio.svg" alt="图-深度优先搜索.drawio" style="zoom:120%;" />

最终的DFS序列应为$$v_1\rightarrow v_2\rightarrow v_5\rightarrow v_3\rightarrow v_4$$。

其中：

- 以$$v_1$$为起始顶点；
- 蓝色箭头表示深度优先搜索方向以及入栈顺序；
- 红色箭头表示回退到上一点以及出栈顺序；
- 栈的始末状态均为空栈。

另外地，深度优先搜索的结果不唯一，这是由图的存储结构决定的。**如果使用邻接矩阵或邻接表，就会得到唯一的DFS序列**。

## 广度优先搜索

基于**队列**：

1. 设图$$G$$的初态是所有顶点都未被访问过，即`visited[]`设置为`false`；
2. 从$$G$$中任选一点作为出发点：
   1. 首先访问出发点$$v$$，且设置`visited[v]`为`true`；
   2. 然后从出发点$$v$$开始，访问它的所有邻接点，并将它们都设置为`true`；
   3. 再依次以这些邻接点为新的出发点，重复第2步，图中所有与$$v$$有路径相通的顶点均被访问为止；
3. 访问顶点时该顶点执行入队操作，将顶点作为新出发点时该顶点执行出队操作。

# 最小生成树

**要求**：

- **树**
  - 不形成回路；
  - $$n$$个顶点有$$n-1$$条边；
- **生成树**
  - 包含全部顶点；
  - $$n-1$$条边都在图里；
- **边的权重和最小**

## Prim算法

1. 任取树中的一个顶点$$v_0$$，将其存储到集合$$U=\{v_0\}$$；
2. 找出与该顶点相连的权值最小的边，将边的另一端顶点加入到集合中，得到$$U=\{v_0,v_1\}$$；
3. 寻找与$$U$$中各顶点相连的权值最小的边，重复第2步，直到集合$$U$$包含所有顶点。

## Kruskal算法

1. 先标出所有顶点；
2. 将权值由小到大的边依次表示出来（连接顶点），并保证不构成回路。

# 最短路径

> 概念

在一个有向图中，边有方向，意味着从一个节点到另一个节点的路径是有方向性的。每条边通常会有一个权重（即代价、距离或时间等）。最短路径问题的目标是找到从源节点（起点）到目标节点（终点）的最短路径，通常根据图中边的权重来计算最短路径。

## Dijkstra算法

- **单源最短路径问题**

  从一个源节点出发，计算到其他所有节点的最短路径。

- **算法原理**
  1. **初始化**：设定源节点的距离为0，相邻节点的距离设置为对应权值，不相邻的则设置为无穷大。然后将源节点标记为已处理。
  2. **选择最小距离节点**：从未处理的节点中，选择当前距离源节点最近的节点。
  3. **更新邻接节点的最短路径**：对于该节点的所有邻接节点，如果通过当前节点到达它们的路径更短，则更新它们的最短路径值。
  4. **重复步骤2和3**：直到所有节点都被处理过。
  5. **最终结果**：所有节点的最短路径都被计算出来。

- **详细步骤**

  1. **初始化**：

     - 定义一个数组`D[]`，其中`D[i]`表示从源节点到节点`i`的最短距离。初始时与源节点相邻的节点的`D[]`设置为相应的权值，不相邻的`D[]`设置为$$\infty$$。

     - 定义一个集合`S{}`来标记哪些节点已被处理过，即确定**已找到了最短路径**。
     - 定义一个数组`P[]`，其中`P[i]`表示从源节点到节点`i`当前最短路径下，节点`i`的前一个节点。

  2. **选择未访问节点中距离最小的节点：**

     - 从`D[]`中选择最小的、未被访问过的节点，并将其标记为已访问。

  3. **更新邻接节点的距离**：

     - 对于当前节点的每个邻接节点`v`，检查通过当前节点到达`v`的路径是否比`v`当前记录的路径更短，如果是，则更新`v`的距离。

  4. **重复直到所有节点都访问过**。

# 拓扑排序

**有向无环图（DAG）的特性**

1. **没有环**：DAG中不存在从某个节点出发，沿着边的方向回到同一个节点的路径。即图中不能有任何闭环。
2. **拓扑排序**：DAG允许对节点进行排序，使得对于图中的每一条边 `(u, v)`，节点 `u` 在排序中排在节点 `v` 前面。拓扑排序对于许多问题（例如任务调度、编译器优化）非常重要。
3. **分层结构**：DAG可以有效地表示具有分层结构的关系。比如，任务的**先后依赖关系**、文件系统中的**目录结构**等。

---

**拓扑排序算法**

1. 在有向图中选一个没有前驱的顶点且输出它；
2. 从图中删除该顶点和所有以它为尾的弧；
3. 重复(1)(2)两步，直到所有顶点输出为止。
