# 存储结构

## 逻辑结构

描绘数据之间的逻辑关系。

- **线性结构**：一对一关系（如数组、链表、栈、队列）
- **树形结构**：一对多关系（如二叉树、堆）
- **图结构**：多对多关系

## 物理结构

**顺序存储**（数组）

- 元素地址连续，支持随机访问
- 插入删除开销大（需要移动元素）

**链式存储**（链表）

- 元素地址不连续，用**指针**链接
- 插入删除灵活，但随机访问困难

# 空间性能对比

## 1. 顺序存储

典型：数组、顺序表

- **存储方式**：
   所有元素**连续存放**在一块内存区域中。
- **空间开销**：
  - 元素本身所需空间：$$n\cdot sizeof(data)$$
  - 额外开销：几乎没有，仅需少量变量（如数组长度、容量）。
- **特点**：
  - 空间利用率高。
  - 需要 **预先分配** 一块连续的内存空间，如果数据量动态变化大，可能导致内存**浪费或溢出**。
  - 扩容时通常需要重新申请更大空间并复制数据，开销较大。

## 2. 链式存储

典型：单链表、双链表

- **存储方式**：
   各元素分散存放，通过指针（或引用）相互链接。
- **空间开销**：
  - 元素本身所需空间：$$n\cdot sizeof(data)$$
  - 额外开销：每个节点需要一个或多个指针：
    - 单链表：1 个指针 → $$n\cdot sizeof(pointer)$$
    - 双链表：2 个指针 →$$2n\cdot sizeof(pointer)$$
- **特点**：
  - 空间利用率低于顺序存储（指针占用额外内存）。
  - 无需预先分配大块内存，**插入/删除灵活**，更适合动态变化的数据量。
  - 节点分散存储，可能造成 **空间碎片化**，降低缓存命中率。

> [!TIP]
>
> 如果数据量 **稳定、存取频繁** → 顺序存储更合适（空间效率高）。
>
> 如果数据量 **动态变化大，插入/删除操作多** → 链式存储更合适（灵活性强，但要付出额外空间开销）。

# 时间性能分析

## 时间复杂度

- 以基本操作的 **执行次数** 为衡量单位。
- 输入规模 n → 算法运行时间的函数 T(n)。
-  平均复杂度 vs 最坏情况复杂度
  - **平均复杂度**：所有可能输入下的平均运行时间。
  - **最坏情况复杂度**：输入规模 n 时的最大运行时间，实际分析更多采用。

## 运行时间函数

假设现在有函数：
$$
f(n)=a_{0}+a_{1}x^{1}+a_{2}x^{2}+...+a_{n-1}x^{n-1}+a_{n}x^{n}
$$
编写一个函数，能够计算$$f(n)$$的值：

```c
double sumf(double p, double x, int n, int i, double a[]){
    p = a[0];
    for(i=1; i<=n; i++){
        p += a[i] * pow(x, i);
    }
    return p;
}
```

- 可以分析得到第$$n$$次for循环要执行$$n-1+1$$（乘幂以及累加求和）次基本操作；
- 一共执行$$n$$次循环，所以总的操作次数为$$1+2+...+n$$次；用$$\Delta t$$表示计算机执行单次操作所需要的时间：

$$
T(n)=\frac{n(n+1)}{2}\cdot\Delta t=\frac{\Delta t}{2}\cdot n^{2}+\frac{\Delta t}{2}\cdot n
$$

- 令$${\Large \frac{\Delta t}{2}=c}$$，则有

$$
T(n)=cn^{2}+cn
$$

## 渐进表示法

**提供统一的效率比较标准。**

若存在常数$$c>0$$和$$n_{0}>0$$，当$$n>n_{0}$$（$$n\rightarrow\infty$$）时有：
$$
T(n)\le c\cdot f(n)
$$
则称：
$$
T(n)=O(f(n))
$$
以上文中的函数为例，由于：
$$
T(n)=cn^2+cn\le2cn^2 \\
T(n)\le Cn^2
$$
所以：
$$
T(n)=O(n^2)
$$

- 若两段算法的复杂度分别为$$T_1(n)=O(f_1(n))$$和$$T_2(n)=O(f_2(n))$$，则：

  - $$T_1(n)+T_2(n)=max\{O(f_1(n)),\ O(f_2(n))\}$$​

  - $$T_1(n)\times T_2(n)=O(f_1(n)\times f_2(n))$$​

- 一个for循环的时间复杂度等于循环次数乘以循环体代码的复杂度；

-  if-else结构的复杂度取决于if的条件判断复杂度两个分枝部分的复杂度，总体复杂度取三者中最大。
