# 查找的概念

- **定义**：在数据集合中寻找满足某种条件（通常是关键字等于给定值）的数据元素的过程。
- **查找表**：同一类型的数据元素（或记录）构成的集合。
- **平均查找长度 (ASL - Average Search Length)**：衡量查找算法效率的最主要指标。
  - 公式：$ASL = \sum_{i=1}^{n} P_i C_i$
  - $P_i$：查找第 $i$ 个元素的概率（通常假设为 $1/n$）。
  - $C_i$：查找第 $i$ 个元素时需要进行的比较次数。

# 静态查找表

**特点**：只做查找操作，不进行插入和删除。

## 1. 顺序查找

- **概念**：从表的一端开始，逐个将记录的关键字和给定值比较。若某个记录的关键字和给定值相等，则查找成功；否则查找失败。
- **特点**：对数据无序要求，适用于线性链表；从**最后一个记录**开始查找。
- **时间复杂度**：$O(n)$​

```go
// 顺序查找演示
func SequentialSearch(arr []int, target int) int {
    for i, v := range arr {
        if v == target {
            return i // 返回索引
        }
    }
    return -1 // 未找到
}
```

## 2. 折半 (二分) 查找

- **概念**：每次取区间中间位置的元素进行比较，若相等则找到；若目标值小于中间值，则在左半区继续查找；反之在右半区查找。
- **核心条件**：**必须采用顺序存储结构**，且**表中元素按关键字有序**。
- **时间复杂度**：$O(\log n)$​

```go
// 二分查找演示 (前提：arr 必须是有序的)
func BinarySearch(arr []int, target int) int {
    low, high := 0, len(arr)-1
    
    for low <= high {
        // 防止 (low+high) 溢出，等同于 (low+high)/2
        mid := low + (high-low)/2 
        
        if arr[mid] == target {
            return mid
        } else if arr[mid] < target {
            low = mid + 1 // 去右边找
        } else {
            high = mid - 1 // 去左边找
        }
    }
    return -1
}
```

- **二叉判定树**

  - 折半查找的过程可以用一棵二叉树来描述，这棵树称为**判定树**。

    - **节点含义**：树中的每个节点对应有序表中的一个记录（数据元素）。
    - **节点值**：节点的值为该记录在表中的**位置**（下标）。

  - 对于有 $n$ 个记录的有序表，其判定树构造规则如下：

    1. **当 $n=0$ 时**：折半查找判定树为空。
    2. **当 $n>0$ 时**：
       - **根节点**：对应有序表中序号为 $mid = (n+1)/2$ 的记录。
       - **左子树**：是与有序表 $F[1] \sim F[mid-1]$ 相对应的折半查找判定树 。
       - **右子树**：是与有序表 $F[mid+1] \sim F[n]$​ 相对应的折半查找判定树。

  - 以一个包含 11 个元素 ($n=11$​) 的有序表为例：

    ![查找算法.drawio](C:\Users\86133\Desktop\学习\dio图表\数据结构\查找算法.drawio.svg)

  - **内部节点 (圆形)**：代表查找**成功**的情况。例如图中序号为 6 的根节点，以及 3、9 等子节点。

  - **外部节点 (方形)**：代表查找**失败**的情况（失败节点）。

    - 若有 $n$ 个关键字，则判定树的失败节点数为 $n+1$ 个。
    - 外部节点代表了数值区间，例如 "1-2" 代表查找值位于索引 1 和 2 之间的情况。

  - **树的高度与比较次数**

    - **比较次数上限**：在查找不成功和最坏情况下查找成功所需的关键字比较次数，都**不超过判定树的高度**。

    - 树高公式：$n$ 个节点的判定树高度与 $n$ 个节点的完全二叉树高度相同。

      $$H = \lfloor \log_2(n) \rfloor + 1$$

    - **结论**：折半查找在查找过程中进行比较次数最多不超过 $\lfloor \log_2(n) \rfloor + 1$。

    - **节点分布**：判定树中度小于 2 的节点只能出现在下面两层上。

  - **平均查找长度 (ASL) 计算**

    根据提供的示例 ($n=11$)，各层的比较次数 ($C_i$​) 分别为 1, 2, 3, 4 次。

    - **ASL 成功 (查找存在的元素)**

      $$ASL_{成功} = \frac{\sum (每层节点数 \times 层数)}{n}$$

      根据图示数据：

      $$ASL_{成功} = \frac{1 \times 1 + 2 \times 2 + 3 \times 4 + 4 \times 4}{11} = \frac{25}{11} \approx 2.27$$

    - **ASL 失败 (查找不存在的元素)**

      $$ASL_{失败} = \frac{\sum (每层失败节点数 \times (上一层层数))}{失败节点总数}$$

      注意：失败节点实际上是在最后一层比较后确认失败的，所以通常计算其父节点的层数或者该“虚节点”所在的层数。根据课件图示计算逻辑：

      $$ASL_{失败} = \frac{3 \times 4 + 4 \times 8}{12} = \frac{44}{12} \approx 3.67$$

      - 分母是 12，因为有 $n+1 = 11+1 = 12$ 个失败节点。

      - 分子是失败节点的路径长度之和。图中第 3 层有 4 个失败节点，第 4 层有 8 个失败节点。

## 3. 分块查找

- **概念**：又称**索引顺序查找**。将表分成若干块，**块间有序**（第 $i+1$ 块中所有元素均大于第 $i$ 块中的最大元素），**块内无序**。
- **查找过程**：
  1. 先在索引表中（可用二分查找或顺序查找）确定待查记录所在的块。
  2. 再在块内进行顺序查找。
- **性能**：介于顺序查找和折半查找之间。

# 动态查找表

**特点**：在查找过程中同时进行插入或删除操作（如文件系统、数据库索引）。

## 二叉排序树

- **概念**：
  1. 若左子树不空，则左子树上所有结点的值均**小于**根结点的值。
  2. 若右子树不空，则右子树上所有结点的值均**大于**根结点的值。
  3. 左、右子树也分别为二叉排序树。
- **特性**：**中序遍历**二叉排序树可以得到一个递增的有序序列。
- **时间复杂度**：
  - 最好情况（平衡）：$O(\log n)$
  - 最坏情况（退化成链表）：$O(n)$​

- **树的节点结构：**

  ```go
  type TreeNode struct {
      Val   int
      Left  *TreeNode
      Right *TreeNode
  }
  ```

### 查找

- **原理**

  - 查找过程非常像二分查找。从根节点开始：
    1. 如果树为空，返回 `nil`（没找到）。
    2. 如果目标值 **等于** 当前节点值，返回当前节点（找到了）。
    3. 如果目标值 **小于** 当前节点值，说明目标在左边，递归查找 **左子树**。
    4. 如果目标值 **大于** 当前节点值，说明目标在右边，递归查找 **右子树**。

- **实现**

  ```go
  func Search(root *TreeNode, target int) *TreeNode {
      // 递归终止条件：树为空 或 找到了目标值
      if root == nil || root.Val == target {
          return root
      }
  
      // 如果目标值小于当前值，去左边找
      if target < root.Val {
          return Search(root.Left, target)
      }
  
      // 否则去右边找
      return Search(root.Right, target)
  }
  ```

### 插入

- **原理**

  插入实际上就是一次“失败的查找”。要找到那个本该存在但却是 `nil` 的位置，把新节点放进去。

  1. 从根节点开始比较。
  2. 若插入值 < 当前节点，向左走；若插入值 > 当前节点，向右走。
  3. 走到某个位置发现是空的（`nil`），就在这里创建新节点。

- **实现**

  ```go
  func Insert(root *TreeNode, val int) *TreeNode {
      // 1. 如果当前位置为空，这里就是我们要插入的地方
      if root == nil {
          return &TreeNode{Val: val}
      }
  
      // 2. 递归寻找插入位置
      if val < root.Val {
          root.Left = Insert(root.Left, val) // 递归左子树，并更新左指针
      } else if val > root.Val {
          root.Right = Insert(root.Right, val) // 递归右子树，并更新右指针
      }
      // 如果 val == root.Val，通常不处理（不插入重复值），或者更新计数
  
      // 3. 返回当前节点，保持树结构
      return root
  }
  ```

### 删除

- **原理**

  删除节点后，必须保证 BST 的性质（左<根<右）依然成立。我们要分三种情况讨论：

  1. **情况一：叶子节点（没有子节点）**
     - 直接删除即可（返回 `nil` 给父节点）。
  2. **情况二：只有一个子节点（只有左孩子 或 只有右孩子）**
     - “子承父业”。让唯一的子节点替代当前被删除节点的位置。
  3. **情况三：有两个子节点（既有左又有右）**
     - 需要找到一个“替身”来顶替该位置。
  4. 这个“替身”通常是：**右子树中的最小节点**（后继节点） 或者 **左子树中的最大节点**（前驱节点）。
  5. **做法：**
     1. 找到右子树的最小值节点 `minNode`。
     2. 把 `minNode` 的值赋给当前要删除的节点（偷梁换柱）。
     3. 在右子树中删除那个 `minNode`（这就转化为了情况 1 或 2，因为最小节点一定没有左孩子）。

- **实现**

  ```go
  // 辅助函数：找到一棵树中最小的节点（一直往左走）
  func findMin(node *TreeNode) *TreeNode {
      for node.Left != nil {
          node = node.Left
      }
      return node
  }
  
  func Delete(root *TreeNode, key int) *TreeNode {
      if root == nil {
          return nil
      }
  
      // 1. 先去寻找要删除的节点
      if key < root.Val {
          root.Left = Delete(root.Left, key)
      } else if key > root.Val {
          root.Right = Delete(root.Right, key)
      } else {
          // 找到节点了！root 就是要删除的节点
  
          // 情况 1 & 2：如果没有孩子，或只有一个孩子
          if root.Left == nil {
              return root.Right // 右孩子上位（如果右也是nil，正好返回nil）
          }
          if root.Right == nil {
              return root.Left  // 左孩子上位
          }
  
          // 情况 3：有两个孩子
          // A. 找到右子树中最小的节点（后继节点）
          minNode := findMin(root.Right)
  
          // B. 用后继节点的值覆盖当前节点的值
          root.Val = minNode.Val
  
          // C. 删除右子树中那个原本的后继节点
          // 注意：这里是对 root.Right 进行递归删除，目标是刚找到的 minNode.Val
          root.Right = Delete(root.Right, minNode.Val)
      }
      return root
  }
  ```

# 哈希表

哈希表本质上是**数组**的升级版。 数组可以通过下标 $O(1)$ 访问，但要求下标必须是整数。哈希表通过**哈希函数**，将任意类型的 Key（字符串、对象等）映射为一个整数下标，从而实现“像数组一样快”的查找。

## 哈希函数

一个好的哈希函数需要满足：计算速度快、分布均匀（减少冲突）。

- **除留余数法**

  - **公式**：$f(key) = key \mod p$

  - **要点**：$p$ 通常取一个不大于哈希表长度 $m$ 的**质数**。

  - **为什么是质数？** 使用质数取模可以最大限度地减少因 Key 的分布规律（如等差数列）导致的哈希冲突。

- **直接定址法**

  - **公式**：$f(key) = a \times key + b$

  - **特点**：线性函数，无冲突。

  - **缺点**：要求 Key 的分布非常紧凑，否则会浪费大量空间（比如 Key 只有 1 和 10000，就需要开 10000 大小的数组）。

- **平方取中法**

  - **原理**：先计算 $key^2$，然后取中间的几位作为哈希地址。

  - **场景**：当 Key 的每一位数字分布都不均匀时，平方后中间几位受每一位的影响比较大，能混合得更均匀。

## 哈希冲突处理

只要 Key 的数量足够多，必然会出现 $f(key_1) = f(key_2)$ 的情况，这就是**哈希冲突**。

### 链地址法

- **原理**：数组的每个位置不再存数据，而是存一个**链表**（或红黑树）的头指针。所有哈希值相同的元素，都挂在这个链表上。

- **优点**：

  - 处理简单，无堆积现象。

  - 对**装载因子**不敏感，即使存满了，也就是链表变长一点。

- **缺点**：

  - 缓存不友好：链表节点在内存中是分散的，遍历时 CPU 缓存命中率低。

  - 需要额外的指针存储空间。

### 开放定址法 | 线性探测

- **原理**：如果坑被占了，就按照某种规则去找**下一个**空坑。数据直接存在数组里。
- **线性探测**：$d_i = 1, 2, 3...$（挨个往后找）。**优点**是缓存极其友好；**缺点**是容易产生“一次聚集”，即冲突的数据连成一大片，后续插入越来越慢。

### 再哈希法

- **原理**：准备多个哈希函数 $H_1, H_2, H_3...$。如果 $H_1(key)$ 冲突了，就计算 $H_2(key)$，直到不冲突。
- **缺点**：计算时间开销大。
