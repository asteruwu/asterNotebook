# 冒泡排序

- **概念**

  将待排序的记录看作是竖着排列的“气泡”，在排序过程中，越小的元素会经由交换慢慢“浮”到数列的顶端（或者越大的元素慢慢“沉”到底部）。

- **算法逻辑**

  冒泡排序的核心思想是**“两两交换，大的后移”**。

  1. **比较相邻的两个元素**。如果第一个比第二个大，就交换它们两个。
  2. 对每一对相邻元素做同样的工作，从开始第一对到结尾的最后一对。
  3. **一轮结束后**，最后的那个元素就是这组数据中**最大**的数（它已经确定了最终位置）。
  4. 针对所有的元素重复以上的步骤，除了最后一个（因为它已经排好了）。
  5. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

- **代码实现**

  ```go
  func BubbleSortOptimized(arr []int) []int {
  	n := len(arr)
  	for i := 0; i < n-1; i++ {
  		// 标记位：假设这一轮没有发生交换（即已经有序）
  		swapped := false
  		
  		for j := 0; j < n-1-i; j++ {
  			if arr[j] > arr[j+1] {
  				arr[j], arr[j+1] = arr[j+1], arr[j]
  				swapped = true // 发生了交换，说明还不是有序的
  			}
  		}
  		
  		// 如果 flag 依然是 false，说明没有任何元素需要交换；否则提前结束
  		if !swapped {
  			break
  		}
  	}
  	return arr
  }
  ```

- **性能分析**

  - **时间复杂度**

    - 当数组是**逆序**的时候（如 `[5, 4, 3, 2, 1]`），每一轮都要比较并交换，效率极低。
    - 时间复杂度为$O(n^2)$。

  - **空间复杂度**

    - 只需要申请一个临时变量用于交换，属于**原地排序**。复杂度为$O(1)$。

  - **稳定性**

    **稳定。**

    - 当两个相邻元素相等（`arr[j] == arr[j+1]`）时，我们**不会**进行交换（代码里写的是 `>` 而不是 `>=`）。

    - 因此，相同元素在排序后的相对位置保持不变。

# 直接插入排序

- **算法逻辑**

  将数组逻辑上分为两个区间：

  1. **已排序区间**：数组的前半部分（初始时只包含第1个元素）。
  2. **未排序区间**：数组的后半部分。

  **具体步骤**：

  1. **取值**：从未排序区间取出第一个元素，作为“待插入元素”（Key）。
  2. **比较与移动**：将 Key 与已排序区间的元素**从后往前**逐个比较。
     - 如果当前元素比 Key 大，则将该元素**向后移一位**（腾出坑位）。
     - 如果当前元素比 Key 小（或相等），说明找到了插入位置，停止移动。
  3. **插入**：将 Key 填入空出来的位置。
  4. 重复上述过程，直到未排序区间清空。

- **代码实现**

  ```go
  func InsertionSort(arr []int) []int {
  	n := len(arr)
      if n == 1 {
  		return arr
      }
  	// 默认下标0的元素自己本身就是有序的
  	for i := 1; i < n; i++ {
  		key := arr[i]
  		j := i - 1 
          
  		// 如果 arr[j] 比 key 大，就把它往后挪（腾出空位）
  		for j >= 0 && arr[j] > key {
  			arr[j+1] = arr[j]
  			j--
  		}
  		arr[j+1] = key
  	}
  	return arr
  }
  ```

- **性能分析**

  - **时间复杂度**

    - **场景**：数组是**逆序**的。

    - **原因**：每一个元素都要跟前面的所有元素比一遍，并且前面的元素都要向后移。
    - 时间复杂度为$O(n^2)$。

  - **空间复杂度**

    - 只需要一个变量 `key` 暂存数据，属于**原地排序**。
    - 复杂度为$O(1)$。

  - **稳定性**

    **稳定。**

    - 当遇到与 `key` 相等的元素时，循环条件 `arr[j] > key` 不成立，移动停止，`key` 会被插入到相等元素的**后面**。因此相对顺序保持不变。

# 希尔排序

希尔排序是**基于“步长（Gap）”分组的直接插入排序**。

- **算法逻辑**

  希尔排序允许元素进行**大跨步的跳跃移动**。

  1. 先将数组按照一定的间隔（步长 Gap）分成若干个子序列。
  2. 对每个子序列分别进行直接插入排序。
  3. 不断缩小步长（Gap），重复上述过程。
  4. 最后当 Gap=1 时，整个数组已经**基本有序**，此时再进行一次微调（标准插入排序），效率极高。

- **性能分析**

  - **时间复杂度**

    - **平均情况**: 取决于步长序列。

      - 使用希尔增量（$N/2$）：约为 $O(n^2)$（最坏情况）。

      - 使用 Hibbard 增量（$2^k-1$）：约为 $O(n^{1.5})$。

      - 使用 Knuth 增量：约为 $O(n^{1.3})$。

    - **最坏情况**: $O(n^2)$（当使用由简陋的增量序列时）。

    - **最好情况**: $O(n)$​（数组已有序）。

  - **空间复杂度**

    - 原地排序，不需要额外的存储空间，复杂度为$O(1)$。

  - **稳定性**

    **不稳定。**

    - 因为采用了“分组”跳跃交换，如果有两个 `5`，前面的 `5` 被分到了 A 组，后面的 `5` 被分到了 B 组。A 组的 `5` 可能没动，但 B 组的 `5` 可能因为排序被调到了前面。

# 快速排序

快速排序的核心是**分治法**。

- **算法逻辑**

  - **选基准**：从数组中挑出一个元素作为“裁判”。

  - **分区**：把数组里的元素重新排列。

    - 比“裁判”小的，统统扔到左边。

    - 比“裁判”大的，统统扔到右边。

  - **递归**：此时“裁判”的位置已经永久固定了。然后对左边那一堆和右边那一堆，重复上述过程。

- **代码实现**

  快排的核心在于 `partition` 函数。这里展示最经典的**双指针交换法**，也是原地排序的标准实现。

  ```go
  func QuickSort(arr []int, left, right int) {
  	// 递归终止条件：区间长度为 0 或 1
  	if left >= right {
  		return
  	}
  
  	// 获取分区点 (pivotIndex)
  	pivotIndex := partition(arr, left, right)
  
  	QuickSort(arr, left, pivotIndex-1)
  	QuickSort(arr, pivotIndex+1, right)
  }
  
  // 核心分区逻辑
  func partition(arr []int, left, right int) int {
  	pivot := arr[left] // 策略：默认选第一个元素做基准
  	i, j := left, right
  
  	for i < j {
  		// 右哨兵先走：寻找比 pivot 小的数
  		for i < j && arr[j] >= pivot {
  			j--
  		}
  		// 左哨兵后走：寻找比 pivot 大的数
  		for i < j && arr[i] <= pivot {
  			i++
  		}
  		// 如果还没相遇，交换这两个数
  		if i < j {
  			arr[i], arr[j] = arr[j], arr[i]
  		}
  	}
  
  	// 循环结束，i 和 j 相遇
  	// 将基准数放到这个相遇位置（它就是基准数的最终位置）
  	arr[left], arr[i] = arr[i], arr[left]
  
  	return i // 返回基准数的下标
  }
  ```

- **性能分析**

  - **时间复杂度**

    - **平均/最好**：$O(n \log n)$

    - **最坏**：$O(n^2)$

  - **空间复杂度**

    主要是递归调用栈的深度。

    - $O(\log n)$

  - **稳定性**

    **不稳定**。

    - 交换过程中可能会改变相等元素的相对顺序。

# 简单选择排序

每一次从待排序的数据元素中选出最小（或最大）的一个元素，存放在序列的起始位置，直到全部待排序的数据元素排完。

- **算法逻辑**

  将数组分为两个区间：**“已排序区间”** 和 **“未排序区间”**。

  1. **初始状态**：整个数组都是“未排序区间”，“已排序区间”为空。
  2. **查找最小**：在“未排序区间”中遍历，找到最小元素的位置（下标）。
  3. **交换位置**：将这个最小元素与“未排序区间”的第一个元素交换。
  4. **更新边界**：“已排序区间”长度加 1，“未排序区间”长度减 1。
  5. **重复**：重复上述过程，直到“未排序区间”只剩下一个元素（或为空），排序完成。

- **性能分析**

  - **时间复杂度**

    - 始终为 $O(n^2)$。无论输入数据是否有序，都要遍历剩余序列。

  - **空间复杂度**

    - $O(1)$。原地排序，不需要额外内存。

  - **稳定性**

    **不稳定**。

    - 例如序列 `[5, 5, 2]`，第一轮会将第一个 `5` 与 `2` 交换，导致两个 `5` 的顺序颠倒。

# 归并排序

- **算法逻辑**

  1. **分解** ：将当前长度为 $n$ 的序列从中间下标 $mid$ 处切分为两个长度约为 $n/2$ 的子序列。
  2. **递归** ：递归地对这两个子序列进行二路归并排序，直到子序列长度为 1（此时天然有序）。
  3. **合并**：这是算法的核心。使用**两个指针**分别指向两个有序子序列的起始位置，比较指针指向的元素，将较小者放入辅助数组，并移动指针。

- **性能分析**

  - **时间复杂度**

    - **最好/最坏/平均**：均为 $O(n \log n)$。
    - **分析**：递归树的高度为 $\log_2 n$，每一层合并操作的时间复杂度为 $O(n)$。由于其执行步骤不依赖于数据的初始分布，它的性能表现极其稳定。

  - **空间复杂度**

    - $O(n)$。在合并阶段需要申请一个与原数组等大的辅助空间。

  - **稳定性**

    **稳定**。
