# docker compose

## docker compose up

启动服务并构建镜像.

```shell
docker compose up [可选项] 

#可选项
-d		#在后台运行
--build	#强制重建镜像
-f		#指定yaml文件创建（若文件名为docker-compose.yml则不需要）
```

## docker compose down

停止并移除容器、网络、卷和镜像.

```shell
docker compose down [可选项] 

#可选项
-v			#同时删除卷
--rmi all	#同时删除所有相关镜像
```

# yaml文件语法

- **`version`:** 指定 Docker Compose 文件格式的版本.
- **`services`:** 定义你要运行的所有服务（即 Docker 容器）.
  - **`image`:** 使用的镜像名。如果镜像不存在，Compose 会自动从 Docker Hub 下载.
  - **`ports`**: 映射容器端口到宿主机端口.
  - **`volumes`:** 配置数据卷，用于容器数据持久化（可选）.
  - **`environment`**: 环境变量，通常用于传递配置.
  - **`networks`:** 配置容器的网络设置（可选）.

```yaml
version: ""

services:
  service1:
	image: 
	ports:
	volumes:
	environment:
	networks:
		
 service2:
	image:
	ports:
	volumes:
	environment:
	networks:
```



# wordpress + mysql

## 编写yaml文件

```yaml
version: "3.9"

services:
  db:
    image: e829314b93fd
    pull_policy: never
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: wordpress
      MYSQL_USER: dantalion
      MYSQL_PASSWORD: youwx2006
    volumes:
      - /home/dantalion/mysql_data:/var/lib/mysql
    networks:
      - test

  wordpress:
    image: c23b6f0d5357
    pull_policy: never
    ports:
      - "8080:80"
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: dantalion
      WORDPRESS_DB_PASSWORD: youwx2006
      WORDPRESS_DB_NAME: wordpress
    networks:
      - test
    depends_on:
      - db

networks:
  test:
    driver: bridge
```
