# docker commit

将容器提交为镜像.

## 格式

```shell
#命令和git原理相似
docker commit -a="作者" -m="描述性信息" [容器id] [目标镜像名]
```

## 示例

### 创建容器

```shell
#启动容器tomcat02，以tomcat为镜像
dantalion@Dantalion:~$ docker run -d -p 8080:8080 --name tomcat02 dcae2ebef501
5dadf419dd643dad6cf03b09cff21add51a24bd0bac55b8ebc136824e39e33b1

#查看正在运行的容器，获得容器id
dantalion@Dantalion:~$ docker ps
CONTAINER ID   IMAGE          COMMAND             CREATED         STATUS         PORTS                                         NAMES
5dadf419dd64   dcae2ebef501   "catalina.sh run"   5 seconds ago   Up 4 seconds   0.0.0.0:8080->8080/tcp, [::]:8080->8080/tcp   tomcat02

#利用已获得的容器id进入容器
dantalion@Dantalion:~$ docker exec -it 5dadf419dd64 /bin/bash

#列出容器中的文件
root@5dadf419dd64:/usr/local/tomcat# ls
bin           conf             filtered-KEYS  LICENSE  native-jni-lib  README.md      RUNNING.txt  upstream-KEYS  webapps.dist
BUILDING.txt  CONTRIBUTING.md  lib            logs     NOTICE          RELEASE-NOTES  temp         webapps        work

#将文件进行复制操作
root@5dadf419dd64:/usr/local/tomcat# cp -r webapps.dist/* webapps

#移动到webapps目录位置
root@5dadf419dd64:/usr/local/tomcat# cd webapps

#查看当前目录下的文件，确认是否复制成功（成功）
root@5dadf419dd64:/usr/local/tomcat/webapps# ls
docs  examples  host-manager  manager  ROOT
```

> [!NOTE]
>
> 此处启动的是官方默认的tomcat镜像，即原镜像，webapps目录下没有文件；
>
> 用户对原镜像建立容器，并且进行操作后产生新的“容器层”，通过`docker commit`能够将容器层+原镜像层打包成一个新镜像，此时得到的tomcat镜像在webapps目录下是有文件的.

### 提交为镜像

```shell
#提交
dantalion@Dantalion:~$ docker commit -a="dantalion" -m="add webapps" 5dadf419dd64 tomcat1.0
sha256:34f785b69c465d450840f0db2f92550823941373ae6661622eb8b0f80bfbbe57
#解释
-a="dantalion"		#镜像作者为dantalion
-m="add webapps"	#镜像描述：对webapps进行了添加操作

#确认是否提交成功（成功）
dantalion@Dantalion:~$ docker images
REPOSITORY                              TAG              IMAGE ID       CREATED         SIZE
tomcat1.0                               latest           34f785b69c46   2 minutes ago   482MB
```

> [!TIP]
>
> 如果想要保存一个容器某时刻的状态，可以通过docker commit命令将其提交为一个镜像，类似于给该时刻的状态拍摄快照.
