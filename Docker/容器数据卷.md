> [!IMPORTANT]
>
> 解决容器被删除则连带着导致容器数据消失的问题：
> 持久化保存数据，把数据同步到本地→实现容器之间数据共享.
>
> ↓
>
> 把容器中的数据挂载到虚拟机中.

# 使用数据卷

## -v命令挂载

### 格式

```shell
docker run -it -v [虚拟机目录]:[容器目录] [镜像id] /bin/bash
#相当于创建了一个新容器并且运行，同时将指定容器目录中的文件挂载到虚拟机目录下
```

### 示例

```shell
#挂载操作
docker run -it -v /home/test:/home 2cd1d97f893f /bin/bash
#基于nginx镜像创建新容器，且将该容器下的home目录挂载到虚拟机的test目录中；如果此目录不存在，会自行创建

#检查是否挂载成功（成功）,使用docker inspect命令查看容器的详细状态信息
docker inspect 96b71e3290cb
...
"Mounts": [
            {
                "Type": "bind",				
                "Source": "/home/test",			#主机地址
                "Destination": "/home",			#docker容器内的地址
                "Mode": "",
                "RW": true,
                "Propagation": "rprivate"
            }
        ]
...
```

> [!NOTE]
>
> 相当于对容器内的文件做一个映射，将其同步映射到虚拟机上.

### 测试1

在容器的home文件下新建一个文件，检查主机指定的目录下是否也同步产生该文件：

```shell
#容器内部终端
dantalion@Dantalion:/home$ docker exec -it 96b71e3290cb /bin/bash
root@96b71e3290cb:/# cd /home
root@96b71e3290cb:/home# touch test.py

#进入容器中的home目录后创建新文件test.py
```

```shell
#主机终端
dantalion@Dantalion:/home$ cd test
dantalion@Dantalion:/home/test$ ls
dantalion@Dantalion:/home/test$ ls
test.py

#访问主机的指定目录/home/test，一开始显示无文件，创建新文件后显示test.py，说明同步成功
```

### 测试2

在本地目录中修改文件，检查容器中的文件是否会同步发生更新：

```shell
#主机终端
#到达相应文件下，使用管理员权限，利用vim修改test.py；此处输入hello
dantalion@Dantalion:/home/test$ sudo vim test.py
```

```shell
#容器内部终端
#进入运行中的容器终端
dantalion@Dantalion:/$ docker attach 0c1beb49c601

#访问建立映射的目录
root@0c1beb49c601:/# cd /home
root@0c1beb49c601:/home# ls
test.py

#查看被修改过的文件（修改成功）
root@0c1beb49c601:/home# cat test.py
hello
```
