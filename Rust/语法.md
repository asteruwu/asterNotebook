# 基础语法

## 基本数据类型

### 标量类型

- **核心职责：表示单一值**
  - Rust 有四类标量：整数、浮点、布尔、字符。

- **整数**
  - 有符号：`i8` `i16` `i32` `i64` `i128` `isize`
  - 无符号：`u8` `u16` `u32` `u64` `u128` `usize`
  - 默认推导为 `i32`；`isize` / `usize` 宽度随平台（32/64位）变化。
    - usize 是 无符号整数类型，专门用于表示数组索引和内存大小。
  - 字面量支持下划线分隔（`1_000`）、十六进制（`0xff`）、二进制（`0b1010`）、八进制（`0o77`）、字节字面量（`b'A'`，仅 `u8`）。
  - 溢出行为：debug 模式 panic，release 模式按补码截断。

- **浮点**
  - `f32`（单精度）、`f64`（双精度），默认推导为 `f64`。
  - 遵循 IEEE 754 标准。

- **布尔**
  - `bool`，值为 `true` / `false`，占 1 字节。

- **字符**
  - `char`，单引号，占 **4 字节**，表示 Unicode 标量值。
  - 与 `&str` / `String` 不同，`char` 是单个 Unicode 字符。

### 复合类型

- **核心职责：将多个值组合为一个类型**

- **元组（Tuple）**
  - 固定长度，各元素类型可不同。
  - 用 `.索引` 访问：`t.0`、`t.1`。
  - 空元组 `()` 即**单元类型（unit type）**，函数无显式返回值时隐式返回它。

  ```rust
  fn main() {
    let tup = (true, 20, 1.5);
    let (result, num, rate) = tup;
    println!("result: {result}, num: {num}, rate: {rate}");
  }
  ```

- **数组（Array）**
  - 固定长度，所有元素类型必须相同，存储在**栈**上。
  - 类型标注形式：`[i32; 5]`（类型; 长度）。
  - 越界访问运行时 panic（编译期可检测的情况直接报错）。
  - 与 `Vec<T>`（堆上动态数组）的核心区别：长度编译期固定。

  ```rust
  fn main() {
    let arr: [i32; 3] = [1,2,3];
    let second = arr[1];
    println!("第2个元素是：{second}");
    println!("{}", arr[0]);
  }
  ```

## 变量与常量

### `let`

- **核心职责：定义变量绑定**
  - 使用 `let` 将名称绑定到一个值。
  - 默认**不可变**，后续不可直接重新赋值。
  - 编译器通常可自动推导类型。

- **基础用法**

    ```rust
    let x = 10;
    let name = "Rust";
    ```

- **设计特征**
  - **默认不可变**：减少无意状态修改。
  - **作用域明确**：变量通常只在当前代码块内有效。
  - **支持遮蔽（Shadowing）**：可使用新的 `let` 重新定义同名变量。

### `mut`

- **核心职责：显式声明可变性**
  - `mut` 配合 `let` 使用，使绑定后的值允许修改。
  - 体现 Rust 的显式可变设计：默认安全，按需放开。

- **基础用法**
  
  ```rust
  let mut x = 10;
  x = 20;
  ```

- **使用边界**
  - 未声明 `mut` 的变量不可重新赋值。
  - `mut` 修改的是**同一个绑定**，不是创建新变量。

### `const`

- **核心职责：定义编译期常量**
  - `const` 用于声明在编译阶段即可确定的固定值。
  - 常量永远不可变，不能配合 `mut` 使用。

- **语法约束**
  - **必须显式标注类型**。
  - 值必须是**常量表达式**，不能依赖运行时结果。
  - 可定义在全局作用域，也可定义在局部作用域。
  - 命名通常采用**全大写下划线**风格。

- **基础用法**
  
  ```rust
  const MAX_POINTS: u32 = 100_000;
  const SECONDS_PER_HOUR: u32 = 60 * 60;
  ```

### 遮蔽（Shadowing）

- **核心职责：创建同名新绑定**
  - 使用新的 `let` 重新定义同名变量。
  - 适合阶段性处理或类型转换。

- **示例**
  
  ```rust
  let x = 5;
  
  {
    let x = x + 1;
  }
  
  {
    let x = x * 2;
  }
  ```

## 控制流

### `if` 表达式

- **核心职责：条件分支**
  - `if` 是**表达式**，可直接用于 `let` 赋值。
  - 条件必须是 `bool`，不做隐式转换。
  - 各分支返回值类型必须一致。

  ```rust
  let number = 6;
  let result = if number % 2 == 0 { "偶数" } else { "奇数" };
  ```

### 循环

- **核心职责：重复执行代码块**
  - Rust 有三种循环：`loop`、`while`、`for`。

- **`loop`**
  - 无限循环，必须用 `break` 退出。
  - `break` 可携带返回值，使 `loop` 成为表达式。
  - 支持**循环标签**（`'label:`），用于多层嵌套时精确指定 `break` / `continue` 作用于哪一层循环。

  ```rust
  // break 携带返回值
  let mut count = 0;
  let result = loop {
      count += 1;
      if count == 10 { break count * 2; }
  };
  
  // 循环标签：break 外层循环
  'outer: for i in 0..5 {
      for j in 0..5 {
          if i + j == 6 { break 'outer; }
      }
  }
  ```

- **`while`**
  - 条件为 `true` 时持续执行，不能作为表达式返回值（返回 `()`）。

  ```rust
  let mut n = 3;
  while n != 0 {
      n -= 1;
  }
  ```

- **`for`**
  - 遍历迭代器，是 Rust **最惯用的循环方式**，不存在越界风险。
  - 常配合 `Range`（`1..5`、`1..=5`）使用。

  ```rust
  for i in 1..=5 {
      println!("{i}");
  }
  ```

### `match` 表达式

- **核心职责：穷举模式匹配**
  - 编译器强制覆盖所有分支，用 `_` 作通配符兜底。
  - `match` 是表达式，可赋值给变量；各分支返回值类型必须一致。
  - 分支语法：`模式 => 表达式`，多条语句用 `{}` 包裹。
  - 支持多模式（`|`）、范围模式（`4..=9`）。

  ```rust
  let x = 2;
  let desc = match x {
      1 => "一",
      2 | 3 => "二或三",
      4..=9 => "四到九",
      _ => "其他",
  };
  ```

## 函数定义与返回值

### 函数定义

- **核心职责：封装可复用逻辑单元**
  - 关键字 `fn`，命名使用**蛇形命名法**（`snake_case`）。
  - 参数**必须显式标注类型**，无默认参数。
  - 函数体末尾**表达式**（无分号）即为返回值；末尾**语句**（有分号）返回 `()`。

  ```rust
  fn add(a: i32, b: i32) -> i32 {
      a + b  // 表达式，无分号，作为返回值
  }
  ```
  类似地，也可以用表达式给变量赋值。
  ```rust
  let y = {
    let x = 5;
    x + 1
  };
  ```

### 返回值

- **核心职责：向调用方传递结果**
  - 返回类型用 `->` 标注；无返回值时隐式返回 `()`，可省略 `->`。
  - 提前返回使用 `return` 关键字（带分号）。
  - 多返回值用**元组**模拟。

  ```rust
  fn is_even(n: i32) -> bool {
      if n % 2 == 0 {
          return true;  // 提前返回
      }
      false  // 末尾表达式返回
  }
  
  fn min_max(a: i32, b: i32) -> (i32, i32) {
      (a.min(b), a.max(b))
  }
  ```

## 注释

- **行注释**：`//`，最常用。
- **块注释**：`/* */`，可嵌套，Rust 中较少使用。
- **文档注释**：
  - `///`：注释紧跟其后的条目（函数、结构体等），生成 API 文档。
  - `//!`：注释当前所在的模块或文件本身。
  - 内容支持 Markdown，通过 `cargo doc` 生成 HTML 文档。

  ```rust
  //! 本模块提供数学工具函数
  
  /// 返回两数之和
  fn add(a: i32, b: i32) -> i32 {
      a + b
  }
  ```

# 所有权

- **核心职责：编译期保证内存安全，无需 GC**
  - 每个值有且只有**一个所有者**（变量）。
  - 同一时刻只能有一个所有者。
  - 所有者离开作用域时，值被**自动释放**（调用 `drop`）。

  ```rust
  {
      let s = String::from("hello"); // s 是所有者
  } // s 离开作用域，内存自动释放
  ```

  > 栈上固定大小类型（整数、bool 等）无需所有权管理；堆上数据（`String`、`Vec` 等）是所有权的主要管理对象。

## 移动（Move）与复制（Copy）

- **移动（Move）**
  - 堆上数据赋值给另一个变量时，所有权**转移**，原变量失效。

  ```rust
  let s1 = String::from("hello");
  let s2 = s1;         // 所有权移动到 s2
  // println!("{s1}"); // 编译错误：s1 已失效
  ```

- **复制（Copy）**
  - 实现了 `Copy` trait 的类型，赋值时**按位复制**，原变量仍有效。
  - 实现 `Copy` 的类型：所有整数、浮点、布尔、`char`；元素全为 `Copy` 类型的元组与固定大小数组。
  - `String`、`Vec`、自定义结构体（默认）均**不实现 `Copy`**，赋值即移动。

  ```rust
  let x = 5;
  let y = x;  // x 被复制，x 和 y 都有效
  ```

## 借用（Borrowing）

- **核心职责：临时访问值而不转移所有权**

- **不可变引用 `&T`**：只读访问，可同时存在多个。

  ```rust
  fn print_len(s: &String) {
      println!("{}", s.len());
  }
  let s = String::from("hello");
  print_len(&s);  // 借用，s 仍有效
  ```

- **可变引用 `&mut T`**：允许修改，同一作用域内只能有一个。

  ```rust
  fn push(s: &mut String) {
      s.push_str(" world");
  }
  let mut s = String::from("hello");
  push(&mut s);
  ```

- **借用规则（编译期强制）**
  - `&T` 可同时存在**多个**。
  - `&mut T` 同一作用域内**只能有一个**。
  - `&T` 与 `&mut T` **不能同时存在**。

## Slice

- **核心职责：对连续数据的不持有所有权的引用视图**
  - 本质是胖指针（指针 + 长度），借用规则同样适用。

- **字符串 Slice `&str`**
  - 对 `String` 或字符串字面量的部分/全部内容进行引用。
  - 范围语法：`[start..end]`（左闭右开）；`..end`、`start..`、`..` 分别省略起始/结尾/全取。
  - 字符串字面量本身类型即为 `&str`，指向二进制数据段，天然不可变。
  - `&String` 可自动强转为 `&str`（Deref 强制转换），函数参数推荐用 `&str`，兼容性更广。

  ```rust
  fn main() {
  let s = String::from("hello world");
  let res = first_word(&s);
  println!("{}", res);
  }
  
  fn first_word(s: &str) -> &str {
    // 用 as_bytes 方法把 String 转换成字节数组并通过 iter 创建一个迭代器；
    // 用 enumerate 包装 iter 的结果，把每个元素作为元组的一部分返回
    let bytes = s.as_bytes().iter().enumerate(); 
    for (i, &item) in bytes {
        if item == b' ' {
            return &s[..i];
        }
    };
    &s
  }
  ```

- **数组 Slice `&[T]`**
  - 对数组或 `Vec<T>` 的部分/全部元素进行引用，类型为 `&[T]`。
  - 函数参数优先用 `&[T]`，可同时接受数组引用与 `Vec` 引用。

  ```rust
  fn sum(nums: &[i32]) -> i32 {
      nums.iter().sum()
  }
  
  let arr = [1, 2, 3];
  let vec = vec![1, 2, 3];
  sum(&arr);  // 数组引用自动转为 &[i32]
  sum(&vec);  // Vec 引用自动转为 &[i32]
  ```

题目：**统计句子中的单词数量**

实现一个函数 word_count，接收一个字符串 slice，返回其中单词的数量（以空格分隔）。在 main 中调用它，并满足以下要求：

1. 函数参数使用 &str（而非 &String）
2. 用 for 循环遍历字节，手动统计空格数来推算单词数
3. 用 match 对结果进行描述：0 个词输出 "空字符串"，1 个词输出 "只有一个词"，其余输出 "共 N 个词"

```rust
use std::io;

fn main() {
    println!("请输入单词：");
    let mut word = String::new();

    io::stdin()
        .read_line(&mut word)
        .expect("Failed to read line");

    let count = word_count(word.trim());

    let res = match count {
        0 => String::from("空字符串"),
        1 => String::from("只有一个词"),
        n => format!("共{}个词",n),
    };

    println!("{}", res);
}

fn word_count(s: &str) -> i32 {
    let bytes = s.as_bytes().iter();
    let mut flag = false;
    let mut count: i32 = 0;
    for &item in bytes {
        if item == b' ' {
            if flag {
                count += 1;
                flag = false;
            }
        } else {
            flag = true;
        }
    };
    if flag {
        count += 1;
    }
    count
}
```

# 核心数据结构与标准库

## 集合类型

### Vec\<T\>

- **核心职责：堆分配的动态数组，长度运行时可变**
  - 内部结构：堆指针 + 长度（len）+ 容量（capacity）。
  - **所有元素类型必须一致**；离开作用域自动释放。

- **容量与扩容**
  
  - `len == capacity` 时再 push 触发重新分配（通常翻倍），代价高。
  - 已知规模时用 `Vec::with_capacity(n)` 预分配。
  
- **创建方式**

  ```rust
  let v1: Vec<i32> = Vec::new();       // 空 Vec，需类型标注
  let v2 = vec![1, 2, 3];              // 宏字面量
  let v3 = vec![0; 5];                 // [0,0,0,0,0]
  let v4 = Vec::with_capacity(10);     // 预分配容量
  ```

- **常用操作**

  - **元素更新** 
    - `v.push(x)`：在末尾追加元素。
    - `v.pop()`：移除并返回末尾元素，返回 `Option<T>`（空时为 `None`）。
    - `v.insert(i, x)`：在索引 `i` 处插入元素，后续元素依次右移。
    - `v.remove(i)`：移除索引 `i` 处的元素并返回它，后续元素依次左移。
    - `v.clear()`：移除所有元素，长度归零，容量不变。

  - **访问**

    - `v[i]`：按索引直接访问，越界时 panic。
    - `v.get(i)`：按索引安全访问，返回 `Option<&T>`，越界时返回 `None` 而非 panic。

  - **状态查询**
    - `v.len()`：返回当前元素个数。
    - `v.is_empty()`：判断是否为空。

- **遍历**

  ```rust
  for x in &v     { /* &T，不可变借用 */ }
  for x in &mut v { *x += 1; /* &mut T，可变借用 */ }
  for x in v      { /* T，所有权移动，循环后 v 失效 */ }
  ```

- **借用规则的体现**
  - 持有 `&v[0]` 时不能 `v.push(...)`：push 可能触发扩容导致旧引用悬垂，编译期报错。

- **与数组 `[T; N]` 的核心区别**
  - 数组长度编译期固定、存栈；`Vec` 长度运行时可变、存堆。
  - 函数参数推荐统一用 `&[T]`（切片），兼容数组引用与 Vec 引用。

### HashMap<K, V>

- **核心职责：基于哈希表的键值对集合，O(1) 平均查找**
  - 需 `use std::collections::HashMap` 引入。
  - Key 必须实现 `Eq + Hash`；元素无固定顺序。

- **创建方式**

  ```rust
  use std::collections::HashMap;

  let mut map = HashMap::new();
  map.insert(String::from("blue"), 10);
  ```

- **所有权行为**
  - `Copy` 类型（如 `i32`）插入时复制值。
  - 非 `Copy` 类型（如 `String`）插入后所有权转移给 HashMap，原变量失效。

- **常用操作**

  - **插入与删除**
    - `map.insert(k, v)`：插入键值对；key 已存在则覆盖旧值，返回 `Some(旧值)`。
    - `map.remove(&k)`：移除键值对，返回 `Option<V>`。

  - **访问**
    - `map.get(&k)`：查询，返回 `Option<&V>`。
    - `map.get_mut(&k)`：返回 `Option<&mut V>`，可就地修改 value。

  - **状态查询**
    - `map.contains_key(&k)`：判断 key 是否存在。
    - `map.len()` / `map.is_empty()`

- **条件插入：`entry` API**
  - `map.entry(k)`：接收一个 key，返回 `Entry` 枚举，表示该 key 在 map 中的"槽位"状态（已占用 `Occupied` 或空闲 `Vacant`）。
  - `.or_insert(default)`：若槽位空闲，插入 `default` 作为 value；若已占用则不动。**无论哪种情况，都返回该 value 的 `&mut V`**。
  - `.or_insert_with(|| expr)`：同上，但 default 通过闭包惰性计算，仅在需要插入时才执行。
  - 返回的 `&mut V` 可直接解引用修改，适合累加、计数等场景。

  ```rust
  // key 不存在 → 插入 50；已存在 → 不动；均返回 &mut V
  map.entry(String::from("blue")).or_insert(50);
  
  // 词频统计：or_insert(0) 保证首次出现时初始化为 0，返回 &mut i32 用于累加
  let text = "hello world hello";
  let mut freq = HashMap::new();
  for word in text.split_whitespace() {
      let count = freq.entry(word).or_insert(0);
      *count += 1;
  }
  ```

- **遍历**

  ```rust
  for (k, v) in &map {
      println!("{k}: {v}");
  }
  ```

### HashSet\<T\>

- **核心职责：无重复元素的集合，O(1) 判断存在性**
  - 底层为 `HashMap<T, ()>`，只存 Key。
  - 需 `use std::collections::HashSet` 引入。
  - 元素必须实现 `Eq + Hash`；无固定顺序；元素唯一。

- **创建方式**

  ```rust
  use std::collections::HashSet;

  let mut set = HashSet::new();
  set.insert("apple");
  set.insert("apple");  // 重复，不生效

  // 从迭代器收集（自动去重）
  let set: HashSet<i32> = vec![1, 2, 2, 3].into_iter().collect();
  ```

- **常用操作**

  - **插入与删除**
    - `set.insert(v)`：插入元素；已存在返回 `false`，不存在则插入并返回 `true`。
    - `set.remove(&v)`：移除元素，成功返回 `true`。

  - **查询**
    - `set.contains(&v)`：判断元素是否存在。
    - `set.len()` / `set.is_empty()`

- **集合运算**
  - `a.intersection(&b)`：交集，两者都有的元素。
  - `a.union(&b)`：并集，合并去重。
  - `a.difference(&b)`：差集，a 有但 b 没有。
  - `a.symmetric_difference(&b)`：对称差集，只在其中一方出现。
  - 以上均返回迭代器，可用 `.collect::<HashSet<_>>()` 收集为新 Set。

  ```rust
  let a: HashSet<i32> = vec![1, 2, 3].into_iter().collect();
  let b: HashSet<i32> = vec![2, 3, 4].into_iter().collect();
  let inter: HashSet<&i32> = a.intersection(&b).collect();
  ```

## 结构体（Struct）

- **核心职责：将多个相关字段组合为自定义类型**
  - 每个字段有独立名称和类型；结构体拥有其字段的所有权。
  - 若字段要存引用，必须标注生命周期。

- **三种形式**

  1. **普通结构体（命名字段）**

     ```rust
     struct User {
         name: String,
         age: u32,
         active: bool,
     }

     let user = User {
         name: String::from("Alice"),
         age: 25,
         active: true,
     };
     println!("{}", user.name);  // 用 . 访问字段
     ```

  2. **元组结构体（无命名字段）**
  
      用 `.0` `.1` 访问，适合给元组赋予类型语义（newtype 模式）。

     ```rust
     struct Color(u8, u8, u8);
     let red = Color(255, 0, 0);
     ```

  3. **单元结构体（无字段）**
     
     不存储数据，通常用于实现 trait 而不需要关联数据。

     ```rust
     struct Marker;
     ```

- **字段简写与结构体更新语法**

  ```rust
  // 变量名与字段名相同时可省略冒号
  let name = String::from("Bob");
  let user = User { name, age: 30, active: true };

  // 从已有实例复制未显式赋值的字段
  let user2 = User {
      name: String::from("Carol"),
      ..user  // age、active 取自 user
  };
  // 若复制了非 Copy 字段（如 String），原实例对应字段所有权转移
  ```

- **方法与关联函数（`impl` 块）**

  ```rust
  struct Rect {
      width: f64,
      height: f64,
  }

  impl Rect {
      // 关联函数（无 self），用 :: 调用
      fn square(size: f64) -> Self {
          Self { width: size, height: size }
      }

      // 方法（&self），只读访问
      fn area(&self) -> f64 {
          self.width * self.height
      }

      // 方法（&self + 其他参数），在 self 后增加多个参数
      fn can_hold(&self, other: &Rect) -> bool {
        self.width > other.width && self.height > other.height
    }

      // 方法（&mut self），可修改实例
      fn scale(&mut self, factor: f64) {
          self.width *= factor;
          self.height *= factor;
      }
  }

  let mut r = Rect::square(5.0);
  println!("{}", r.area());
  r.scale(2.0);
  ```

- **`self` 的三种形式**
  - `&self`：不可变借用实例，最常用。
  - `&mut self`：可变借用实例，需要修改字段时使用。
  - `self`：获取所有权，调用后实例失效（常用于转换/消耗场景）。

## 枚举（Enum）

- **核心职责：定义一个类型，其值为若干变体（variant）之一**
  - 每个变体可关联不同类型和数量的数据。
  - 可通过 `impl` 块为枚举实现方法。

- **基础定义**

  ```rust
  enum Direction {
      Up,
      Down,
      Left,
      Right,
  }

  let d = Direction::Up;
  ```

- **带数据的变体**

  ```rust
  enum Message {
      Quit,                       // 无数据（类似单元结构体）
      Move { x: i32, y: i32 },   // 命名字段（类似普通结构体）
      Write(String),              // 单个值（类似元组结构体）
      Color(u8, u8, u8),          // 多个值
  }

  let msg = Message::Write(String::from("hello"));
  ```

- **为枚举实现方法**

  ```rust
  impl Message {
      fn call(&self) {
          // 用 match 匹配 self 的变体进行处理
      }
  }
  msg.call();
  ```

### Option\<T\>

- **核心职责：显式表达"值可能不存在"，替代 null**
  - 定义在标准库，自动引入无需 `use`。
  - `Option<T>` 和 `T` 是不同类型，不能直接混用运算，必须先提取内部值。

  ```rust
  enum Option<T> {
      Some(T),  // 有值
      None,     // 无值
  }
  
  let x: Option<i32> = Some(5);
  let y: Option<i32> = None;
  ```

- **常见提取方式**
  - `match` 匹配：穷举 `Some(v)` 和 `None` 分支。
  - `.unwrap()`：有值返回，`None` 时 panic（仅调试用）。
  - `.unwrap_or(default)`：`None` 时返回默认值。
  - `.is_some()` / `.is_none()`：判断状态。

### Result<T, E>

- **核心职责：表达操作"成功或失败"，是错误处理的基石**

  ```rust
  enum Result<T, E> {
      Ok(T),   // 成功，携带结果
      Err(E),  // 失败，携带错误信息
  }
  
  use std::fs::File;
  let f = File::open("hello.txt"); // 返回 Result<File, io::Error>
  match f {
      Ok(file) => { /* 使用 file */ },
      Err(e) => { println!("打开失败: {e}"); },
  }
  ```

## 模式匹配深入

- **核心职责：对任意结构进行拆包、过滤和绑定**
  - 模式可出现在 `match` 分支、`let` 绑定、函数参数、`if let`、`while let`、`for` 循环中。

### 解构

可以使用模式来解构结构体、枚举和元组，以便使用这些值的不同部分。主要的动作是**将复合类型内部字段一次性提取到独立变量**。

- **结构体**
  
  展示带有两个字段 x 和 y 的结构体 Point，可以通过带有模式的 let 语句将其分解：
  ```rust
  struct Point {
    x: i32,
    y: i32,
  }
  
  fn main() {
      let p = Point { x: 0, y: 7 };
  
      // 创建变量 a 和 b 来匹配结构体 p 中的 x 和 y 字段
      let Point { x: a, y: b } = p;
      assert_eq!(0, a);
      assert_eq!(7, b);
  }
  ```

- **枚举**
  
  在 match 中对每个变体拆出其关联数据，按变体形式（元组/结构体）提取。
  ```rust
  enum Shape {
    Circle(f64),
    Rect { w: f64, h: f64 },
  }
  
  let s = Shape::Rect { w: 3.0, h: 4.0 };
  match s {
      // s 是 Circle 变体时，把内部的 f64 值取出绑定到 r
      Shape::Circle(r) => println!("半径 {r}"),
      // s 是 Rect 变体时，把 w 和 h 两个字段分别取出
      Shape::Rect { w, h } => println!("面积 {}", w * h),
  }
  ```

- **元组**
  
  按位置提取元素，不需要的可以用 _ 占位忽略。
  ```rust
  let (a, b, _) = (1, 2, 3);  // a=1, b=2，第三个值丢弃
  ```

### 守卫

匹配守卫是一个**指定于 match 分支模式**之后的额外 if 条件，用于做**二次过滤**。它也必须被满足才能选择此分支。
```rust
let num = 4;
match num {
    n if n < 0 => println!("负数"),
    n if n == 0 => println!("零"),
    n => println!("正数: {n}"),
}
// num 被绑定到 n，然后逐分支用 if 判断 n 的范围
```

### @绑定

在创建一个存放值的变量的同时测试其值是否匹配模式。

下例希望测试 `Message::Hello` 的 `id` 字段是否位于 `3..=7` 范围内，同时也希望能将其值绑定到 `id_variable` 变量中以便此分支相关联的代码可以使用它。
```rust
enum Message {
    Hello { id: i32 },
}

let msg = Message::Hello { id: 5 };

match msg {
    Message::Hello {
        id: id_variable @ 3..=7,
    } => {
        println!("Found an id in range: {id_variable}")
    }
    Message::Hello { id: 10..=12 } => {
        println!("Found an id in another range")
    }
    Message::Hello { id } => println!("Found some other id: {id}"),
}
```
在这个例子中，如果 `x` 的值在 1-12 的范围内，则将实际值绑定给 `n`。
```rust
let x = 15;
match x {
    n @ 1..=12 => println!("月份: {n}"),
    n @ 13..=19 => println!("青少年年龄: {n}"),
    n => println!("其他: {n}"),
}
```
也可以解构结构体中的字段，然后作范围判断，将值绑定到变量：
```rust
enum Event {
    Click { x: i32, y: i32 },
}

let e = Event::Click { x: 10, y: 20 };

match e {
    // 解构 Click 的 x 字段，同时用 @ 确认其在 0~100 范围并绑定到 vx
    Event::Click { x: vx @ 0..=100, y } => println!("有效点击 x={vx}, y={y}"),
    _ => println!("超出范围"),
}
```

# 错误处理

## `panic!` 与不可恢复错误

- **触发方式**
  - 显式调用 panic!("错误信息")
  - 隐式触发：数组越界、unwrap() 遇到 None / Err 等

- **程序行为**
  - **unwind（默认）**：沿调用栈逐层回退，执行每层的析构函数（drop）清理资源，然后退出
  - **abort**：可在 `Cargo.toml` 中设为 abort 模式，程序会直接终止进程，不回退，二进制更小
    ```toml
    [profile.release]
    panic = "abort"
    ```
- **使用场景**
  - 原型开发时用 `unwrap()` / `expect("msg")` 快速跑通逻辑：
    - `unwrap()` / `expect("msg")`
      - 有值（Some / Ok）时直接返回内部值；
      - 无值（None / Err）时直接 panic，错误信息是固定的默认提示。
      ```rust
      let x: Option<i32> = Some(5);
      let v = x.unwrap();  // v = 5
      
      let y: Option<i32> = None;
      let v = y.unwrap();  // panic: called `Option::unwrap()` on a `None` value
      ```
      `expect("msg")` 与 `unwrap()` 基本一致。区别是 panic 时会打印自定义的信息 msg。
  - **正式代码中，可恢复的错误应该用 Result，而不是 panic。**
  - **示例**
    ```rust
    // 显式 panic
    panic!("crash and burn");
    
    // unwrap：None 时 panic
    let x: Option<i32> = None;
    x.unwrap();  // panic: called `Option::unwrap()` on a `None` value
    
    // expect：panic 时带自定义信息，比 unwrap 更易调试
    x.expect("x 不应该为空");
    ```

## `Result` 与可恢复错误

- **定义**
  
  有如下两个变体，Ok 和 Err：
  ```rust
  enum Result<T, E> {
      Ok(T),
      Err(E),
  }
  ```
  其中 `T` 代表成功时返回的 `Ok` 变体中的数据的类型， `E` 代表失败时返回的 `Err` 变体中的错误的类型。

- **示例**

  ```rust
  use std::fs::File;
  
  let f = File::open("data.txt");
  
  // 方式一：match 穷举
  let file = match f {
      Ok(file) => file,
      Err(e) => panic!("无法打开: {e}"),
  };
  
  // 方式二：unwrap / expect（快捷但粗暴，失败直接 panic）
  let file = File::open("data.txt").unwrap();
  let file = File::open("data.txt").expect("打开 data.txt 失败");
  
  // 方式三：unwrap_or_else（失败时执行闭包，灵活处理）
  let file = File::open("data.txt").unwrap_or_else(|e| {
      panic!("自定义处理: {e}");
  });
  ```
  也可以去进一步**匹配不同的错误**：
  ```rust
  use std::fs::File;
  use std::io::ErrorKind;
  
  fn main() {
      let greeting_file_result = File::open("hello.txt");
  
      let greeting_file = match greeting_file_result {
          Ok(file) => file,
          // 调用 error.kind() 方法来获取错误类型
          Err(error) => match error.kind() {
              // 如果错误类型是 NotFound，则创建文件并返回
              ErrorKind::NotFound => match File::create("hello.txt") {
                  Ok(fc) => fc,
                  // 文件不能被创建时会另外返回错误信息
                  Err(e) => panic!("Problem creating the file: {e:?}"),
              },
              // 否则 panic
              _ => {
                  panic!("Problem opening the file: {error:?}");
              }
          },
      };
  }
  ```
  如果使用 `unwrap_or_else()` 方法，代码如下：
  ```rust
  use std::fs::File;
  use std::io::ErrorKind;
  
  fn main() {
      let greeting_file = File::open("hello.txt").unwrap_or_else(|error| {
          // 匹配错误类型
          if error.kind() == ErrorKind::NotFound {
              File::create("hello.txt").unwrap_or_else(|error| {
                  // 返回文件创建错误
                  panic!("Problem creating the file: {error:?}");
              })
          } else {
              panic!("Problem opening the file: {error:?}");
          }
      });
  }
  ```

## ? 操作符传播错误

- **语法**

  - 放在返回 `Result` 或 `Option` 的表达式后面；
  - **`Ok(v)` 时提取 `v` 继续执行；`Err(e)` 时立即从当前函数返回 `Err(e)`**；
  - 使用 ? 的函数返回类型必须是 `Result`（或 `Option`）。

- **示例**
  
  ```rust
  use std::fs::File;
  use std::io::{self, Read};
  
  // 失败则直接 return err
  fn read_username_from_file() -> Result<String, io::Error> {
      let mut username_file = File::open("hello.txt")?;
      let mut username = String::new();
      username_file.read_to_string(&mut username)?; // 读出文件内容，追加写入 string
      Ok(username)
  }
  ```
  参数是 `&mut String` 的原因：

    - `read_to_string` 需要修改这个 `String`；
    - **修改一个值必须持有它的可变引用**，所以传 `&mut username`；
    - 函数不需要拥有 `username` 的所有权，因为后续还需要使用（作为返回值）。

  此处也可以是**链式调用**：
  ```rust
  use std::fs::File;
  use std::io::{self, Read};
  
  fn read_username_from_file() -> Result<String, io::Error> {
      let mut username = String::new();
  
      File::open("hello.txt")?.read_to_string(&mut username)?;
  
      Ok(username)
  }
  ```

# Trait 与泛型

## 泛型

- **核心职责：以类型参数替代具体类型，实现逻辑复用**

  类型参数在函数签名、结构体定义中作占位符，调用/实例化时由编译器推导或显式指定为具体类型。

- **泛型函数**

  ```rust
  fn largest<T: PartialOrd>(a: T, b: T) -> T {
      if a > b { a } else { b }
  }
  ```

- **泛型结构体**

  ```rust
  struct Point<T> {
      x: T,
      y: T,
  }

  // 推断为 int 类型
  let int_point = Point { x: 5, y: 10 };
  // 推断为 float 类型
  let float_point = Point { x: 1.0, y: 4.0 };
  ```

- **单态化**
  
  编译期为每个实际使用的具体类型独立生成一份专用代码，泛型参数在生成阶段被完全替换为具体类型。
  
  - **性能特征**：运行时无类型判断开销，等价于手写多份具体类型代码；代价是编译产物体积随实例化类型数量增长。
  - 与 `dyn Trait` 的运行时动态分发相对，属于**静态分发**。

## Trait

- **核心职责：定义一组类型必须实现的行为契约**

  只声明方法签名，不关心实现者的内部数据结构。

- **定义与实现**

  ```rust
  trait Summary {
      fn summarize(&self) -> String;
  }

  struct Article {
      title: String,
  }

  // 自定义实现
  impl Summary for Article {
      fn summarize(&self) -> String {
          format!("标题：{}", self.title)
      }
  }
  ```

- **默认方法**

  Trait 方法可提供默认实现，实现者可直接复用，也可在 `impl` 块中重写覆盖。

  ```rust
  trait Summary {
      fn summarize(&self) -> String {
          String::from("(默认摘要)")
      }
  }

  impl Summary for Article {}  // 复用默认实现
  ```

- **常用标准 Trait**
  
  - `Display`：面向用户输出，决定 `{}` 格式化行为，需手动实现。
  - `Debug`：面向调试输出，决定 `{:?}` 格式化行为，通常通过 `#[derive(Debug)]` 自动生成。
  - `Clone`：提供显式 `.clone()` 深拷贝能力，绕开所有权移动限制。
  - `Iterator`：赋予类型被 `for` 遍历及链式调用 `map`/`filter` 等方法的能力。

### Trait bounds

- **核心职责：约束泛型参数必须实现指定 trait**

  编译期验证调用方传入的具体类型是否满足约束，避免生成非法单态化代码。

- **`T: Trait` 语法**

  ```rust
  // 只有实现了 PartialOrd 方法的类型可作为函数参数
  fn largest<T: PartialOrd>(a: T, b: T) -> T { ... }
  ```

- **`where` 子句**

  约束数量增多时，将约束从 `<>` 中抽离至函数签名后集中声明，提升可读性，功能与内联约束完全等价。

  ```rust
  fn some_function<T, U>(t: T, u: U) -> i32
  where
      T: Display + Clone,
      U: Clone + Debug,
  { ... }
  ```

### Trait 对象

- **核心职责：以运行时多态突破泛型的单一类型限制**

  具体类型在编译期被抹除，仅保留其满足的 trait 约束，通过胖指针（数据指针 + 方法表指针）在运行时定位实现。

- **`dyn Trait`**

  ```rust
  // item 的具体类型在编译期被抹除，仅保留“实现了 Summary”这一约束
  fn notify(item: &dyn Summary) {
      println!("{}", item.summarize());
  }

  // Article、Tweet 是两种不同的具体类型，通过 Box<dyn Summary> 统一装箱后混装
  let items: Vec<Box<dyn Summary>> = vec![
      Box::new(article),
      Box::new(tweet),
  ];
  ```

  - 类型大小编译期**不确定**，必须通过指针使用：`&dyn Trait` 或 `Box<dyn Trait>`。可类比 go 中的接口。
  - 方法调用经由方法表间接寻址，属于**动态分发**。

- **与泛型的核心区别**

  | | 泛型（`T: Trait`） | Trait 对象（`dyn Trait`） |
  | --- | --- | --- |
  | 类型确定时机 | 编译期（单态化） | 运行时 |
  | 分发方式 | 静态分发，零开销 | 动态分发，方法表寻址开销 |
  | 混装不同具体类型 | 不支持，单个泛型参数对应唯一具体类型 | 支持，只需满足同一 trait |
  | 编译产物 | 随实例化类型数量增长 | 单份代码，体积更小 |

# 模块与包管理

## 模块系统

- **核心职责：组织代码的命名空间与可见性边界**

- **`mod`**：声明模块，将相关函数/结构体/枚举打包进独立命名空间，用 `::` 访问内部项。

  ```rust
  mod network {
      fn connect() {
          println!("连接中...");
      }
  }

  fn main() {
      network::connect();
  }
  ```

- **`pub`**：显式声明可见性
  - 模块内容默认**私有**，仅本模块及子模块可访问，跨模块访问必须显式标注 `pub`。
  - 收紧默认可见性，强制主动暴露需要暴露的接口，避免内部实现细节被外部随意依赖。

  ```rust
  mod network {
      pub fn connect() {
          println!("连接中...");
      }
  }
  ```

- **`use`**：将路径引入当前作用域，减少重复书写完整路径，不影响模块本身的组织结构。

  ```rust
  use network::connect;
  
  fn main() {
      connect();
  }
  ```

### 路径表示

- **核心职责：定位模块树中某一项的位置**
  
  路径分为绝对路径与相对路径两类，均用 `::` 分隔层级。

- **绝对路径 `crate::`**
  - 从当前 crate 根开始，路径固定不变，不随书写位置改变。
  - 适合引用项目里位置稳定、不常挪动的公共模块。

  ```rust
  crate::network::client::connect();
  ```

- **相对路径 `self`**
  
  显式表示"从当前模块开始"，效果等价于不写前缀直接给路径，多用于强调意图。

- **相对路径 `super`**
  - 表示当前模块的**上一级**，可叠加多层（`super::super::xxx`），但层级过深通常说明模块拆分需要重新设计。
  - 适合子模块访问父模块内容，路径不依赖模块树在 crate 中的绝对位置。

  ```rust
  mod network {
      fn log(msg: &str) {
          println!("[network] {msg}");
      }
  
      mod client {
          pub fn connect() {
              super::log("connecting...");  // 跳出 client，回到父模块 network
          }
      }
  }
  ```

- **绝对路径与 `super` 的核心区别**

  | | 绝对路径（`crate::`） | 相对路径（`super`） |
  | --- | --- | --- |
  | 起点 | crate 根 | 当前模块的父级 |
  | 稳定性 | 模块内容挪动位置不受影响 | 模块树被整体移动时相对关系不受影响 |
  | 适用场景 | 引用位置固定的公共模块 | 子模块访问父模块内容 |

### 文件与目录的模块映射

- **核心职责：以文件系统结构承载模块树结构**
  
  模块与文件按固定命名约定一一对应，无需额外配置扫描规则。

- **映射规则**
  - `mod xxx;`（无大括号）在同级目录下寻找 `xxx.rs` 作为该模块的内容。
  - 若模块下还有子模块，建同名目录存放子模块文件（如 `network` 模块的子模块 `client` 对应 `src/network/client.rs`，并在 `src/network.rs` 中声明 `mod client;`）。

- **目录结构示例**

  ```
  src/
  ├── main.rs
  ├── network.rs          // network 模块本体
  └── network/
      └── client.rs       // network 的子模块 client
  ```

- **`src/main.rs`**

  ```rust
  mod network;  // 声明 network 模块，编译器据此去找 src/network.rs

  fn main() {
      network::connect();
      network::client::ping();
  }
  ```

- **`src/network.rs`**

  ```rust
  pub mod client;  // 声明子模块 client，据此去找同名目录下的 src/network/client.rs

  pub fn connect() {
      println!("network 连接中...");
  }
  ```

- **`src/network/client.rs`**

  ```rust
  pub fn ping() {
      println!("client ping");
  }
  ```

- **要点**
  - `network` 目录名与 `network.rs` 同名，二者共同构成 `network` 模块：`network.rs` 是模块本体内容，`network/` 目录存放其子模块文件。
  - 子模块要被外部（`main.rs`）访问到，链路上每一级都需显式 `pub`：`network.rs` 里 `pub mod client;` 暴露子模块，`client.rs` 里 `pub fn ping` 暴露函数。

### Crate 类型

- **核心职责：区分编译产物的最终形态**
  
  Crate 是 Rust 编译的最小单元，一个 Cargo 项目至少包含一个 crate。

- **Binary crate（二进制包）**
  
  含 `main` 函数，编译为可执行程序，入口文件为 `src/main.rs`。

- **Library crate（库包）**
  
  无 `main` 函数，编译为供其他项目引用的功能代码，入口文件为 `src/lib.rs`。

- **常见组合**
  
  同一项目可同时拥有 `main.rs` 与 `lib.rs`：核心逻辑写入 `lib.rs`（便于测试与复用），`main.rs` 仅负责调用与启动，二者职责解耦。

## Cargo 依赖管理

- **`Cargo.toml` 特性标志（features）**

  按需勾选第三方库的可选功能模块，避免全量编译导致体积与编译时间膨胀。

  ```toml
  [dependencies]
  tokio = { version = "1", features = ["rt", "macros"] }
  ```

- **workspace 多包管理**
  - 在顶层 `Cargo.toml` 统一声明多个子 crate，解决大项目拆分为多个关联子项目后的依赖版本冲突与重复编译问题。
  - 各子 crate 共享同一份依赖版本解析结果与编译输出目录。

  ```toml
  [workspace]
  members = ["api", "db", "common"]
  ```

## 文档测试

- **核心职责：保证文档示例与实际代码行为同步**
  - `///` 文档注释中的代码块会被 `cargo test` 当作独立测试用例编译执行，断言失败即测试失败。
  - 从机制上杜绝"文档示例过期、与真实逻辑脱节"的问题。

  ```rust
  /// 计算两数之和
  ///
  /// # Examples
  ///
  /// ```
  /// let result = my_crate::add(2, 3);
  /// assert_eq!(result, 5);
  /// ```
  pub fn add(a: i32, b: i32) -> i32 {
      a + b
  }
  ```

# 函数式编程特性

## 闭包（Closure）

- **核心职责：捕获环境变量的匿名函数，将行为本身作为参数传递**
  
  与普通 `fn` 的本质区别：`fn` 无法访问定义作用域之外的变量；闭包可捕获并携带上下文，编译器据此自动推导捕获方式。

### 基础场景

- **基础语法**

  ```rust
  let threshold = 10;
  let is_big = |x: i32| x > threshold;  // 闭包捕获外部变量 threshold，普通 fn 无法做到
  println!("{}", is_big(15));  // true
  ```

- **场景：将判断逻辑作为参数传入，而非只传数据**
  
  函数内部逻辑固定时，调用方只能决定"用什么数据"；接收闭包后，调用方可以决定"用什么逻辑"。

  ```rust
  // 对 nums 数组根据 cond 中定义的规则进行过滤，返回过滤后的数组
  fn filter_by<F: Fn(i32) -> bool>(nums: &[i32], cond: F) -> Vec<i32> {
      nums.iter().filter(|&&x| cond(x)).copied().collect()
  }

  let nums = vec![1, -3, 8, -5, 12];
  let positives = filter_by(&nums, |x| x > 0);        // 传入“大于 0”规则
  let evens = filter_by(&nums, |x| x % 2 == 0);        // 传入“偶数”规则，函数本身未改动
  ```

### 捕获环境的方式

- `Fn`、`FnMut`、`FnOnce`
  - 三者对应闭包对捕获变量的使用方式，与借用规则（`&T`/`&mut T`/所有权转移）一一对应；编译器据闭包函数体自动推导归属哪一种。
  - 三者能力递减：满足 `Fn` 的闭包自动满足 `FnMut`、`FnOnce`；反之不成立。

  | Trait | 对捕获变量的操作 | 可调用次数 |
  | --- | --- | --- |
  | `Fn` | 只读（`&T`） | 多次 |
  | `FnMut` | 修改（`&mut T`） | 多次 |
  | `FnOnce` | 取得所有权 | 仅一次 |

  - **`Fn` 场景：多次调用的只读判断规则**（即上方 `filter_by` 示例）。

  - **`FnMut` 场景：闭包内部需要修改被捕获的外部变量**

    ```rust
    fn retry<F: FnMut() -> Result<(), String>>(times: u32, mut task: F) {
        for i in 0..times {
            match task() {
                Ok(_) => { println!("第 {i} 次成功"); return; }
                Err(e) => println!("第 {i} 次失败：{e}"),
            }
        }
    }

    let mut attempts = 0;
    retry(3, || {
        attempts += 1;  // 修改外部变量 attempts，要求 FnMut
        println!("第 {attempts} 次尝试");
        Err("超时".to_string())
    });
    ```

  - **`FnOnce` 场景：闭包取得外部变量所有权，只执行一次**

    ```rust
    use std::thread;

    let connection = String::from("db-connection-handle");

    let handle = thread::spawn(move || {
        // connection 的所有权被 move 进闭包，仅在新线程执行一次
        println!("在新线程里使用: {connection}");
    });

    handle.join().unwrap();
    ```

### 闭包工厂
  
  闭包可将调用时的参数永久捕获，生成一批行为相关但参数不同的函数，`fn` 无法实现该能力。

  ```rust
  fn make_multiplier(factor: i32) -> impl Fn(i32) -> i32 {
      move |x| x * factor  // factor 被闭包永久捕获
  }

  let double = make_multiplier(2);
  let triple = make_multiplier(3);
  println!("{} {}", double(5), triple(5));  // 10 15
  ```

## 迭代器（Iterator）

- **核心职责：将遍历与数据变换抽象为可链式组合的操作**
  
  替代手动下标遍历，通过链式调用逐步变换数据，各步骤职责单一、可自由组合。

### 常用方法及使用

- **`map`：对每个元素做同一种变换**

  ```rust
  let prices = vec![10.0, 20.0, 30.0];
  let with_tax: Vec<f64> = prices.iter().map(|p| p * 1.1).collect();  // 每个元素乘以 1.1
  ```

- **`filter`：只保留满足条件的元素**

  ```rust
  let nums = vec![1, 2, 3, 4, 5, 6];
  let evens: Vec<&i32> = nums.iter().filter(|x| **x % 2 == 0).collect();  // [2, 4, 6]
  ```

- **`fold`：将序列折叠为单一值**

  ```rust
  let nums = vec![1, 2, 3, 4];
  let sum = nums.iter().fold(0, |acc, x| acc + x);  // 0 为初始值，acc 为累计结果，逐步累加为 10
  ```

- **`chain`：首尾拼接两个迭代器**

  ```rust
  let a = vec![1, 2];
  let b = vec![3, 4];
  let combined: Vec<i32> = a.into_iter().chain(b.into_iter()).collect();  // [1, 2, 3, 4]
  ```

- **`zip`：按位置配对两个迭代器，长度取较短者**

  ```rust
  let names = vec!["a", "b"];
  let scores = vec![90, 85, 100];
  let paired: Vec<(&str, i32)> = names.into_iter().zip(scores.into_iter()).collect();
  // [("a", 90), ("b", 85)]，scores 中的 100 因长度不匹配被丢弃
  ```

- **组合链式调用：筛选 → 变换 → 汇总**

  ```rust
  struct Order { qty: u32, price: f64 }

  let orders = vec![
      Order { qty: 5, price: 3.0 },
      Order { qty: 1, price: 8.0 },
      Order { qty: 10, price: 2.0 },
  ];

  let total: f64 = orders.iter()
      .filter(|o| o.qty > 2)                  // 只保留数量大于 2 的订单
      .map(|o| o.qty as f64 * o.price)        // 计算每笔订单金额
      .fold(0.0, |acc, amount| acc + amount); // 累加总金额
  ```

### 自定义迭代器

- 仅需实现唯一必要方法 `next`，即可自动获得 `map`、`filter`、`fold` 等全部默认方法（trait 默认方法机制）。
- `next` 语义：仍有下一个值返回 `Some(值)`，迭代结束返回 `None`；`for` 循环及各链式方法底层均基于反复调用 `next` 实现。

```rust
struct Counter {
    count: u32,
}

impl Iterator for Counter {
    type Item = u32;  // 关联类型：声明该迭代器产出的元素类型

    fn next(&mut self) -> Option<u32> {
        if self.count < 5 {
            self.count += 1;
            Some(self.count)
        } else {
            None
        }
    }
}

let sum: u32 = Counter { count: 0 }.map(|x| x * 2).sum();  // 自动获得 map，无需额外实现
```

## 惰性求值与零成本抽象

- **惰性求值（Lazy Evaluation）**
  - 迭代器链在构建阶段仅描述变换规则，不产生实际计算；调用 `collect`、`sum`、`for` 等消费者方法时才逐个触发求值。
  - 配合 `take(n)` 等方法可提前终止链条，避免遍历完整集合。

  ```rust
  let v = vec![1, 2, 3, 4, 5];
  let iter = v.iter().map(|x| {
      println!("处理 {x}");  // 此处尚未执行
      x * 2
  });
  // 到此为止，println! 未被调用过

  let result: Vec<i32> = iter.collect();  // 此处才逐个触发求值
  ```

- **零成本抽象（Zero-cost Abstraction）**
  - 迭代器链方法本身是泛型代码，编译期经单态化与内联展开，生成的机器码效率与手写下标循环等价，不引入运行时开销。
  - 与 `dyn Trait` 的运行时动态分发相对，属于编译期完全展开的静态路径。
