## 时间戳

### awk内置函数

在 **awk** 里，`strftime()` 是一个内置函数，用来把当前系统时间格式化成想要的字符串。

**常用形式（输出当前时间）**

```bash
strftime("%F %T")
```

**`awk`示例**

```bash
dantalion@Dantalion:~$ ping -c 10 8.8.8.8 | awk -F, '/packet loss/ {
						pl=$3+0;
						ts=strftime("%F %T");
						if (pl>0){
							printf("[%s]公网8.8.8.8的丢包率高：%s%%\n", ts, pl);
						}else{
							print("与公网8.8.8.8连接正常")
						}
}'
[2025-09-12 22:51:25]公网8.8.8.8的丢包率高：20%
```

**常用格式说明**

| 占位符 | 含义              | 示例       |
| ------ | ----------------- | ---------- |
| `%Y`   | 年（4 位）        | 2025       |
| `%m`   | 月（01-12）       | 09         |
| `%d`   | 日（01-31）       | 12         |
| `%H`   | 小时（00-23）     | 17         |
| `%M`   | 分钟（00-59）     | 45         |
| `%S`   | 秒（00-59）       | 08         |
| `%F`   | 等价于 `%Y-%m-%d` | 2025-09-12 |
| `%T`   | 等价于 `%H:%M:%S` | 17:45:08   |
| `%a`   | 星期（缩写）      | Fri        |
| `%A`   | 星期（全称）      | Friday     |

### echo - date

如果用 **`echo`**（也就是在 shell 里输出），就得用 **`date` 命令** 来实现；形式为`$(date +'格式')`。

- 标准格式：`$(date '+%Y-%m-%d %H:%M%S')`

  ```bash
  dantalion@Dantalion:~$ echo "当前时间是$(date '+%Y-%m-%d %H:%M:%S')"
  当前时间是2025-09-12 22:54:38
  ```

- 只要日期`$(date '+%F')`

  ```bash
  dantalion@Dantalion:~$ echo "今天的日期是$(date '+%F')"
  今天的日期是2025-09-12
  ```

- 只要时间`$(date '+%T')`

  ```bash
  dantalion@Dantalion:~$ echo "现在的时间是$(date '+%T')"
  现在的时间是22:57:16
  ```

## test

用于测试条件，返回布尔值0或1。常用于判断文件属性以及字符串比较。

### 参数

```bash

-ne		不等于

-ge		大于等于
-lt		小于
-le		小于等于

-n		长度不为0(字符串非空)

#文件测试参数
#格式为test [参数] [文件名]
```

## if-else逻辑判断

### 语法

```bash
if [ 判断条件 ]; then			#注意判断条件语句和中括号之间要用空格
	执行语句1;
else
	执行语句2;
fi							  #用fi表示结尾
```

### 脚本书写

```bash
#! /bin/bash

#判断数字大小
read -p "请输入一个数字：" num
if [ $num -gt 10 ]; then
        echo "$num大于10"
else
        echo "$num小于10"
fi

#判断文件是否存在
read -p "请输入文件名：" filename
if [ -e $filename ]; then
        echo "文件存在"
else
        echo "文件不存在"
fi

#判断字符串是否相等
read -p "请输入字符串1：" str1
read -p "请输入字符串2：" str2
if [ "$str1" = "$str2" ]; then
        echo "两个字符串相同"
else
        echo "字符串不相同"
fi
```

### 运行测试

```bash
dantalion@Dantalion:~/shell$ bash test.sh

3小于10
请输入文件名：test.py
文件存在
请输入字符串1：aster
请输入字符串2：ass
字符串不相同
```

## for循环/遍历循环

### 语法

```bash
for [元素] in [列表/范围]
do
	执行语句
done
```

### 脚本书写

```bash
#! /bin/bash

read -p "请输入一个数字：" num
for i in $(seq $num)
do
        echo "当前数字为：$i"
done

index=1
read -p "请输入一个字符串：" str
for (( i=0; i<${#str}; i++ ))
do
        echo "这个字符串的第$index个元素为：${str:i:1}";
        ((index++))
done
```

> [!CAUTION]
>
> 使用`seq`时要用`$`将序列值取出；
>
> `for`循环遍历字符串中的单个元素只能用索引切片操作。

# 函数

1. 使用 `function_name()` 来定义函数。
2. 使用 `$1`, `$2`, ... 来访问传入的参数。
3. 使用 `return` 来设置退出状态。
4. 使用 `local` 来创建局部变量，防止污染全局命名空间。

## 定义

```bash
function_name() {
	执行语句
}
```

## 调用

```bash
function_name [参数1] [参数2] ...[参数n]
```

## 脚本书写和测试

```bash
#! /bin/bash
#函数中的参数和局部变量

greet() {
        local name=$1
        echo "hello, $name"
}

read -p "请输入你的名字" name
greet $name
```

```bash
dantalion@Dantalion:~/shell$ bash func.sh
请输入你的名字aster
hello, aster
```

```bash
#! /bin/bash
#函数的返回值

get_sum() {
        local num1=$1 num2=$2
        sum=$((num1 + num2))
        echo "这两个数字的和是$sum"
}

read -p "请输入两个数字：" num1 num2
get_sum $num1 $num2
```

```bash
dantalion@Dantalion:~/shell$ bash func.sh
请输入两个数字：3 5
这两个数字的和是8
```

> [!IMPORTANT]
>
> 使用关键字`local`设置局部变量；
>
> `$n`表示传入的第n个参数；
>
> `function_name() {}`中`{}`与函数体之间必须有空格隔开（如果使用单行表示函数）。

# 管道

将一个命令的标准输出（stdout）传递给另一个命令的标准输入（stdin）。管道可以帮助将多个命令链接起来，进行数据流的处理和传递。

## 语法

```bash
command1 | command2 | command3 ... | commandn
```

## 示例

> 此处pstree结合使用了grep，是一个筛选命令，将在后续文本处理环节中进行详细介绍。

```bash
#将pstree显示的内容传递给了grep
dantalion@Dantalion:~/shell$ pstree | grep "bash"
        |-init-systemd(Ub-+-SessionLeader---Relay(752)---bash-+-grep
        |                 |-login---bash
```

## *xargs

`xargs` 从标准输入读取数据，再通过管道，将这些数据转换为命令的参数。

```bash
#找出当前目录及其所有子目录下的日志文件并删除
find . -name '*.log' -type f | xargs rm
```

### 常见选项

- `-I {}` ：定义一个占位符，通过管道传递并被xargs转换的参数会被插入到占位符中

  ```bash
  #备份所有txt文件
  find . -name '*.txt' -type f | xargs -I {} cp {} {}.bak
  #等同于执行命令：cp ./hello.txt ./hello.txt.bak
  
  #批量修改文件路径
  find . -type f | xargs -I {} mv {} /new/path
  #等同于执行命令：mv [所有文件] /new/path
  ```

- `-n`：指定每次传递给命令的参数数量

  ```bash
  echo "a b c d e" | xargs -n 3 echo
  #输出：
  a b c
  d e
  
  find . -name '*.txt' -type f | xargs -I {} -n 10 mv {} /new/path
  #相当于每次只移动10个文件，执行多次，直到移动完毕
  ```

- `-p`：交互式地操作，执行每个命令前询问用户是否确定要执行

# 重定向

## 输出重定向

- `>`：覆盖输出

  ```bash
  echo "Hello, World!" > output.txt
  #不管该txt文件是否为空，最终都只有Hello, World!
  ```

- `>>`：追加输出

  ```bash
  echo "Hello, World!" > output.txt
  #在文件末尾追加内容Hello, World!
  ```

- `2>>`：错误输出重定向追加

  ```bash
  dantalion@Dantalion:~/shell$ ls non_existent_file 2>> error.log
  dantalion@Dantalion:~/shell$ ls
  error.log  func.sh  special_var.sh  test.py  test.sh
  dantalion@Dantalion:~/shell$ cat error.log
  ls: cannot access 'non_existent_file': No such file or directory	#日志存储了错误信息
  ```

- `&>>` ：将标准输出和错误输出一起追加到同一个文件

  ```bash
  dantalion@Dantalion:~/shell$ rm test.txt &>> output.log
  dantalion@Dantalion:~/shell$ cat output.log
  rm: cannot remove 'test.txt': No such file or directory
  ```

- `command 2>&1`：将错误输出重定向到文件描述符`1`所指向的地方；将`&`与C语言的指针联系起来，为取地址的操作

  - **文件描述符**

    - **0 - stdin (标准输入)**：默认是键盘输入。程序从此读取数据。
    - **1 - stdout (标准输出)**：默认是终端屏幕。程序将常规输出写入此处。
    - **2 - stderr (标准错误)**：默认也是终端屏幕。程序将错误信息和警告写入此处。

  - 先让 `command` 执行，然后将其标准错误（stderr）的输出，合并到标准输出（stdout）的同一目的地。

  - 使用`echo $$`查看当前进程pid，然后再执行`ls /proc/pid/fd/`即可查看当前所有文件描述符所指向的地址

    ```bash
    dantalion@Dantalion:~/shell$ echo $$
    386
    dantalion@Dantalion:~/shell$ ls -l /proc/386/fd
    total 0
    lrwx------ 1 dantalion dantalion 64 Aug 21 21:05 0 -> /dev/pts/2
    lrwx------ 1 dantalion dantalion 64 Aug 21 21:05 1 -> /dev/pts/2
    lrwx------ 1 dantalion dantalion 64 Aug 21 21:05 2 -> /dev/pts/2
    lrwx------ 1 dantalion dantalion 64 Aug 21 21:05 255 -> /dev/pts/2
    lrwx------ 1 dantalion dantalion 64 Aug 21 21:05 7 -> /dev/ptmx
    ```

    由此可见文件描述符0、1、2指向的地址都是`/dev/pts/`，也即当前终端窗口；2表示当前终端窗口的序号，通常从0开始。

## 输入重定向

- `<`：将文件内容作为命令的输入，而不是通过键盘输入

  ```bash
  grep "hello" < name.txt
  #将文件内容传递给grep，grep再此内容上进行筛选并输出
  ```

# 正则表达式

用于模式匹配，广泛应用于文件搜索、文本处理等任务。

## BRE - 基本正则表达式

- **`.`**：匹配任何单个字符（除了换行符）。

  `a.c` 匹配 "abc", "aac", "a c", "a-c" 等。

- **`*`**：匹配零个或多个前面的字符或表达式。

  `ab*c` 匹配 "ac", "abc", "abbc", "abbbc" 等。

- **`[]`**：匹配方括号中的任何一个字符。

  `[aeiou]` 匹配任意一个元音字母。

  `[0-9]` 匹配任意一个数字（等价于 `[0123456789]`）。

  `[a-zA-Z]` 匹配任意一个字母。

- **`^`**：匹配输入的开始。

  `^abc` 匹配以 "abc" 开头的行。

- **`$`**：匹配输入的结束。

  `xyz$` 匹配以 "xyz" 结尾的行。

- **`\`**：转义字符，用来匹配特殊字符本身（如`.`、`*`、`[ ]`等）。

## ERE - 扩展正则表达式

> [!IMPORTANT]
>
> 在`grep`和`sed`命令中使用ERE必须加上参数`-E`，否则需要额外加上转义字符；`awk`则默认使用ERE。

- **`+`**：匹配一个或多个前面的字符或表达式。

  `ab+c` 匹配 "abc", "abbc"，但不匹配 "ac"。

- **`?`**：匹配零个或一个前面的字符或表达式。

  `colou?r` 匹配 "color" 和 "colour"。

- `|`：表示逻辑或。

  `cat|dog|bird` 匹配 "cat", "dog" 或 "bird"。

- **`{n,m}`**：匹配前面的字符或表达式，至少n次，最多m次。也可写`{n}`，`{n,}`分表代表恰好n次和n次及以上。

  `o{2,4}` 匹配 "oo", "ooo", "oooo"。

  `[0-9]{1,3}`匹配由0-9组成的一到三位数，如`[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}`常用于匹配ip地址。

- **`()`**：将几个元素组合为一个整体，以便对其使用量词或捕获。

  `(ab)+` 匹配 "ab", "abab", "ababab"。

## 预定义字符 - 快捷方式

> [!CAUTION]
>
> 必须包含在`[]`内。
>
> - **正确**：`[[:digit:]]+` 匹配一个或多个数字。
> - **错误**：`[:digit:]+` 这不会按预期工作。

- `[:alnum:]`：字母和数字，即 `[0-9a-zA-Z]`。
- `[:alpha:]`：字母，即 `[a-zA-Z]`。
- `[:digit:]`：数字，即 `[0-9]`。
- `[:lower:]`：小写字母，即 `[a-z]`。
- `[:upper:]`：大写字母，即 `[A-Z]`。
- `[:space:]`：空白字符，包括空格、制表符、换行符等。

## 示例

> 书写一个用来判断用户输入的邮箱地址格式是否正确的脚本。

> [!IMPORTANT]
>
> 条件判断使用`[[ ... =~ ... ]]`，其中`=~` 运算符使用扩展正则表达式（ERE）进行匹配。

### 脚本书写

```bash
#! /bin/bash

read -p "请输入你的邮箱地址：" addr

format="^[[:alnum:]+-.%_]+@[[:alnum:].-]+\.[[:alpha:]]{2,}$"
    #^表示邮箱开头
    
    #用户名格式：
    #[[:alnum:]+-.%_]+最外层结构为[]+，表示可以与[]中的任意字符匹配多次；[]包含[:alnum:]即所有字母数字，以及+—.%_这些符号，说明用户名允许这些符号出现。
    
    #@域名名称：
    #[[:alnum].-]+最外层结构同样为[]+，表示可以与[]中的任意字符匹配多次；[]包含[:alnum:]即所有字母数字，以及.-这些符号，说明域名名称允许这些符号出现。
    
   	#\.顶级域名：
   	#使用转义字符\来表示此处的.并不匹配任何字符
    #[[:alpha:]]{2,}由[]和{}组成，[:alpha:]表示所有字母，{2,}表示其前面的字符可以出现2次或以上；说明顶级域名允许2个或以上任意字母。
    
    #$表示邮箱结尾

if [[ $addr =~ $format ]]; then
        echo "邮箱格式正确"
else
        echo "邮箱格式错误"
fi
```

## sed

一个强大的文本处理工具，常用于对文本进行替换、删除、插入等操作。

### 语法

```bash
sed [options] 'command' [file...]
```

- 参数
  - `-i`：直接修改文件内容并显示到终端（默认情况下不会真正修改文件）
  - `-e`：执行多条`sed`命令
  - `-f`：从脚本文件读取命令

### 定位

- **行号定位**

  - `n`：第 n 行

  ```bash
  sed '5s/old/new/' file.txt  # 只替换第5行的 old
  ```

  - `n,m`：第 n 到 m 行

  ```bash
  sed '1,3s/old/new/' file.txt  # 替换1-3行的 old
  ```

  - `n,+m`：第 n 行到第 n+m 行

  ```bash
  sed '2,+3s/old/new/' file.txt  # 替换2-5行的 old
  ```

  - `$`：最后一行

  ```bash
  sed '$s/old/new/' file.txt  # 只替换最后一行的 old
  ```

- **正则表达式定位**

  - `/pattern/`：匹配模式的行

  ```bash
  sed '/error/s/old/new/' file.txt  # 只在包含 error 的行中替换
  ```

  - `/pattern1/,/pattern2/`：从匹配 pattern1 到匹配 pattern2 的行范围

  ```bash
  sed '/start/,/end/d' file.txt  # 删除从 start 到 end 之间的所有行
  ```

- **步进定位**

  - `n~m`：从第 n 行开始，每隔 m 行操作一次

  ```bash
  sed '1~2s/old/new/' file.txt  # 替换所有奇数行的 old
  ```

### 核心命令

- `s` - 替换

  ```bash
  sed 's/old/new/' file.txt           # 每行第一个 old 替换为 new
  sed 's/old/new/g' file.txt          # 替换所有的 old
  sed 's/old/new/2' file.txt          # 只替换每行第二个 old
  sed -n 's/old/new/p' file.txt       # 只打印发生替换的行
  sed 's/old/new/w changed.txt' file.txt  # 将修改的行写入文件
  ```

- `d` - 删除

  ```bash
  sed '5d' file.txt           # 删除第5行
  sed '1,3d' file.txt         # 删除1-3行
  sed '/pattern/d' file.txt   # 删除匹配模式的行
  sed '/start/,/end/d' file.txt  # 删除从 start 到 end 的范围
  ```

- `p` - 打印

  ```bash
  sed -n '1,5p' file.txt      # 只打印1-5行（需配合 -n）
  sed -n '/error/p' file.txt  # 只打印包含 error 的行
  ```

- `r` - 读取文件并插入

  ```bash
  sed '/pattern/r otherfile.txt' file.txt  # 在匹配行后插入文件内容
  ```

- `w` - 写入文件

  ```bash
  sed '/pattern/w output.txt' file.txt  # 将匹配行写入文件
  ```

### 示例

```bash
#替换文件内容，将函数名改为了sum
dantalion@Dantalion:~/shell$  sed 's/get_sum/sum/' func.sh
#! /bin/bash
#函数的返回值

sum() {
        local num1=$1 num2=$2
        sum=$((num1 + num2))
        echo "这两个数字的和是$sum"
}

read -p "请输入两个数字：" num1 num2
sum $num1 $num2
#将函数名替换为sum后输出到终端
```

> [!CAUTION]
>
> 此处使用替换操作时记得加`''`，以及在新字符串末尾也要添加`/`。

```bash
#删除文件中的行
dantalion@Dantalion:~/shell$ sed '/echo/d' test.sh
#! /bin/bash

read -p "请输入选择:" option

case $option in
        a|A) df -h;;
        b|B) free -h;;
        c|C) pstree;;

#删除test.sh文件中包含echo的行后输出到终端
```

## awk

非常强大的文本处理工具，特别擅长按列操作。

### 语法

```bash
awk 'pattern {action}' file
```

- **pattern：**用于匹配文本的条件，匹配成功则执行后面的action。
- **action：**匹配成功后执行的操作，用`{}`括起来的代码块，常常是打印、修改或调整某些字段的值。
- **file：**要进行操作的文件，可选，为空的话则默认从标准输入读取。

> [!IMPORTANT]
>
> 当只有pattern而action为空时，默认输出为`{print $0}`，即打印所有行。

### 参数

```bash
-F[符号]					#设置分隔符
-v var="$var_name"		#传入系统外部变量，引号中的var_name写变量名，var填入在awk命令中使用的变量名
```

### 模式

- `BEGIN`：在读取任何输入行之前执行一次。

  ```bash
  awk 'BEGIN {print "开始处理文件..."} {print $0}' file.txt
  ```

- `END`：处理完所有输入行之后最后执行一次。

  ```bash
  awk '{sum += $1} END {print "总和是：", sum}' file.txt
  ```

- `/正则表达式/`：处理能够匹配正则表达式中文本的行。

  ```bash
  # 打印包含 "error" 或 "Error" 的行
  awk '/[Ee]rror/ {print $0}' logfile
  ```

  匹配运算符：`~`与`!~`，用于将字段值与正则表达式进行匹配判断。

  ```bash
  # 1. 打印第一个字段包含数字的行
  awk '$1 ~ /[0-9]/' data.txt
  
  # 2. 打印第二个字段不以 "test" 开头的行
  awk '$2 !~ /^test/' data.txt
  ```

- `关系表达式`：使用比较运算符（`==`, `!=`, `<`, `>`, `<=`, `>=`）来筛选。

  ```bash
  # 打印第一列数字大于 100 的行
  awk '$1 > 100 {print $0}' data.txt
  ```

  `print $0`表示打印整行。

- `模式1, 模式2`：范围模式，处理从匹配`模式1`的行开始，到匹配`模式2`的行结束的所有行。

  ```bash
  # 打印从包含 "START" 的行到包含 "END" 的行之间的所有内容
  awk '/START/, /END/ {print}' file.txt
  ```

- 数组：常用于统计。

  > 假设有一个`web.log`，内容如下：
  >
  > ```
  > 192.168.1.1 - - [01/Jan/2023:10:00:01] "GET /index.html"
  > 192.168.1.2 - - [01/Jan/2023:10:00:02] "GET /about.html"
  > 192.168.1.1 - - [01/Jan/2023:10:00:03] "POST /login"
  > 192.168.1.3 - - [01/Jan/2023:10:00:04] "GET /index.html"
  > ```

  ```bash
  # 统计每个 IP 的访问次数
  awk '{ip_count[$1]++} END {for (ip in ip_count) print ip, "访问了", ip_count[ip], "次"}' web.log
  # 输出（示例）：
  # 192.168.1.1 访问了 2 次
  # 192.168.1.2 访问了 1 次
  # 192.168.1.3 访问了 1 次
  ```

  > [!NOTE]
  >
  > 此处使用`ip_count[]`关联`$1`，即第一个字段数据创建了一个数组，执行完后数组数据如下：
  >
  > ```bash
  > ip_count["192.168.1.1"] = 2
  > ip_count["192.168.1.2"] = 1
  > ip_count["192.168.1.3"] = 1
  > ```

### 动作

- `print`：打印。
- `printf()`：格式化打印（类似 C 语言）。
- 赋值语句（如 `sum=$1+$2`）。
- 流程控制语句（`if`, `for`, `while`, `do-while`）。
- 函数调用。

### 内建变量

| 变量                            | 描述                                | 示例                                       |
| :------------------------------ | :---------------------------------- | :----------------------------------------- |
| `$0`                            | 当前行的全部内容                    | `awk '{print $0}'`                         |
| `$1, $2, ... $NF`               | 当前行的第 1, 2, ... 最后一个字段   | `awk '{print $1, $3}'`                     |
| `NF` (Number of Fields)         | 当前行的字段总数                    | `awk '{print $NF}'` (打印最后一个字段)     |
| `NR` (Number of Records)        | 当前处理的是第几行（所有文件累计）  | `awk '{print NR ":" $0}'`                  |
| `FNR` (File Number of Records)  | 当前处理的是当前文件的第几行        | 处理多个文件时，每个文件都会从 1 开始计数  |
| `FS` (Field Separator)          | 输入字段分隔符（默认是空格/制表符） | `awk -F: '{print $1}'` 或 `BEGIN {FS=":"}` |
| `OFS` (Output Field Separator)  | 输出字段分隔符（默认是空格）        | `awk 'BEGIN {OFS="---"} {print $1, $2}'`   |
| `RS` (Record Separator)         | 输入行分隔符（默认是换行符`\n`）    |                                            |
| `ORS` (Output Record Separator) | 输出行分隔符（默认是换行符`\n`）    |                                            |
| `FILENAME`                      | 当前正在处理的文件名                | `awk '{print FILENAME, $0}' file.txt`      |

### 示例

> 文件内容：
>
> ```
> Alice 25 50000
> Bob 30 45000
> Charlie 22 60000
> David 35 55000
> Eve 28 48000
> ```

```bash
#打印文件全部内容
dantalion@Dantalion:~/shell$ awk '{print}' data.txt
Alice 25 50000
Bob 30 45000
Charlie 22 60000
David 35 55000
Eve 28 48000

#打印第一列和第二列
dantalion@Dantalion:~/shell$ awk '{print $1,$2}' data.txt
Alice 25
Bob 30
Charlie 22
David 35
Eve 28

#条件过滤，只打印第二列数值大于30的行
dantalion@Dantalion:~/shell$ awk '$2 > 30 {print $0}' data.txt
David 35 55000

#正则匹配，只打印含有Bob的行
dantalion@Dantalion:~/shell$ awk '/Bob/ {print}' data.txt
Bob 30 45000

#使用BEGIN-END块
dantalion@Dantalion:~/shell$ awk 'BEGIN{sum=0; count=0} {sum+=$3; count++} END{printf("平均月薪为:%.2f\n", sum/count)}' data.txt
平均月薪为:51600.00
```

## sort

用于对文本文件的行进行排序。可以将输入内容按字典顺序、数值顺序、月份顺序等多种方式进行排序，并支持去重、合并等高级功能。

### 语法

```bash
sort [options] [文件...]
```

### 排序选项

- **基础排序选项**

  - `-r`：反向排序
  - `-f`：忽略大小写

- **字段排序选项**

  - `-k`：指定排序的字段范围

    ```bash
    sort -k 2 file.txt      # 按第2字段排序
    sort -k 2,3 file.txt    # 按第2字段到第3字段排序
    sort -k 2.3,2.5 file.txt # 按第2字段的第3-5个字符排序
    ```

    可以在字段序号后添加修饰符：

    - `n`：数值排序
    - `r`：反向排序
    - `f`：忽略大小写
    - `d`：字典顺序

- **数值排序选项**

  - `-n`：按数值大小排序

    ```bash
    sort -n numbers.txt  # 数值排序（10 在 2 后面）
    ```

  - `-h`：按照人类可读的数值排序（带单位）

    ```bash
    sort -h sizes.txt  # 正确排序：1K, 2M, 1G
    ```

  - `-g`：通用数值排序（支持科学计数法）

    ```bash
    sort -g scientific.txt  # 排序：1e-5, 0.001, 1e+3
    ```

- **特殊排序选项**

  - `-M`：按月份名称排序
  - `-V`：按版本号排序
  - `-R`：随机打乱顺序

- **其他选项**

  - `-u`：去重，只输出唯一的行

  - `-c`：检查文件是否已排序

    ```bash
    sort -c file.txt && echo "已排序" || echo "未排序"
    ```

  - `-m`：合并已排序的文件

  - `-o`：指定输出文件（可覆盖原文件）

    ```bash
    sort file.txt -o sorted_file.txt
    ```

### 示例

> 假设有文件`data.txt`，内容如下：
>
> ```
> Alice 25 50000
> Bob 30 45000
> Charlie 22 60000
> David 35 55000
> Eve 28 48000
> ```

```bash
#基础排序默认按照第一字段字典排序

#按字段排序，第二字段的值且降序排列
dantalion@Dantalion:~/shell$ sort -k 2nr data.txt
David 35 55000
Bob 30 45000
Eve 28 48000
Alice 25 50000
Charlie 22 60000

#多字段排序，先根据薪资降序，再根据年龄升序
dantalion@Dantalion:~/shell$ sort -k 3nr -k 2n data.txt
Charlie 22 60000
David 35 55000
Alice 25 50000
Eve 28 48000
Bob 30 45000
```
