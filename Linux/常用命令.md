
## 周期性执行

### watch

能够周期性执行命令并刷新显示，特别适合实时观察命令的输出。

**基本用法：**

```bash
watch -n 2 'command'

#-n 2表示每2秒执行一次命令（默认也为2秒）
#'command'则填入要运行的命令
```

**示例：**

```bash
watch -n 1 df -h								# 实时查看磁盘空间
watch -n 1 free -m								# 实时查看内存
watch -n 1 ps aux --sort=-%cpu | head -10		  # 实时查看CPU占用最高的进程
```

## CPU和内存

### top

类似于Windows的任务管理器，可以实时显示系统的整体运行状态以及各个进程的资源占用情况。

```bash
dantalion@Dantalion:~/shell$ top
top - 11:59:53 up 13 min,  1 user,  load average: 0.00, 0.04, 0.08
Tasks:  23 total,   1 running,  22 sleeping,   0 stopped,   0 zombie
%Cpu(s):  0.0 us,  0.0 sy,  0.0 ni,100.0 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
MiB Mem :   7799.2 total,   7327.1 free,    502.1 used,    106.2 buff/cache
MiB Swap:   2048.0 total,   2048.0 free,      0.0 used.   7297.0 avail Mem
----------------------------------------------------------------------------------------
    PID USER      PR  NI    VIRT    RES    SHR S  %CPU  %MEM     TIME+ COMMAND
      1 root      20   0   21716  12160   9280 S   0.0   0.2   0:00.30 systemd
      2 root      20   0    3060   1440   1440 S   0.0   0.0   0:00.00 init-systemd(Ub
      7 root      20   0    3060   1760   1760 S   0.0   0.0   0:00.00 init
```

分为两个部分：系统总体信息（前5行）和具体进程信息（表格部分）。

**系统信息：**

```bash
top - 11:59:53 up 13 min,  1 user,  load average: 0.00, 0.04, 0.08
Tasks:  23 total,   1 running,  22 sleeping,   0 stopped,   0 zombie
%Cpu(s):  0.0 us,  0.0 sy,  0.0 ni,100.0 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
MiB Mem :   7799.2 total,   7327.1 free,    502.1 used,    106.2 buff/cache
MiB Swap:   2048.0 total,   2048.0 free,      0.0 used.   7297.0 avail Mem
```

1. **系统时间&运行信息：**

   - `11:59:53`为当前时间；
   - `up 13 min`为系统已经运行的时间；
   - `1 user`为当前登录的用户个数；
   - `load average: 0.00, 0.04, 0.08`为系统平均负载（最近 1、5、15 分钟的平均运行队列长度）。

2. **任务信息：**

   - `total`为总进程数；
   - `running`、`sleeping`、`stopped`、`zombie`分别代表正在运行的进程数、睡眠（等待）的进程数、停止的进程以及僵尸进程。

3. **CPU（时间）信息：**

   - `us` 为用户空间 CPU 占比 (user)；**`us`高说明应用本身CPU密集**

   - `sy` 为内核空间 CPU 占比 (system)；**`sy`高说明系统调用多，可能内核开销大（网络/磁盘繁忙）**

   - `ni` 为nice 调整过优先级的进程占比

   - `id` 为空闲 (idle)

   - `wa` 为等待 I/O 的 CPU 占比 (I/O wait)；**`wa`高说明可能有磁盘或网络I/O瓶颈**

   - `hi` 为硬件中断占比 (hardware interrupt)

   - `si` 为软件中断占比 (software interrupt)

   - `st` 为被虚拟机偷走的 CPU 时间 (steal time)；**`st`高说明虚拟机CPU被征用，需要调度更多物理资源**

   > [!NOTE]
   >
   > **CPU时间**
   >
   > CPU 在不同状态下花费的时间；Linux 内核会通过周期性时钟中断（tick）来统计 CPU 在各个状态下度过了多少时间。

4. **内存信息：**

   - `total`为总内存；
   - `free`、`used`、`buff/cache`分别表示空闲内存、已使用内存以及缓冲缓存。

5. **交换分区swap信息：**

   - `total`表示总swap；
   - `free`、`used`、`avail Mem`分别表示空闲swap、已用swap、估算应用可用内存。

   > [!NOTE]
   >
   > swap是单独划出来的一块磁盘分区，作为一种**虚拟内存**，在物理内存（RAM）不够用的时候暂存不用的数据，能够起到应急缓冲的作用。

**进程信息：**

> 一般用`ps -ef`或`ps aux`命令查看进程信息，`top`主要关注的是系统总体情况。

```bash
 	PID USER      PR  NI    VIRT    RES    SHR S  %CPU  %MEM     TIME+ COMMAND
      1 root      20   0   21716  12160   9280 S   0.0   0.2   0:00.30 systemd
      2 root      20   0    3060   1440   1440 S   0.0   0.0   0:00.00 init-systemd(Ub
      7 root      20   0    3060   1760   1760 S   0.0   0.0   0:00.00 init
```

- **PID** → 进程号

- **USER** → 进程所属用户

- **PR** → 进程优先级 (priority)

- **NI** → nice 值（影响优先级）

- **VIRT** → 进程使用的虚拟内存总量

- **RES** → 常驻内存（实际使用的物理内存）

- **SHR** → 共享内存大小

- **S** → 进程状态：

  - `R` → running（运行中）

  - `S` → sleeping（睡眠）
  - `Z` → zombie（僵尸）
  - `T` → stopped（暂停）

- **%CPU** → 占用 CPU （时间）百分比

- **%MEM** → 占用内存百分比

- **TIME+** → 占用 CPU 的累计时间

- **COMMAND** → 启动该进程的命令

**交互模式：**

进入`top`后，

- 按 **`P`** → 按 CPU 使用率排序
- 按 **`M`** → 按内存使用率排序
- 按 **`N`** → 按 PID 排序
- 按 **`T`** → 按运行时间排序

### vmstat

```bash
dantalion@Dantalion:~/shell$ vmstat
procs -----------memory---------- ---swap-- -----io---- -system-- -------cpu-------
 r  b   swpd   free   buff  cache   si   so    bi    bo   in   cs us sy id wa st gu
 0  0      0 7504752   2576 111128    0    0   161    12  114    0  0  0 100  0  0  0
```

> [!IMPORTANT]
>
> 重点关注`r`、`si/so`、`cs`、`wa`字段。

- `r`代表处于运行队列的进程数，如果大于CPU核数，说明CPU可能过载。
- `si/so`代表swap的读入读出数据量，长期非零的话说明系统内存不足，系统在不断交换数据，拖慢性能。
- `cs`代表每秒上下文切换（context switch）的次数，如果数值很高的话说明进程切换频繁，CPU在切换进程上的开销会变大。
- `wa`表示等待 I/O 的 CPU 时间百分比。

### free -m

`free`命令本身会显示内存使用信息，`-m`表示以`MB`为单位。

```bash
dantalion@Dantalion:~/shell$ free -m
               total        used        free      shared  buff/cache   available
Mem:            7799         500        7325           3         111        7298
Swap:           2048           0        2048
```

其中`Mem`、`Swap`行分别表示物理内存和虚拟内存。

> [!NOTE]
>
> ### `buff/cache`缓存：
>
> 指 **内存中保存的、可以加速访问的数据副本**，它不属于某个进程私有，而是内核为了提高性能主动维护的；并且可以在系统内存不足时主动回收/释放以腾出更多内存给活动进程。
>
> ### **为什么要缓存？**
>
> - 读写磁盘的速度比内存慢几个数量级。
> - 缓存可以把“热数据”放到内存里，大幅提升 IO 性能。
> - 举个例子：
>   - 第一次 `cat big.log` → 很慢，因为要从磁盘读。
>   - 第二次 `cat big.log` → 很快，因为文件已经在 page cache 里。

## IO

### 用法

```bash
iostat [ options ] [ <interval> [ <count> ] ]
```

如`iostat -x 1 5`表示每隔1秒采样并输出1次扩展统计信息，一共输出5次；一般可用于短时间内观察磁盘I/O负载波动。

`-x`比默认 `iostat` 多输出了磁盘的更多性能指标，比如：

- `await`：平均 I/O 等待时间 (ms)
- `svctm`：平均每次 I/O 服务时间 (ms)
- `%util`：磁盘利用率/忙碌率（接近 100% 说明磁盘已跑满）

> [!IMPORTANT]
>
> 重点关注`await`和`%util`字段，如果二者很高说明可能存在磁盘I/O瓶颈。

```bash
           0.01    0.00    0.04    0.00    0.00   99.95

sda               0.25        16.25         0.00         0.00      74137          0          0
sdb               0.03         1.64         0.00         0.00       7481          0          0
```

- **CPU部分：**

>
> **Z** = 僵尸进程

> **Ss** = 后台守护
>
> **Sl** = 多线程，轻量进程

### ip addr

```bash
dantalion@Dantalion:~$ ip addr
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00

       valid_lft forever preferred_lft forever
    inet 10.255.255.254/32 brd 10.255.255.254 scope global lo
       valid_lft forever preferred_lft forever
    inet6 ::1/128 scope host
       valid_lft forever preferred_lft forever
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc mq state UP group default qlen 1000
    link/ether 00:15:5d:52:bf:22 brd ff:ff:ff:ff:ff:ff
    inet 172.18.204.191/20 brd 172.18.207.255 scope global eth0
       valid_lft forever preferred_lft forever
    inet6 fe80::215:5dff:fe52:bf22/64 scope link
       valid_lft forever preferred_lft forever
```

> [!IMPORTANT]
>
> 重点关注接口状态、IP地址、子网&广播、MAC地址、MTU。

**接口状态：**

- `<UP,LOWER_UP>`与`state UP`

  - 网卡是否启用？
  - 物理链路是否连通？（虚拟机/容器内就相当于 veth 连接是否还在）
  - 如果 `state DOWN` 或 `NO-CARRIER` → 多半是网线/虚拟链路断了。

  > [!NOTE]
  >
  > **veth：**Virtual Ethernet， 是 Linux 容器、网络命名空间里最核心的“虚拟网线”，用来把不同网络命名空间（容器、虚拟机、bridge、路由）连接起来，是容器网络的最小积木。
  >
  > **主要用途：**
  >
  > 1. **连接容器与宿主机**
  >
  >    - 创建一个 veth pair，一端放进容器 netns，另一端放在宿主机。
  >
  >    - 宿主机这端一般挂到 **bridge（如 docker0, cni0, br0）** 上。
  >
  >    - 这样容器就能通过宿主机网桥和外部通信。
  >
  > 2. **连接两个命名空间**
  >
  >    - 可以用 veth 把两个自定义 netns 连在一起，模拟网络拓扑。

**IP地址：**

- 通过`inet`查看
  - 是否正确配置IP？
  - `scope` → 判断是仅本地、链路，还是可路由的全局地址。

**子网和广播：**

- CIDR掩码（比如`/20`）
- 广播地址（`brd`）

**MAC地址：**

- `link/ether`
  - 用来排查 ARP 冲突，比如IP地址更新，但ARP缓存没有更新，导致MAC地址对不上的情况。

**MTU：**

- `mtu 1500`
  - MTU 不一致可能导致某些流量丢包（特别是 VPN、Overlay 网络）。

## ARP和局域网

### arp -a

能够查看ARP缓存表，体现IP地址和MAC地址的映射关系。

```bash
dantalion@Dantalion:~$ arp -a
? (172.18.192.0) at <incomplete> on eth0
? (192.168.1.99) at <incomplete> on eth0
? (192.168.1.0) at <incomplete> on eth0
Dantalion.mshome.net (172.18.192.1) at 00:15:5d:a7:02:89 [ether] on eth0
```

**以第一行信息【有误】为例：**

- `?`表示没有反向解析到主机名（只显示了 IP）；

- 其后紧跟的`172.18.192.0`表示目标IP地址；

- `at <incomplete>`表示ARP 请求过，但没有拿到响应；

- 一般说明：

  - 目标不存在/不在线（此案例为不存在）

  - 目标不在同一个二层网络

  - 或者目标拒绝 ARP 响应

- `on eth0`表示这是通过网卡 `eth0` 访问时记录下来的 ARP 条目。

**再以最后一行信息【标准】为例：**

- `Dantalion.mshome.net`为目标主机名，通过反向DNS解析得到；
- `172.18.192.1`为目标IP地址；
- `00:15:5d:a7:02:89`为MAC地址；
- `[ether]` 和`on eth0`分别代表协议类型为以太网和网卡。

### ip neigh

同样也用于查看apr缓存表，直接体现IP地址与MAC地址之间的映射关系。

```bash
dantalion@Dantalion:~$ ip neigh
172.18.192.0 dev eth0 FAILED
192.168.1.99 dev eth0 FAILED
192.168.1.0 dev eth0 FAILED
172.18.192.1 dev eth0 lladdr 00:15:5d:a7:02:89 REACHABLE
```

- `172.18.192.0`、`192.168.1.99`、`192.168.1.0` → 全部 **FAILED**，说明本机尝试解析这些地址时没拿到 MAC，对应主机/网络不可达。
- `172.18.192.1` → 已解析到 `00:15:5d:a7:02:89`，状态 `REACHABLE`，说明默认网关（通常 `.1` 是网关）是正常的。

## ICMP

### ping

**连通性测试：**发送 **Echo Request**，收到 **Echo Reply** 就说明 IP 层可达。

因为不是 TCP/UDP，常被防火墙单独放行/拦截，所以：

- ping 不通 ≠ 一定网络坏了，可能是：

  - 没路由 → 主机不可达（Destination Unreachable）。

  - TTL 用完 → Time Exceeded。（`Time to live exceeded` → 表明包在环路里打转，比如 ping 容器时遇到的情况）

  - 被防火墙拦截 → 没响应

  > [!NOTE]
  >
  > **TTL：**
  >
  > TTL（Time To Live）是 **IP 报文头部**里的一个字段，表示这个报文还能经过多少个路由器（hop）。
  >
  > 初始值：由操作系统/应用设置（常见是Linux - 64、Windows - 128、255 等）。
  >
  > 每经过一个路由器：TTL 会减 1。
  >
  > TTL = 0 时：报文会被丢弃，并返回一个 ICMP 错误（`Time to live exceeded`）。

- ping 通 ≠ 服务一定正常，只能说明 IP 通。

```bash
dantalion@Dantalion:~$ ping 223.5.5.5
PING 223.5.5.5 (223.5.5.5) 56(84) bytes of data.
64 bytes from 223.5.5.5: icmp_seq=1 ttl=116 time=6.08 ms
...
--- 223.5.5.5 ping statistics ---
4 packets transmitted, 4 received, 0% packet loss, time 3154ms
rtt min/avg/max/mdev = 6.078/6.594/7.058/0.350 ms
```

- **bytes**：ICMP 负载 + 头部大小（与 `-s` 相关）。
- **icmp_seq**：序号，便于看是否乱序/丢包。
- **ttl**：回包到达你时剩余的 TTL；从该值大致可推测经过跳数（但各系统初始 TTL 不同）。
- **time**：该次 RTT。
- **statistics**：发送/接收/丢包率/总用时；最后一行给 RTT 统计。

- **持续探测每一跳**：像 `traceroute` 一样逐跳探路，同时对每一跳持续发包，统计 **丢包率、时延均值/抖动**。

>
> 注意到最后一行为启动Jenkins容器后映射的主机端口9090，容器内端口为8080，需要交互式进入容器内部才能看到。

重点看：

- **服务监听在哪个地址/端口**？（Local Address:Port）

  - 特殊情况：
    - `0.0.0.0:53` → 绑定在所有 IPv4 地址。
    - `127.0.0.1:53` → 仅本地回环可访问。
    - `10.255.255.254:53` → 仅绑定在指定网卡 IP。
    - `[::1]:323` → 仅绑定 IPv6 回环。
  - **端口号常见用途**：
    - `53` → DNS 服务。
    - `323` → chronyd / systemd-timesyncd（NTP 服务端口）。

- **监听状态正常吗**？（State 是否 LISTEN/ESTAB）

- **连接有无堆积**？（Recv-Q / Send-Q 是否异常）

  - **Recv-Q**
    - **接收队列**（Receive Queue）长度。
    - 表示内核为该套接字缓存的数据包数，如果应用层没及时读，值会变大。
    - **一般应该为 0**，否则说明应用没及时处理数据。

  - **Send-Q**

    - **发送队列**（Send Queue）长度。

    - 表示等待发出的数据量，如果网络不通或应用卡住，数值会升高。

    - 对 LISTEN 状态的 TCP 来说，数值表示 **最大连接 backlog**（允许的排队连接数）


### tcpdump

可以抓取指定接口、指定端口的流量（数据包）信息；常用来观察TCP三次握手，分析HTTP请求是否到达。

```bash
dantalion@Dantalion:~$ sudo tcpdump -i eth0 port 80
```

- **-i eth0**
   指定抓包的接口是 `eth0`。
  - 如果不写，`tcpdump` 会默认找一个“最合适的接口”（通常是第一个 up 状态的网卡）。
  - 常用选项：`-i any` 可以抓取所有接口上的包。
- **port 80**
   抓取 **源端口或目的端口为 80 的包**。
  - 也就是说既能抓到客户端访问 80 端口（HTTP）的请求，也能抓到服务端从 80 端口发出的响应。
  - 只抓目的端口为80的包：`dst`
    - 比如只看发往ssh的请求：`dst port 22`
  - 只抓源端口为80的包：`src`
    - 比如只看DNS服务的相应：`src port 53`

```bash
; EDNS: version: 0, flags:; udp: 1232
;; QUESTION SECTION:
;www.baidu.com.                 IN      A

;; ANSWER SECTION:
www.baidu.com.          526     IN      CNAME   www.a.shifen.com.
www.a.shifen.com.       594     IN      A       183.2.172.17
www.a.shifen.com.       594     IN      A       183.2.172.177

;; Query time: 10 msec
;; SERVER: 10.255.255.254#53(10.255.255.254) (UDP)
;; WHEN: Fri Aug 29 20:30:09 CST 2025
;; MSG SIZE  rcvd: 101
```

> [!IMPORTANT]
>
> 重点关注`status`、`ANSWER SECTION`、TTL、`SERVER`。

- `status: NOERROR`：说明查询成功

  - 常见状态：

    - `NXDOMAIN`：域名不存在
    - `SERVFAIL`：DNS 服务器出问题

    - `REFUSED`：拒绝查询

- `ANSWER SECTION`：DNS的回答

  - `www.baidu.com` 实际上是个 **CNAME 别名** → `www.a.shifen.com`

  - 最终解析到了两个 A 记录（IPv4 地址），可能做 DNS 轮询负载均衡。

  - 数字 `526` / `594` 表示 TTL（缓存有效时间，秒）。

  > [!NOTE]
  >
  > #### **CNAME：**
  >
  > **别名记录**，意思是“这个域名是另一个域名的别名”。
  >
  > `www.baidu.com` → CNAME → `www.a.shifen.com`
  >
  > 真正的域名是`www.a.shifen.com`，`www.baidu.com`是一个别名。
  >
  > #### **负载均衡：**
  >
  > 把大量的访问请求，分散到多台服务器上去处理，避免单台服务器被压垮。
  >
  > 常见方式：
  >
  > - **DNS 负载均衡**：返回不同的 IP，分散请求。
  > - **硬件/软件负载均衡器（如 Nginx、F5）**：在服务器前放一个“流量调度器”，转发请求到不同的后端。
  >
  > #### CDN：
  >
  > 内容发放网络，主要解决 **访问速度和稳定性** 的问题。
  >
  > 原理：
  >
  > - 在全球/全国各地部署很多缓存服务器（节点）。
  > - 用户访问时，通过 DNS 或 HTTP 302 跳转，找到最近的节点。
  > - 访问的不是源站（北京/美国的服务器），而是本地城市的 CDN 节点，所以速度快、延迟低。

- `SERVER`：
  - 回答的DNS服务器。

## 带宽监控

### iftop

像 `top` 一样显示主机的**网络连接和带宽使用情况**，可用于快速观察是否有异常流量。

```bash
#终端1监控eth0网卡流量
dantalion@Dantalion:~$ sudo iftop -i eth0

#终端2向1.1.1.1发送ping请求
dantalion@Dantalion:~$ ping 1.1.1.1
```

![image-20250830152643928](C:\Users\86133\AppData\Roaming\Typora\typora-user-images\image-20250830152643928.png)

- **顶部刻度条**：表示带宽范围。

- **中间连接列表**：显示每条连接的实时发送/接收速率（分 2s、10s、40s 平均值）。

  - 左侧是 **通信双方的 IP（或主机名）**。
    - `=>` 表示本机发出的流量（TX, transmit）。
    - `<=` 表示本机接收的流量（RX, receive）。

  - 右边的 **三列数字**：

    - 分别代表 **最近 2 秒、10 秒、40 秒** 的平均带宽使用情况。

    - 单位通常是 **Kbit/s 或 Mbit/s**。

- **底部统计**：显示全局的总发送、接收和合计流量。

### nload

用于实时监控指定网卡的 **入站（Incoming）** 和 **出站（Outgoing）** 流量。

它显示的内容包括：

- 当前速率（实时值）
- 平均速率
- 最大速率
- 已经传输的总流量

```bash

dantalion@Dantalion:~$ nload

#终端2向bilibili.com发送ping请求

```

### ifstat

轻量级的 **网络接口实时流量监控工具**，比`iftop` 更简洁，常用于巡检或排障时快速看网络带宽。

```bash
dantalion@Dantalion:~$ ifstat -t -i eth0 1 1
  Time           eth0
HH:MM:SS   KB/s in  KB/s out
22:12:34      0.00      0.00
```

**参数说明：**

- `-t`：以文本形式输出（否则为实时输出）；
- `-i eth0`：指定网卡`eth0`；
- `1 1`：每隔1秒刷新一次，总共输出1次。

## 防火墙

### iptables -L -n

- **`iptables`**：是 Linux 系统中用于配置防火墙规则的命令行工具。
- `iptables -L -n`列出当前的防火墙规则链，并以数字格式显示（而不是解析为主机名或域名），以便更直接地查看和理解。

```bash
dantalion@Dantalion:~$ sudo iptables -L -n
Chain INPUT (policy ACCEPT)
target     prot opt source               destination

Chain FORWARD (policy ACCEPT)
target     prot opt source               destination

Chain OUTPUT (policy ACCEPT)
target     prot opt source               destination
```

- `INPUT`、`FORWARD`、`OUTPUT`：
  - 分表表示系统所有入站、转发和出站流量的规则链。
- `policy ACCEPT`：
  - 默认策略都设置为 **ACCEPT**，这意味着 **没有任何具体规则** 阻止流量的进出。
- `target`：
  - 指定匹配规则时要执行的操作，比如 `ACCEPT`（允许通过）、`DROP`（丢弃数据包）。
- `prot`：
  - 协议类型，例如 `tcp`、`udp`、`icmp` 等。
- `opt`：
  - 表示任何额外的选项，通常为 `-`，表示没有设置。
- `source`：源 IP 地址。
- `destination`：目标 IP 地址。
