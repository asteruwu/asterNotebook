# 计算机系统监控

## 1. 系统负载异常但CPU占用率不高

### 情境1 - I/O等待

- **原因：** I/O 完成，导致负载高。
- **表现：**`top`里的`%wa`字段值明显偏高。
- **处理：**`iostat`查看哪个设备或进程拖慢了I/O。

### 情境2 - 僵尸/不可中断进程

- **原因：**有些进程卡在 `D` 状态（不可中断 I/O），占着队列，负载计入但 CPU 不消耗。
- **处理：**可通过`ps -eo pid,stat,cmd | grep " D"`排查。

### 情境3 - 内存不足，频繁换页

- **原因：**系统频繁把数据写到 swap，CPU 不高，但 load average 飙升。
- **处理：**用`vmstat`查看`si/so`情况。

## 2. 内存占用高，区分缓存和实际进程

1. **是否真的内存不足？**

   使用`free -m`命令，关注字段`available`，这是实际可用的内存；如果很低说明进程占用真的很高，否则大部分为缓存；所以`buff/cache` 字段占用高时并不是真的内存不足，Linux 会自动回收。

2. **排查大内存进程：**

   使用`top`命令，关注`%MEM`字段，找出内存占用大的进程。

## 3. 磁盘空间够，但 %util 高

### 原因：

1. **I/O负载过重：**

   - 应用频繁写入日志、大量读写数据库文件等
   - 内存不足导致频繁写swap

2. **磁盘面临大量随机I/O卡死：**

   - 机械硬盘（HDD）遇到大量随机读写（多个进程同时读写不同位置的文件）时，很容易卡死。

   > [!NOTE]
   >
   > **顺序I/O：**
   >
   > ​	数据存储在磁盘上是**连续的块**。
   >
   > ​	读写时磁头（HDD）或控制器（SSD）可以一口气读取大段数据。
   >
   > ​	速度快，吞吐量高。
   >
   > ​	常见场景：大文件拷贝、视频流、顺序扫描日志。
   >
   > **随机I/O：**
   >
   > ​	数据**分散**存放，读写时需要**频繁跳转位置**，导致**数据访问不连续**。
   >
   > ​	在 HDD 上尤其慢，因为磁头要不停寻道（转到不同位置）。
   >
   > ​	在 SSD 上虽然没有机械寻道，但随机 I/O 多了也会导致性能下降。
   >
   > ​	常见场景：数据库随机查询、频繁的小文件读写、缓存/索引查找。

3. **小文件读写过多：**

   - 每次打开/关闭文件都涉及 inode、元数据操作。
   - 即使文件总量不大，频繁的小文件操作也能把磁盘打满。

4. **单进程阻塞I/O（BIO）：**

   - 比如一个数据库单线程处理 I/O，请求堆积，表现为 `%util` 高。

5. **硬件或驱动问题：**

   - 磁盘坏块、RAID 控制器问题，可能导致 I/O 请求处理异常缓慢。

   > [!NOTE]
   >
   > **RAID 控制器：**负责管理多个磁盘，提高磁盘吞吐量、防止读写过程中数据丢失（做副本）、监控磁盘健康的核心组件。

### 处理：

1. 使用`iostat -x`命令，查看`await`和`%util`字段查看磁盘总体I/O情况；
2. 使用`sudo iotop`命令查看具体是哪个进程导致大量I/O。

## 4. 如果某个服务挂掉了，怎么进行快速确认

1. 使用`ps aux | grep <服务名>`进行基本确认，若没有输出信息，说明服务已经挂掉；
2. 如果有输出，关注`STAT`字段内容是否为`D`（可能处于I/O等待，异常卡死）；
3. 如果彻底消失，则需查看日志文件。

# 文件系统

## 1. inode 耗尽但磁盘空间还很多

**inode：**

在 Linux 文件系统中：

- 每个文件都对应一个 **inode（索引节点）**。
- inode 保存的是文件的“属性信息”，而不是文件内容本身。
- 可以使用命令`df -i`查看各个文件系统的inode使用情况：

```bash
dantalion@Dantalion:~/shell$ df -i
Filesystem       Inodes   IUsed    IFree IUse% Mounted on
none             998293       5   998288    1% /usr/lib/modules/6.6.87.2-microsoft-standard-WSL2
none             998293       2   998291    1% /mnt/wsl
...
C:\                 999 -999001  1000000     - /mnt/c
D:\                 999 -999001  1000000     - /mnt/d
tmpfs            998293      41   998252    1% /run/user/1001
```

**元数据：**

- 文件的属性信息，不包括实际内容。
- 元数据操作（metadata operations）就是修改或查询这些属性。

常见元数据操作：

- `ls -l` → 查看权限、大小、时间戳（就是读 inode 信息）
- `chmod / chown / touch` → 修改 inode 里的权限、所有者、时间戳
- 创建/删除文件 → 修改目录项 + inode 分配/释放
- 打开文件 → 需要查找目录项，再定位 inode

### 原因：

1. 每个文件都要占用1个inode，但inode数量有限；
2. 大量小文件导致inode先于磁盘空间被耗尽。
