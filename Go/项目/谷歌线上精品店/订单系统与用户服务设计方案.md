# 1. 核心架构与业务流程

**1.1 设计目标**

针对高并发场景下对于服务性能与数据一致性的要求，通过流量削峰、Lua脚本 + Redis Stream、批量聚合写库实现上游服务承接流量压力、下游数据库减负以及最终数据一致性。

**1.2 订单生成流程** | 上游Producer端

解耦扣减库存和订单信息的发送，此处只接收请求$\rightarrow$发起扣减库存的请求$\rightarrow$返回响应。

- **步骤序列**
  1. **Request**: 前端发起 `CreateOrder(OrderItems, UserToken)` 请求；
  2. **Prepare OrderID**: 基于传入的变量生成可唯一标识的`OrderID`；
  3. **RPC Call**: 通过gRPC调用productcatalogservice的`ChargeProduct`方法来扣减库存；
  4. **Response**: 立即返回前端：“订单已接收，处理中”，附带 `OrderID`。

**1.3 订单落库流程** | 下游 [Kafka] Consumer 端

采用消费者组模式，支持多副本并行消费。

- **步骤序列**
  1. **Poll**: 从Kafka拉取一批订单消息；
  2. **Logic**: 遍历消息，解析`Order`对象；
     - 引入重试机制，在这一步提取出故障消息，在有限次重试失败后提交到死信队列
  3. **Idempotency Check**: 
     - **Action**: 直接执行 `INSERT IGNORE INTO orders ...` 或者 `INSERT INTO ... ON DUPLICATE KEY UPDATE ...`。
     - **Result**：
       - 如果数据库返回“影响行数 1” $\rightarrow$ 成功；
       - 如果数据库报错“Duplicate entry” $\rightarrow$ 说明重复消费，**直接捕获异常并吞掉**，视为成功；
  4. **Manual ACK**: 批量处理完成后，手动提交偏移量。

**1.4 异步消费与补偿** | 上游 [Redis] Consumer端

通过异步的Consumer与Recovery执行阻塞批量读取Redis Stream$\rightarrow$遍历并生成Order$\rightarrow$发送到Kafka。

- **[Redis Consumer] 步骤序列**
  1. **XReadGroup**: 阻塞读取Redis Stream的消息；
  2. **Produce Message**: 遍历消息，提取订单信息，组装成`Order`对象；
  3. **Send Message To Kafka**: 打包成Kafka消息并发送；
  4. **ACK**: 收到发送成功的响应后向Redis Stream发送ACK。
- **[Recovery] 步骤序列**
  1. **XPending**: 定时读取Pending List的消息；
  2. **XClaim**: 强制夺权，声明信息归属；
  3. **Produce Message**: 遍历消息，提取订单信息，组装成`Order`对象；
  4. **Send Message To Kafka**: 打包成Kafka消息并发送；
  5. **ACK**: 收到发送成功的响应后向Redis Stream发送ACK。

func (o *OrderService) createOrder(context, OrderItems, UserToken) (string, bool, string) {
    OrderID, err := o.PrepareOrderID(ctx, UserToken)
    if err != nil {
        return nil, false, fmt.Sprintf("[CreateOrder] failed to create OrderID: %v", err)
    }
    // Lua脚本的逻辑在productcatalogservice的chargeproduct方法中，待修改

        return OrderID, true, "[CreateOrder] start creating order..."
    } else {
        return nil, false, fmt.Sprintf("CreateOrder] failed to create order: %s", res.message)
    }
}

func (o *OrderService) insertOrderToMySQL(ctx, messages) (bool, error) {
    for _, msg := range messages {
        order, err := o.getOrderFromMessage(ctx, msg)
        if err != nil {
            continue
        }
        if err := o.db(ctx, 'INSERT ...', order.OrderID, order.UserID, order.OrderItems, order.TotalPrice, order.Status, order.Address); err == "Duplicate entry" || err == nil {
            continue
        } else {
            log.Errorf("[InsertOrderToMySQL] failed to insert: %v", err)
            return false, err
        }
    }
    kf.ACK(consumerName, msg)
    return true, nil
}

func (o *OrderService) getOrderFromMessage(ctx, message) (*Order, error) {
    for i := 0; i < 3; i++ {
        result, err := kf.Parse(message)
        if err == nil {
            return &Order{
            OrderID:	result.order_id,
            UserID:		result.user_id,
            OrderItems:	result.order_items,
            TotalPrice:	result.total_price,
            Status:		result.status,
            Address:	result.address.
            }, nil
        }
    }
    kf.SendToDeadList(message)
    return nil, err
}

func (o *OrderService) startRecoveryStream(ctx) {
    ticker := NewTicker(1 * time.Minute)
    consumerName := prepareConsumerNameRedis()
    
    for {
        select {
        case <- ctx.Done():
            return
        case ticker.C:
            pendingList := XPending(streamKey, batchCount ...)
            if len(pendingList) == 0 {
                continue
            }
            messages := XClaim(pendingList ...)
            OrderList := o.produceOrderForKafka(messages)
            success := o.sendOrderToKafka(OrderList)
            if success {
                XAck(streamKey, consumerName ...)
            }
        }
    }
}
