# 变量

## 定义

1. 用`var`声明

   ```go
   var b bool = true //定义布尔变量并赋值为真
   var name string = "aster" //定义字符串变量并赋值为aster
   var s int = 3 //定义整型变量并赋值为3
   ```

   > [!NOTE]
   >
   > 标准的变量定义方式，需要指明变量类型，适合不需要马上给变量赋值的情况

   ```go
   //一次性定义多个变量
   var (
   	addr string = "ShenZhen"
       name string = "aster"
       age int
   )
   ```

2. `:=`短变量声明

   - 左边的变量 **必须是新定义的**。

   - 右边给了初始值，Go 会**自动推断**类型。

   ```go
   //go自动判断为字符串（双引号，单引号为字符）、整型、切片
   m1 := "aster"
   m2 := 3
   m3 := []int{1,2,3}
   ```

   ```go
   var name string
   name := "aster" //错误，已经定义过变量name
   ```

## 类型

- **值类型**

  - **布尔型**

  - **整型**

    - 有符号：`int8`, `int16`, `int32`, `int64`, `int`

    - 无符号：`uint8`（即 `byte`）, `uint16`, `uint32`, `uint64`, `uint`

  - **浮点型**
    - `float32`, `float64`

  - **复数**

    - `complex64`（实部和虚部分别是 `float32`）

    - `complex128`（实部和虚部分别是 `float64`）

  - **字符型**

    - `byte`（`uint8` 的别名，常用于表示 ASCII 字符）

    - `rune`（`int32` 的别名，表示 Unicode 字符）

- **字符串类型**

  - `string`（不可变的字节序列）

- **派生/复合类型**

  - **指针**

    ```go
    var p *int //定义一个整型的指针变量
    ```

  - **数组**

    ```go
    var arr [3]int = [3]int{4, 5, 6} //定义一个整型数组，大小为3
    ```

  - **切片**（更常用，动态大小）

    ```go
    var slice []int = []int{1, 2, ... , n} //定义一个切片，大小由切片内的元素个数n决定
    ```

  - **映射**

    ```go
    var m map[string]int = map[string]int{"a":1, "b":2} //定义一组映射关系，类型为键值对
    ```

  - **结构体**

    ```go
    type Person struct{
        name string
        age int
    } //定义一个以结构体为底层结构的新变量类型，名为Person
    ```

  - **函数类型**

    ```go
    var f func(int, int) int //定义一个输入参数为2个整型数、返回为1个整型数的函数变量
    ```

# 运算符

## 算术运算符

用于数字类型（整数、浮点数）之间的基本运算。

```go
a, b := 10, 3
fmt.Println(a + b) // 13
fmt.Println(a - b) // 7
fmt.Println(a * b) // 30
fmt.Println(a / b) // 3
fmt.Println(a % b) // 1
```

## 赋值运算符

用于给变量赋值，并可结合运算操作。

```go
a := 5
a += 3 // a = 8
a *= 2 // a = 16
```

## 比较运算符

用于比较两个值，返回布尔值（`true` 或 `false`）。

```go
a, b := 5, 3
fmt.Println(a == b) // false
fmt.Println(a != b) // true
fmt.Println(a > b)  // true
```

## 逻辑运算符

用于布尔类型变量。

```go
a, b := true, false
fmt.Println(a && b) // false
fmt.Println(a || b) // true
fmt.Println(!a)     // false
```

## 位运算符

用于整数的二进制位操作。

```go
a, b := 6, 3 // 二进制：6=110, 3=011
fmt.Println(a & b)  // 2  -> 010
fmt.Println(a | b)  // 7  -> 111
fmt.Println(a ^ b)  // 5  -> 101
fmt.Println(a &^ b) // 4  -> 100
fmt.Println(a << 1) // 12 -> 1100
fmt.Println(a >> 1) // 3  -> 011
```

## 其他运算符

取出地址和取出指针所指向的值。

```go
a := 10
p := &a      // p 是 a 的指针
fmt.Println(*p) // 10
```

# 函数和方法

在 Go 里：

- **函数（function）**：独立存在，可以直接调用。

  ```go
  //格式
  func 函数名(参数列表[参数名 变量类型]) 返回值类型 {
      // 函数体
      return 返回值
  }
  
  //定义
  func add(a int, b int) int {
      return a + b
  }
  
  //调用
  sum := add(3, 5)
  fmt.Println(sum) // 8
  ```

  > 输入CPU使用率，判断是否要告警

  ```go
  package main
  
  import "fmt"
  
  func checkCPUUsage(usage int) string {
  	switch {
  	case usage > 85:
  		return "CPU告警：过高"
  	case usage > 60:
  		return "CPU告警：较高"
  	default:
  		return "CPU正常"
  	}
  }
  
  func main() {
  	fmt.Println(checkCPUUsage(90))
  	fmt.Println(checkCPUUsage(70))
  	fmt.Println(checkCPUUsage(20))
  }
  ```

- **方法（method）**：绑定在某个类型（通常是 `struct`）上的函数。

  ```go
  type Student struct {
      name string
      age  int
  }
  
  // 定义方法
  func (s Student) sayHello() {
      fmt.Printf("大家好，我是 %s，今年 %d 岁\n", s.name, s.age)
  }
  
  func main() {
      stu := Student{name: "Tom", age: 20}
      stu.sayHello() // 调用方法
  }
  ```

  > SRE巡检场景，最终结果整合打包成JSON格式输出

  ```go
  package main
  
  import (
  	"encoding/json" // 导入 json 包，用于将 Go 结构体编码为 JSON 格式
  	"fmt"           // 导入 fmt 包，用于打印输出
  )
  
  // 定义 Server 结构体，表示一台服务器的基本信息和资源使用情况
  type Server struct {
  	name      string // 服务器名称
  	cpuUsage  int    // CPU 使用率，单位为百分比
  	memUsage  int    // 内存使用率，单位为百分比
  	diskUsage int    // 磁盘使用率，单位为百分比
  }
  
  // 定义 Server 的方法 check_cpu，用于检查 CPU 使用情况
  func (s Server) check_cpu() string {
  	if s.cpuUsage > 85 { // 如果 CPU 使用率大于 85%
  		return "CPU告警" // 返回告警信息
  	} else {
  		return "CPU正常" // 否则返回正常信息
  	}
  }
  
  // 定义 Server 的方法 check_mem，用于检查内存使用情况
  func (s Server) check_mem() string {
  	if s.memUsage > 80 { // 如果内存使用率大于 80%
  		return "内存告警" // 返回告警信息
  	} else {
  		return "内存正常" // 否则返回正常信息
  	}
  }
  
  // 定义 Server 的方法 check_disk，用于检查磁盘使用情况
  func (s Server) check_disk() string {
  	if s.diskUsage > 90 { // 如果磁盘使用率大于 90%
  		return "磁盘告警" // 返回告警信息
  	} else {
  		return "磁盘正常" // 否则返回正常信息
  	}
  }
  
  // 定义 check_result 结构体，用于存储服务器巡检结果，方便转换为 JSON
  type check_result struct {
  	Server string `json:"server"` // 使用 JSON 标签指定字段名为 "server"
  	Cpu    string `json:"cpu"`    // JSON 标签 "cpu"
  	Mem    string `json:"mem"`    // JSON 标签 "mem"
  	Disk   string `json:"disk"`   // JSON 标签 "disk"
  }
  
  // 定义 Server 的方法 check_all，汇总检查 CPU、内存、磁盘并返回结构化结果
  func (s Server) check_all() check_result {
  	return check_result{
  		Server: s.name,      // 填充服务器名称
  		Cpu:    s.check_cpu(), // 调用 check_cpu 方法获取 CPU 状态
  		Mem:    s.check_mem(), // 调用 check_mem 方法获取内存状态
  		Disk:   s.check_disk(), // 调用 check_disk 方法获取磁盘状态
  	}
  }
  
  // main 函数，程序入口
  func main() {
  	// 创建两台服务器对象，分别初始化各自的资源使用情况
  	srv1 := Server{name: "web01", cpuUsage: 20, memUsage: 90, diskUsage: 20}
  	srv2 := Server{name: "db01", cpuUsage: 90, memUsage: 20, diskUsage: 95}
  
  	// 对服务器进行巡检，并将结果存入切片
  	results := []check_result{srv1.check_all(), srv2.check_all()}
  
  	// 将巡检结果切片转换为格式化的 JSON 字符串
  	jsonData, err := json.MarshalIndent(results, "", "\t") // 第二个参数 "" 表示前缀为空，"\t" 表示缩进使用制表符
  	if err != nil {                                        // 检查是否转换失败
  		fmt.Println("json转换失败：", err) // 打印错误信息
  		return
  	}
  
  	// 打印最终 JSON 结果
  	fmt.Println(string(jsonData))
  }
  ```

  输出：

  ```json
  [
          {
                  "server": "web01",
                  "cpu": "CPU正常",
                  "mem": "内存告警",
                  "disk": "磁盘正常"
          },
          {
                  "server": "db01",
                  "cpu": "CPU告警",
                  "mem": "内存正常",
                  "disk": "磁盘告警"
          }
  ]
  ```

  > [!NOTE]
  >
  > \` \`是结构体标签，可以给结构体中的元素添加元数据。
  >
  > 常见用途：
  >
  > - **JSON / XML / YAML 序列化**：告诉库输出时用哪个名字
  > - **数据库 ORM**：指定表字段映射
  > - **验证框架**：指定规则（例如 \`validate:"required,min=3"\`）

# 匿名函数和闭包

**匿名函数**：

- **匿名函数** 就是一个没有名字的函数，可以像其他值一样赋给变量。通过这种方式，匿名函数就成为了一个变量的值，而这个变量就可以用来调用匿名函数。

  **关键点：**

  - **匿名函数是一个值**，就像数字、字符串或者切片、字典一样可以赋值给变量。
  - 可以通过这个变量调用匿名函数，并传递所需的参数。

  > 将匿名函数赋值给变量

  ```go
  package main
  
  import "fmt"
  
  func main() {
  	greet := func(name string) {
  		fmt.Println("hello,", name)
  	}
  	greet("go")
  }
  ```

  > 将匿名函数作为函数的参数传递进行**排序**

  ```go
  package main
  
  import (
  	"fmt"
  	"sort"
  )
  
  func main() {
  	type ServiceHealth struct {
  		ServiceName  string
  		ResponseTime float64
  	}
  
  	services := []ServiceHealth{
  		{"Auth Service", 150.5},
  		{"API Gateway", 100.2},
  		{"Database", 200.3},
  		{"Cache", 50.0},
  	}
  
  	sort.Slice(services, func(i, j int) bool {
  		return services[i].ResponseTime > services[j].ResponseTime
  	})
  
  	fmt.Println("按降序排列后的服务响应时间为：")
  	for _, service := range services {
  		fmt.Printf("%s %.2fms\n", service.ServiceName, service.ResponseTime)
  	}
  }
  ```

  其中用到`sort`包的`Slice()`函数，需要传递两个参数：

  - **要进行排序的数据集**，如示例代码中以结构体为基本元素的切片；
  - **自定义的排序逻辑**，也就是**匿名函数`func(i, j int) bool {}`**，`i`、`j`代表的是数据集元素的索引，当执行匿名函数后返回的`bool`值为`true`时，将索引`i`对应的元素排在索引`j`对应的元素的前面。

  > 匿名函数作回调函数

  实现具体动作与执行逻辑的解耦。(类似接口)

  ```go
  package main
  
  import (
  	"fmt"
  )
  
  func HealthCheck(service string, callback func(string)) { //使用匿名函数作为回调函数（形参）返回服务状态
  	if service == "API Gateway" {
  		callback("API Gateway is down!")
  	} else {
  		callback(fmt.Sprintf("%s is healthy.", service))
  	}
  }
  
  //定义具体动作的函数
  func SendEmail(message string) {
  	fmt.Println("Sending Email:", message)
  }
  
  func LogStatus(message string) {
  	fmt.Println("Logging status:", message)
  }
  
  func main() {
  	services := []string{"Auth Service", "API Gateway", "Cache"}
  
  	for _, service := range services {
          //定义调用不同回调函数的逻辑
  		if service == "API Gateway" {
  			HealthCheck(service, SendEmail)
  		} else {
  			HealthCheck(service, LogStatus)
  		}
  	}
  }
  ```

**闭包**：

- 闭包是函数与其引用环境变量的组合。在 Go 中，函数可以“捕获”外部作用域的变量，并且在函数外部仍然能够访问这些变量。闭包通常用于封装状态或创建回调函数。

  > 封装模块名

  ```go
  func Logger(module string) func(msg string) {
      return func(msg string) {
          fmt.Printf("[%s] %s\n", module, msg)
      }
  }
  
  func main() {
      apiLogger := Logger("API")
      dbLogger := Logger("DB")
  
      apiLogger("请求开始")
      dbLogger("连接数据库成功")
  }
  ```

  - 不需要每次打印日志时都传入"API"或"DB"，直接使用函数即可

  > 动态限流器

  ```go
  package main
  
  import (
  	"fmt"
  	"time"
  )
  
  // InterfaceLimiter 返回一个闭包函数，用来限制某个接口的访问频率。
  // limit  表示在窗口时间内允许的最大请求数
  // window 表示窗口时间长度（例如 5 秒内最多 3 次）
  func InterfaceLimiter(limit int, window time.Duration) func() bool {
  	// timestamps 用于保存每次允许访问的时间点
  	// 因为在闭包外定义，闭包可以“记住”它，每次调用时都会共享这个切片
  	timestamps := []time.Time{}
  
  	// 返回闭包函数
  	// 每次调用这个闭包函数就相当于对接口发起一次访问请求
  	return func() bool {
  		now := time.Now()          // 当前时间
  		valid := []time.Time{}     // 用于存放窗口内仍然有效的时间戳
  
  		// 1. 过滤掉窗口外的时间戳（超过 window 的全部丢弃）
  		for _, t := range timestamps {
  			if now.Sub(t) < window { // 如果当前时间 - 之前的时间 < 窗口长度，表示仍在窗口内
  				valid = append(valid, t) // 留下仍在窗口内的请求时间
  			}
  		}
  		// 2. 将 timestamps 替换为过滤后的有效时间戳
  		timestamps = valid
  
  		// 3. 判断当前窗口内请求数是否小于 limit
  		if len(timestamps) < limit {
  			// 还没超限，允许访问
  			// 同时把当前时间加入 timestamps（占用一个名额）
  			timestamps = append(timestamps, now)
  			return true // 表示放行
  		} else {
  			// 已经达到 limit，不允许访问
  			return false
  		}
  	}
  }
  
  func main() {
  	// 分别为两个接口创建独立的限流器闭包
  	// apiLimiter：5 秒内最多允许 3 次访问
  	apiLimiter := InterfaceLimiter(3, 5*time.Second)
  	// dbLimiter：3 秒内最多允许 2 次访问
  	dbLimiter := InterfaceLimiter(2, 3*time.Second)
  
  	// 模拟连续 10 次请求 API 接口，每秒一次
  	for i := 1; i <= 10; i++ {
  		if apiLimiter() {
  			fmt.Printf("第%d次api接口允许访问\n", i)
  		} else {
  			fmt.Printf("第%d次api接口拒绝访问\n", i)
  		}
  		time.Sleep(1 * time.Second) // 模拟 1 秒一个请求
  	}
  
  	// 模拟连续 10 次请求 DB 接口，每秒一次
  	for i := 1; i <= 10; i++ {
  		if dbLimiter() {
  			fmt.Printf("第%d次db接口允许访问\n", i)
  		} else {
  			fmt.Printf("第%d次db接口拒绝访问\n", i)
  		}
  		time.Sleep(1 * time.Second) // 模拟 1 秒一个请求
  	}
  }
  ```

  - API接口和DB接口的有效访问时间timestamps相互独立，且不使用全局变量就能够记住各自的有效时间

# 接口

**语法**

```go
type 接口名 interface {
    方法名1(参数列表) 返回值类型
    方法名2(参数列表) 返回值类型
    ...
}
```

1. 多个类型实现一个接口

   ```go
   package main
   
   import "fmt"
   
   //定义类型为接口的变量，声明接口的实现方法（名）
   type Animal interface {
   	speak() string
   }
   
   //定义3个结构体变量
   type Cat struct {}
   type Dog struct {}
   type Cow struct {}
   
   //各自的接口实现方法 -- 只要变量类型的方法能够与接口中声明的匹配，则可将结构体变量赋给接口变量
   func (c Cat) speak() string {
   	return "Meow"
   }
   
   func (d Dog) speak() string {
   	return "Woof"
   }
   
   func (c Cow) speak() string {
   	return "Mow"
   }
   
   func main() {
   	animals := []Animal{Cat{}, Dog{}, Cow{}} //定义一组以结构体变量为元素的接口切片（已经匹配，所以可实现）
   	for _, a := range animals { //用range会返回索引值和元素值，使用"_"占位符可以不输出索引
   		fmt.Println(a.speak()) //方法实现
   	}
   }
   ```

2. 一个类型实现多个接口

   ```go
   package main
   
   import (
   	"fmt"
   )
   
   //定义2个类型为接口的变量，分别声明各自的实现方法
   type Runner interface {
   	Run()
   }
   
   type Swimmer interface {
   	Swim()
   }
   
   //定义1个类型为结构体的变量
   type athlete struct{}
   
   //将这一变量连上两个接口，分别写入接口实现方法
   func (a athlete) Run() {
   	fmt.Println("Run fast!")
   }
   
   func (a athlete) Swim() {
   	fmt.Println("Swim smooth!")
   }
   
   func main() {
       //定义2个接口变量
   	var a Runner
   	var b Swimmer
   
       //把结构体变量赋给接口，使之能够通过结构体走对应的实现方法
   	a = athlete{}
   	a.Run()
   	b = athlete{}
   	b.Swim()
   }
   ```

3. 接口解耦

   ```go
   package main
   
   import "fmt"
   
   // 定义接口
   type Logger interface {
   	Log(msg string)
   }
   
   // 控制台日志实现
   type ConsoleLogger struct{}
   
   func (ConsoleLogger) Log(msg string) {
   	fmt.Println("[Console]", msg)
   }
   
   // 业务逻辑，依赖接口而不是具体实现
   func Process(l Logger) {
   	l.Log("start processing...")
   	// ... 省略业务逻辑
   	l.Log("end processing...")
   }
   
   func main() {
   	var logger Logger = ConsoleLogger{}
   	Process(logger)
   }
   ```

   > [!IMPORTANT]
   >
   > 接口就是一个统一的插口，业务逻辑只认插口，不关心后面接的是“显示器、文件、数据库”。这样日志怎么存储都能随意切换，这就是解耦。


## Mock

- **Mock** 是测试中常用的一种手法，用来**模拟接口的行为**，避免真实依赖带来的副作用。

- 能够验证告警逻辑、自动恢复逻辑、报警通知流程。

- **实际应用**

  | 场景                  | 接口的作用                                                   | Mock 的作用                                                  |
  | --------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
  | **日志收集工具**      | 定义 `LogSink` 接口（写 ElasticSearch、写本地、写 Kafka…），主程序可插拔。 | 测试时用 `MockLogSink` 验证数据格式正确，无需真的连外部系统。 |
  | **配置管理/文件操作** | 定义 `ConfigLoader` 接口（从本地、远程、API加载配置）。      | 测试时用 `MockConfigLoader` 模拟各种配置返回或错误状态。     |
  | **健康检查工具**      | 定义 `Checker` 接口（HTTP、TCP、Ping…）。                    | 测试时用 `MockChecker` 模拟不同服务的健康状态，验证告警逻辑。 |
  | **限流/熔断器**       | 定义 `Store` 接口（存储计数器：内存、Redis、Etcd…）。        | 测试时用 `MockStore` 模拟限流/错误情况，检查行为是否符合预期。 |

> 健康检查工具

![Go_interface-接口+mock.drawio](../../dio图表/SRE/Go_interface-接口+mock.drawio.svg)

-  `main` 中使用真实检查器（HTTP/TCP）
- 测试中使用 `MockChecker` 模拟结果
- `NotifyIfUnhealthy` 统一处理接口返回错误
- 错误包装 + `defer` 保证资源释放

`main.go`

```go
package main

import (
	"fmt"
	"net"
	"net/http"
	"time"
)

// Checker 接口定义了健康检查的契约
// 任何实现了Check()方法的类型都可以作为健康检查器
type Checker interface {
	Check() error
}

// HTTPChecker 用于检查HTTP服务的健康状态
type HTTPChecker struct {
	URL string // 要检查的HTTP地址
}

// Check 实现Checker接口，执行HTTP健康检查
func (h HTTPChecker) Check() error {
	// 发送GET请求
	resp, err := http.Get(h.URL)
	if err != nil {
		return err // 网络错误或连接失败
	}
	defer resp.Body.Close() // 确保响应体被关闭，防止资源泄露

	// 检查HTTP状态码，200表示健康
	if resp.StatusCode != 200 {
		return fmt.Errorf("服务出错，状态码：%d", resp.StatusCode)
	}

	return nil // 检查通过
}

// TCPChecker 用于检查TCP服务的健康状态
type TCPChecker struct {
	IP      string        // 目标IP地址
	Port    int           // 目标端口号
	Timeout time.Duration // 连接超时时间
}

// Check 实现Checker接口，执行TCP健康检查
func (t TCPChecker) Check() error {
	// 格式化地址为 "IP:Port" 形式
	addr := fmt.Sprintf("%s:%d", t.IP, t.Port)
	
	// 尝试建立TCP连接，带有超时控制
	conn, err := net.DialTimeout("tcp", addr, t.Timeout)
	if err != nil {
		return fmt.Errorf("%w", err) // 包装原始错误
	}
	defer conn.Close() // 确保连接被关闭
	
	return nil // 连接成功
}

// NotifyIfUnhealthy 执行健康检查并在不健康时返回错误
// 接受任何实现了Checker接口的类型
func NotifyIfUnhealthy(c Checker) error {
	err := c.Check() // 调用具体的检查方法
	if err != nil {
		// 使用错误包装提供更详细的上下文信息
		return fmt.Errorf("连接出错：%w", err)
	}
	return nil // 服务健康，无错误
}

func main() {
	// 测试HTTP健康检查
	var checker Checker = HTTPChecker{URL: "http://baidu.com"}
	err := NotifyIfUnhealthy(checker)
	if err == nil {
		fmt.Println("HTTP连接成功")
	} else {
		fmt.Println("HTTP", err) // 输出详细的错误信息
	}

	// 测试TCP健康检查
	checker = TCPChecker{
		IP:      "127.0.0.1",        // 本地回环地址
		Port:    8080,               // 常用开发端口
		Timeout: 10 * time.Second,   // 10秒超时
	}
	err = NotifyIfUnhealthy(checker)
	if err == nil {
		fmt.Println("TCP连接成功")
	} else {
		fmt.Println("TCP", err)
	}
}
```

# 错误处理

Go强调显式错误处理，在**函数返回值**中就能看到错误，不需要猜测隐藏的异常，提高稳定性和可读性。

- **可预期的业务错误**（比如文件不存在、连接超时）→ 用 `error` 返回。
- **不可恢复的运行时错误**（比如数组越界、nil 指针、严重逻辑错误）→ `panic` 触发，配合 `recover` 挽救程序。

## error

- 函数返回 `(T, error)`。
- 调用方显式检查 `if err != nil`。

> 以读取配置为例

```go
package main

import (
	"errors"
	"fmt"
)

func ReadConfig(filename string) (string, error) {
	if filename == "" {
		return "", errors.New("filename cannot be empty")
	}
    //模拟读取成功
	return "config content", nil
}

func main() {
	content, err := ReadConfig("")
	if err != nil {
		fmt.Println("配置读取失败：", err)
		return
	}
	fmt.Println("配置读取成功：", content)
}
```

输出示例

```go
配置读取失败： filename cannot be empty
```

## panic

`panic` 会**让当前函数立刻停止执行**，并开始沿调用栈向上“**恐慌传递**”，依次执行所有已注册的 `defer`，然后终止程序（如果没人 `recover`）。

**适用场景**

- 逻辑绝不该发生的情况（比如程序 Bug）。
- 初始化失败且无法恢复。
- **不是**常规错误处理方式。

```go
package main

import "fmt"

func mustPositive(n int) {
    if n <= 0 {
        panic(fmt.Sprintf("n must > 0, got %d", n))
    }
}

func main() {
    mustPositive(0)
    fmt.Println("这行不会执行") //真的不会输出
}
```

## recover

`recover` 只能在 `defer` 中使用，能捕获当前 goroutine 的 `panic`，阻止程序崩溃，确保程序能够恢复并继续执行。

```go
package main

import "fmt"

func risky() {
    panic("something went wrong")
}

func main() {
    // 提前注册defer，捕获panic
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("捕获到 panic:", r)
        }
    }()

    risky() // 可能会panic
    fmt.Println("程序继续运行")
}
```

执行时：

1. `main` 先执行，遇到 `defer func(){…}()` → 注册一个延迟函数，暂时不执行。

2. 调用 `risky()`。

3. `risky()` 触发 `panic("something went wrong")`。

4. Go 运行时发现 panic，开始沿调用栈向上退：

   - 把 `risky()` 的调用栈展开。

   - 在 `main()` 里发现有 `defer`，就执行这个 `defer`。

5. `defer` 里的 `recover()` 捕获到了 panic 值 `"something went wrong"`，从而“挽救”了程序，没有直接退出。

6. `recover()` 返回后，`panic` 状态被清除，`main()` 从 `risky()` 返回点继续向下执行。

7. 所以 `fmt.Println("程序继续运行")` 这行**可以被执行**。

> [!IMPORTANT]
>
> 先注册延迟函数，遇到`panic`再往上查找延迟函数，延迟函数中的`recover()`捕获`panic`并清除。

> 综合示例 HTTP服务中的防御式错误处理

```go
package main

import (
	"errors"
	"fmt"
	"net/http"
)

func ReadData() (string, error) {
    //模拟读取数据错误的情形
	return "", errors.New("data not found")
}

//定义http服务的处理器，负责解析请求头、返回响应信息
func handler(w http.ResponseWriter, r *http.Request) {
	//定义延迟函数捕获panic并优雅地退出
    defer func() {
		if err := recover(); err != nil {
			fmt.Println("捕获panic：", err)
			http.Error(w, "服务异常", 500)
		}
	}()

    //调用读取数据的函数并返回错误信息
	data, err := ReadData()
	if err != nil {
		http.Error(w, "请求失败："+err.Error(), 400)
		fmt.Println("请求失败：", err.Error())
		return
	}

    //不出错则返回数据
	fmt.Fprintf(w, "Data: %s", data)
}

func main() {
    //指定路由以及处理器
	http.HandleFunc("/", handler)
    //启动http服务，设置监听的端口8000
	http.ListenAndServe(":8000", nil)
}
```

- 输出示例

```
[控制台]请求失败： data not found
```
