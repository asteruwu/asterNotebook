# 错误处理

1. **错误是返回值，不是异常**

   - Go 里没有 `try/catch`，错误就是一个普通的返回值。

   - 调用函数时，习惯上会返回 `(result, error)` 这种形式。

2. **为什么要 `if err != nil`**

   - 保证出错时立刻处理，不让错误“默默传播”。

   - 这种风格叫 **early return**（错误优先，正常逻辑最后）。

3. 常见写法

   ```go
   func doSomething(path string) (string, error) {
       data, err := os.ReadFile(path) //返回两个值，类型分别为字符串和error
       if err != nil {
           return "", err // 出错立刻返回
       }
       return string(data), nil
   }
   
   func main() {
       result, err := doSomething("config.txt")
       if err != nil {
           fmt.Println("读取失败:", err)
           return
       }
       fmt.Println("读取成功:", result)
   }
   ```

# os

## 文件操作

- **常用操作**

  - **读/写小文件**：`os.ReadFile`, `os.WriteFile`
    - `os.ReadFile(name string) ([]byte, error)`
      - 参数：`name` → 文件路径
      - 返回：文件内容（字节切片），`error`
      - 用途：一次性读完整个文件（配置文件读取常用）。
    - `os.WriteFile(name string, data []byte, perm FileMode) error`
      - 参数：
        - `name` → 文件路径
        - `data` → 要写入的字节数据
        - `perm` → 文件权限（如 `0644`）
      - 返回：`error`
      - 用途：一次性写入整个文件（覆盖写）。
  - **日志写入**：`os.OpenFile`
    - `os.OpenFile(name string, flag int, perm FileMode) (*os.File, error)`
    - 参数：
      - `name` → 文件路径
      - `flag` → 打开方式（如 `os.O_RDONLY`, `os.O_WRONLY`, `os.O_RDWR`, `os.O_CREATE`, `os.O_APPEND`, `os.O_TRUNC`）
      - `perm` → 文件权限（如 `0644`）
    - 返回：`*os.File`，`error`
    - 用途：最灵活的打开方式（常用在日志写入）。

  - **判断文件是否存在**：`os.Stat` + `os.IsNotExist`
  - **删除/重命名**：`os.Remove`, `os.Rename`
  - **目录操作**：`os.MkdirAll`, `os.ReadDir`
  - **读取并存储命令行参数：**`os.Args`
    - `os.Args[0]`：永远是程序本身的路径（可执行文件名）。
    - `os.Args[1:]`：才是真正的参数列表。

- **重要flag**

  ```go
  - os.O_RDONLY
  - os.O_WRONLY
  - os.O_RDWR
  - os.O_APPEND   // 每次写入追加
  - os.O_CREATE   // 不存在则创建
  - os.O_TRUNC    // 打开并截断
  - os.O_EXCL     // 与 O_CREATE 结合，确保“创建且独占”
  ```

> 追加写日志

```go
f, err := os.OpenFile("/var/log/myapp.log", os.O_CREATE|os.O_APPEND|os.O_WRONLY, 0644)
if err != nil { /* handle */ }
defer f.Close()
```

> 读写小文件，配置覆盖

```go
// 读配置
data, err := os.ReadFile("/etc/myapp/config.yaml")
if err != nil { /* handle */ }

// 写配置（覆盖）
err = os.WriteFile("/etc/myapp/config.yaml", []byte("new config"), 0644)
```

## 系统操作

`import "os/exec"`

- `exec.Command()`

  - 创建一个命令对象。几乎所有操作的入口。

    ```go
    cmd := exec.Command("ls", "-l", "/var/log")
    ```

- `cmd.Run()`

  - 运行命令并等待完成，不收集输出。适合只关心 **执行成功与否** 的情况。

    ```go
    cmd := exec.Command("systemctl", "restart", "nginx")
    err := cmd.Run()
    if err != nil {
        fmt.Println("重启失败:", err)
    }
    ```

- `cmd.Output()`

  - 运行命令并返回标准输出（stdout）。常用来收集命令的输出结果。

    ```go
    out, err := exec.Command("df", "-h").Output()
    if err != nil {
        fmt.Println("执行失败:", err)
    }
    fmt.Println(string(out))
    ```

- `cmd.CombinedOutput()`

  - 运行命令并返回 **stdout + stderr**。适合需要同时查看错误信息的场景。

    ```go
    out, err := exec.Command("curl", "-I", "http://localhost:8080").CombinedOutput()
    fmt.Println(string(out))
    ```

- `cmd.Start()`+`cmd.Wait()`

  - 分开启动和等待。适合需要 **异步执行** 的命令。

    ```go
    cmd := exec.Command("ping", "8.8.8.8")
    //把外部命令的标准输出和标准错误流，重定向到当前 Go 程序自己的标准输出/错误流
    //此处还需要import "os"
    cmd.Stdout = os.Stdout
    cmd.Stderr = os.Stderr
    
    // 异步启动
    if err := cmd.Start(); err != nil {
        fmt.Println("启动失败:", err)
    }
    fmt.Println("ping 已启动...")
    
    // 等待结束
    if err := cmd.Wait(); err != nil {
        fmt.Println("执行失败:", err)
    }
    ```

- `cmd.Stdin`,`cmd.Stdout`,`cmd.Stderr`

  - 手动设置输入输出，常用于 **管道重定向**。

    ```go
    cmd := exec.Command("grep", "error")
    
    // 从 Go 程序里写输入给 grep
    stdin, _ := cmd.StdinPipe()
    go func() {
        defer stdin.Close()
        stdin.Write([]byte("ok\nerror found\nhello\n"))
    }()
    
    // 读取输出
    out, _ := cmd.Output()
    fmt.Println(string(out)) // 只会输出 "error found"
    ```

## 系统信号发送/接收

一般使用`os/signal`和`syscall`包：

- 捕获终止信号进行优雅退出

- 监听操作系统级别的事件（例如，收到杀死进程的信号）


### 核心函数和方法

- `Notify(c chan<- os.Signal, sig ...os.Signal)`
   监听指定的信号并发送到 `chan` 通道。

- `Stop(c chan<- os.Signal)`
   停止接收信号通知。

- `os.Interrupt`
   表示终止程序的信号（通常是 Ctrl+C）。

- `syscall.SIGINT`

  与`os.Interrupt`相同，也是用户中断Ctrl+C。

- `syscall.SIGTERM`
   表示程序终止请求的信号。

> 示例 - 优雅地退出程序

```go
func WaitforExit(cancel context.CancelFunc) {
	sig := make(chan os.Signal, 1) //定义一个channel
	signal.Notify(sig, os.Interrupt, syscall.SIGTERM) //把os.Interrupt和syscall.SIGTERM中断信号发往signal

	<-sig //等待channel中的终止信号
	fmt.Println("\n🛑 Stopping agent...")
	cancel()//通知所有使用这个context的程序停止
	time.Sleep(1 * time.Second)
	fmt.Println("✅ Agent stopped gracefully.")
}
```

# io

- **流**：数据在内存/网络/文件中的传输过程

  - 从 **源头**（Reader）流向 **出口**（Writer）。

  - **Reader**：水龙头，提供数据

  - **Writer**：水桶，接收数据

    ```go
    r := strings.NewReader("hello") // Reader，提供数据
    io.Copy(os.Stdout, r)           // Writer，接收数据
    ```

- **管道**：类似 Linux `|`，把一个命令的输出直接作为另一个命令的输入

- `io.Copy()`

  - **作用**：从一个 `Reader` 复制数据到一个 `Writer`。

  -  常用于 **命令输出重定向、日志转储、文件复制**。

    ```go
    package main
    
    import (
    	"io"
    	"os"
    	"strings"
    )
    
    func main() {
    	// 把字符串作为 Reader，直接写到 stdout
    	r := strings.NewReader("health check ok\n")
    	io.Copy(os.Stdout, r)
    }
    ```

    ```go
    io.Copy(file, cmd.Stdout) //把命令输出写文件
    ```

    > 以`df -h`为例，写入日志文件`log.txt`
    
    ```go
    package main
    
    import (
    	"fmt"
    	"io"
    	"os"
    	"os/exec"
    )
    
    func main() {
    	cmd := exec.Command("df", "-h")
        
        // 获取命令的 stdout 管道
    	stdout, err := cmd.StdoutPipe()
    	if err != nil {
    		fmt.Println("获取管道失败：", err)
    	}
    
    	if err := cmd.Start(); err != nil {
    		fmt.Println("命令运行失败：", err)
    	}
    
    	f, err := os.OpenFile("log.txt", os.O_CREATE|os.O_APPEND|os.O_WRONLY, 0644)
    	if err != nil {
    		fmt.Println("无法打开文件：", err)
    	}
    	defer f.Close()
    
        // 把 stdout 拷贝到文件
    	io.Copy(f, stdout)
    }
    ```

# context

提供了一种**统一的任务控制模型**：
它让我们能在任意深度的并发结构中，通过父子关系、超时机制和协作信号，实现精准、优雅、可传播的任务生命周期管理。

**常用机制**

![Go_interface-context.drawio](../../dio图表/SRE/Go_interface-context.drawio.svg)

- `context.Background()`

  根 context，主协程的起点

- `context.WithCancel(parent)`

  创建一个可手动取消的子 context；

  返回context.CancelFunc，可调用它的方法`cancel()`对这一子context进行手动取消

- `context.WithTimeOut(parent, duration)`

  创建一个带超时机制的子context

- `context.WithValue(parent, key, value)`

  在context中传递键值数据

- `ctx.Done()`

  返回一个channel，表示context是否被取消

> 取消协程

```go
package main

import (
    "context"
    "fmt"
    "time"
)

func worker(ctx context.Context) {
    for {
        select {
        case <-ctx.Done(): // 收到取消信号
            fmt.Println("worker stopped:", ctx.Err())
            return
        default:
            fmt.Println("working...")
            time.Sleep(500 * time.Millisecond)
        }
    }
}

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    go worker(ctx)

    time.Sleep(2 * time.Second)
    cancel() // 通知协程停止
    time.Sleep(1 * time.Second)
}
```

> 超时控制

```go
ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
defer cancel()

select {
case <-time.After(3 * time.Second):
    fmt.Println("task done")
case <-ctx.Done():
    fmt.Println("timeout:", ctx.Err())
}
```

# 综合示例分析

- **日志分析工具**

  ```go
  package main
  
  import (
  	"bufio"
  	"fmt"
  	"os"
  	"strings"
  	"time"
  )
  
  func main() {
      //打开日志文件
  	f, err := os.Open("sample.log")
  	if err != nil {
  		fmt.Println("日志文件打开失败：", err)
  	}
  	defer f.Close()
  
      //定义日期变量和初始错误日志切片，用于存储用户输入的起始结束时间以及错误日志信息
  	var start, end string
  	errlogs := []string{}
  
  	fmt.Println("请输入起始日期(yyyy-mm-dd)：")
  	fmt.Scan(&start)
  	fmt.Println("请输入结束日期(yyyy-mm-dd)：")
  	fmt.Scan(&end)
  
      //将字符串类型的时间转为time类型，便于进行时间的计算
  	start_p, err := time.Parse("2006-01-02", start)
  	if err != nil {
  		fmt.Println("日期格式错误：", err)
  		return
  	}
  	end_p, err := time.Parse("2006-01-02", end)
  	if err != nil {
  		fmt.Println("日期格式错误：", err)
  		return
  	}
      
  	//定义“日志级别”：出现次数键值对，建立映射关系
  	logcount := map[string]int{
  		"INFO":  0,
  		"ERROR": 0,
  	}
  
      //创建流阅读器，逐行读取日志文件信息
  	scanner := bufio.NewScanner(f)
  	for scanner.Scan() {
  		line := scanner.Text()
  		parts := strings.SplitN(line, " ", 4)
  		if len(parts) < 3 {
  			continue
  		}
  
          //对每行日志信息进行拆分并分别存储为相应的变量
  		log_time := parts[0]
  		level := parts[2]
  		message := parts[3]
  
          //跳过时间戳和级别不符合要求的行
  		time_p, err := time.Parse("2006-01-02", log_time)
  		if err != nil {
  			continue
  		}
  
  		if time_p.Before(start_p) || time_p.After(end_p) {
  			continue
  		}
  
  		if level != "INFO" && level != "ERROR" {
  			continue
  		}
  
          //追加存储错误日志信息
  		if level == "ERROR" {
  			errlogs = append(errlogs, message)
  		}
  
          //累加日志级别信息
  		logcount[level]++
  	}
  	if err := scanner.Err(); err != nil {
  		fmt.Println("日志文件读取失败：", err)
  		return
  	}
  
      //输出统计信息
  	fmt.Printf("在%s到%s时间段内日志统计数据如下：\n", start, end)
  	fmt.Println("INFO出现次数：", logcount["INFO"])
  	fmt.Println("ERORR出现次数：", logcount["ERROR"])
  
  	if len(errlogs) > 0 {
  		fmt.Println("错误日志详细信息：")
  		for _, msg := range errlogs {
  			fmt.Println(msg)
  		}
  	}
  }
  ```

  - 相关知识点

    - **`time` 包**：
      - `time.Parse`：用于将时间字符串转换为 `time.Time` 类型，便于比较和判断。
      - `time.Before` 和 `time.After`：用于比较两个 `time.Time` 对象，判断是否在给定区间内。

    - **`bufio` 包**：

      - `bufio.NewScanner`：逐行读取文件内容，适用于逐行处理大文件。

      - `scanner.Text()`：获取当前扫描行的文本。

      - `scanner.Err()`：检查文件读取过程中是否发生错误。

    - **`strings` 包**：
      - `strings.SplitN`：按指定分隔符将字符串分割成切片，限制分割次数。

    - **切片（`[]string`）**：

      - 动态增长的数据结构，可以用来存储一系列同类型的数据。

      - 使用 `append()` 向切片添加元素。

    - **字典（`map`）**：

      - 键值对的数据结构，用于高效查找和存储数据。

      - 在示例代码中，用于记录每种日志级别的出现次数。

  - 输出示例
