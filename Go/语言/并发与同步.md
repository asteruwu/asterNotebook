# goroutine

- 使用 `go` 关键字启动一个新的 goroutine，使任务在后台并发执行。

- goroutine 是轻量级的，相较于传统线程消耗的内存和资源更少，可以启动成千上万的并发任务。

  - Go 的并发是通过调度器 **伪随机地** 调度多个 goroutine 的执行，而主程序需要确保所有 goroutine 都执行完毕，才能正常退出。

  - 通过使用 **同步机制**（如 `sync.WaitGroup` 或 `channel`），可以避免主程序提前退出，确保所有任务完成。

> 使用goroutine模拟ping不同服务时的并发场景

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

//定义函数，模拟ping场景
func PingServer(ServerId int, wg *sync.WaitGroup) {
	defer wg.Done() //确保任务已经完成执行后才操作计数器-1，表示待执行线程-1
	fmt.Printf("Ping server %d ...\n", ServerId)
	time.Sleep(1 * time.Second) //模拟网络延迟
	fmt.Printf("Server %d finished.\n", ServerId)
}

func main() {
	var wg sync.WaitGroup //定义waitgroup，用来跟踪所有并发线程的完成情况

	for i := 1; i <= 5; i++ {
		wg.Add(1) //每启动一个goroutine，则计数器+1，表示待执行线程+1
		go PingServer(i, &wg)
	}

	wg.Wait() //确保计数器清零后主程序才退出
}
```

> [!IMPORTANT]
>
> `WaitGroup` 在多个 goroutine 中共享同一个实例，因此需要通过指针来传递，确保各个 goroutine 对同一个 `WaitGroup` 实例（计数器）进行修改。

**计数器**

- 它是一个 **同步机制**，确保主程序在所有任务完成之前不会退出。这样，无论任务是否并发执行，主程序都能够正确地等待所有任务完成后再退出，避免因主程序提前退出导致 goroutine 被中途终止。
- 它提供了一种 **可靠的方式** 来协调多个并发任务，尤其是在 goroutine 执行顺序不确定的情况下，确保所有任务都完成。

**编译器执行顺序**

1. **循环开始：**

   - 第一次循环：`i = 1`

   - 启动一个 goroutine（但只是排队执行，还没立刻执行 `fmt.Println(i)`）

2. **第二次循环：**

   - `i = 2`

   - 又启动一个新的 goroutine（同样只是排队）

> [!IMPORTANT]
>
> goroutine 启动的那一刻只是告诉 Go 调度器：“我有个函数要执行，等你有空就跑一下。”
>
> 但主 goroutine（也就是 `main`）会继续执行下一次循环，而不会等它。所以大多数时候，for循环中的goroutine都是在**循环结束后**才开始执行的。

# channel

Channel 是一种用于在 goroutines 之间传递数据的机制，它可以实现线程安全的数据传递。Go 的并发模型中，goroutines 之间并不通过共享内存来传递数据，而是通过 **channel** 来进行通信。

- **创建channel**

  通过`make()`语句可以创建无缓冲/有缓冲的channel。

  - **无缓冲**

    ```go
    ch := make(chan int) // 创建一个无缓冲的 int 类型 channel
    ```

    - 在无缓冲 channel 中，发送方和接收方必须同步，发送操作会阻塞，直到接收方接收到数据。

      ```go
      ch := make(chan int) // 无缓冲
      go func() {
          ch <- 42 // 会阻塞，直到主 goroutine 接收
      }()
      fmt.Println(<-ch) // 主 goroutine 接收，释放发送方阻塞
      ```

  - **有缓冲**

    ```go
    ch := make(chan int, 3) // 创建一个缓冲区大小为 3 的 int 类型 channel
    ```

    - 带缓冲的 channel 可以容纳一定数量的数据，发送者只有在缓冲区满时才会阻塞，接收者只有在缓冲区为空时才会阻塞。

      ```go
      ch := make(chan int, 3) // 带缓冲区的 channel，容量为 3
      ch <- 1 // 不会阻塞
      ch <- 2 // 不会阻塞
      ch <- 3 // 不会阻塞
      ch <- 4 // 会阻塞，直到缓冲区有空间
      ```

- **发送和接收数据**

  - **发送数据到channel**

    ```go
    ch <- 5 // 将 5 发送到 channel ch
    ```

  - **从channel接收数据**

    ```go
    value, ok := <- ch // 从 channel ch 接收数据，并将其赋值给 value
    ```

- **channel的关闭**

  - **关闭channel**

    ```go
    close(ch) // 关闭 channel
    ```

  - **接收已关闭的channel**

    ```go
    value, ok := <-ch
    if !ok {
        fmt.Println("Channel 已关闭")
    } else {
        fmt.Println(value)
    }
    ```

    - 当 channel 关闭后，接收方如果继续接收数据，会得到该类型的零值。

    - 可以通过第二个返回值来判断 channel 是否已经关闭。

      当从一个 channel 接收数据时，接收操作可以返回两个值：数据本身和一个布尔值。

      - **数据本身**：这是从 channel 中接收到的值。
      - **布尔值**：如果 channel 已经关闭且没有更多的数据可以接收，布尔值 `ok` 会是 `false`；如果仍有数据可接收，`ok` 会是 `true`。

> 生产消费模型

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

//定义生产数据的函数，完成向channel发送数据的工作
func producer(ch chan int, wg *sync.WaitGroup) {
	defer wg.Done()
	for i := 1; i <= 5; i++ {
		fmt.Printf("生产：%d\n", i)
		ch <- i
		time.Sleep(time.Second)
	}
	close(ch)
}

//定义从channel接收数据的函数
func consumer(ch chan int, wg *sync.WaitGroup) {
	defer wg.Done()
	for item := range ch { //实际依然从channel中接收数据再赋值给item变量，只是使用range时简化了语法
		fmt.Printf("消费：%d\n", item)
		time.Sleep(2 * time.Second)
	}
}

func main() {
	var wg sync.WaitGroup
	ch := make(chan int, 5)

	wg.Add(2) //向计数器中添加两个待执行任务线程
	producer(ch, &wg)
	consumer(ch, &wg)

	wg.Wait()
}
```

输出示例

```go
生产：1
生产：2
生产：3
生产：4
生产：5
消费：1
消费：2
消费：3
消费：4
消费：5
```

# select

`select` 语句可以同时等待多个 channel 操作，并选择其中一个执行。它类似于 `switch` 语句，但 `select` 是用于 channel 的。

```go
select {
case msg1 := <-ch1:
    fmt.Println("从 ch1 接收到了：", msg1)
case msg2 := <-ch2:
    fmt.Println("从 ch2 接收到了：", msg2)
case <-time.After(2 * time.Second):
    fmt.Println("超时")
}
```

**用法：**

- 如果多个 case 都准备好了，`select` 会随机选择一个执行。
- 如果没有 channel 准备好，`select` 会阻塞直到有一个 case 可执行。
- 可以使用 `time.After()` 来设定超时。
  - 此处`time.After`会在设定的时间后通过channel返回一个值（当前时间），当case接收到它后就会执行相应的超时逻辑。

> 带超时控制的goroutine

```go
package main

import (
	"fmt"
	"time"
)

//模拟任务的完成，将任务完成的信息发送给channel
func doTask(ch chan string) {
	time.Sleep(3 * time.Second)
	ch <- "任务完成"
}

func main() {
	ch := make(chan string)

	go doTask(ch)
	select {
	case result := <-ch: //从channel接收完成信息并赋值给result
		fmt.Println(result)
	case <-time.After(2 * time.Second): //超时控制
		fmt.Println("任务超时")
	}
}
```

**综合示例1** - 自动化健康检查任务

```go
package main

import (
	"fmt"
	"math/rand"
	"time"
)

//定义健康检查函数
func healthcheck(serviceName string, resultCh chan string) {
    //模拟不同服务检查所使用的时间
	time.Sleep(time.Duration(rand.Intn(5)) * time.Second)

    //模拟服务检查错误，并向channel发送检查信息
	if serviceName == "Service3" {
		resultCh <- fmt.Sprintf("服务检查失败：%s\n", serviceName)
	} else {
		resultCh <- fmt.Sprintf("服务检查成功：%s\n", serviceName)
	}
}

func main() {
	services := []string{"Service1", "Service2", "Service3", "Service4"}
	resultCh := make(chan string, len(services)) //定义一个缓冲区大小为服务个数的channel

	timeout := time.After(10 * time.Second)

    //发送检查信息（生产者）
	for _, service := range services {
		go healthcheck(service, resultCh) 
	}

    //接收检查信息（消费者），并结合select进行超时控制
	for i := 0; i < len(services); i++ {
		select {
		case result := <-resultCh:
			fmt.Printf(result)
		case <-timeout:
			fmt.Println("服务检查超时")
		}
	}
}
```

**综合示例2** - 自动拉取节点日志

```go
package main

import (
	"fmt"
	"math/rand"
	"time"
)

func fetchLogs(nodeName string, logCh chan string) {
	time.Sleep(time.Duration(rand.Intn(5)) * time.Second) //模拟从节点；拉取日志所需要的时间

	result := rand.Intn(2) == 0 //模拟日志拉取成功/失败
	if result {
		logCh <- fmt.Sprintf("[%v]%s节点日志拉取成功\n", time.Now().Format("2006-01-02 15:04:05"), nodeName)
	} else {
		logCh <- fmt.Sprintf("[%v]%s节点日志拉取失败\n", time.Now().Format("2006-01-02 15:04:05"), nodeName)
	}
}

func main() {
	node := []string{"Nginx", "MySQL", "Redis", "Kafka"}
	logCh := make(chan string, len(node))
	timeoutPerNode := time.After(3 * time.Second)

	for _, item := range node {
		go func(nodeName string) {
			select {
			case <-timeoutPerNode: //单点超时控制
				fmt.Printf("[%v]%s节点日志拉取超时\n", time.Now().Format("2006-01-02 15:04:05"), nodeName)
			default:
				fetchLogs(nodeName, logCh)
			}
		}(item)
	}

	for i := 0; i < len(node); i++ {
		select {
		case result := <-logCh:
			fmt.Printf(result)
		}
	}
}
```

# sync

- `Mutex/RWMutex`

  - 在 Go 中，多个 goroutine 同时访问共享资源（如 map、文件、计数器等）会引发数据竞争。
     `sync.Mutex` 用于**保护共享资源**，保证同一时刻**只有一个 goroutine** 可以访问它。

    ```go
    package main
    
    import (
    	"fmt"
    	"sync"
    )
    
    func main() {
    	var mu sync.Mutex //定义锁
    	data := make(map[string]int)
    
    	var wg sync.WaitGroup
    	for i := 0; i < 5; i++ {
    		wg.Add(1)
    		go func(id int) {
    			defer wg.Done()
    			mu.Lock() //加锁，只允许一个线程能够访问data["count"]
    			data["count"]++
    			fmt.Printf("Goroutine %d incremented count to %d\n", id, data["count"])
    			mu.Unlock() //解锁
    		}(i)
    	}
    	wg.Wait()
    }
    ```

    > [!CAUTION]
    >
    > 如果不加锁，即删除`var mu sync.Mutex`、`mu.Lock()`和`mu.Unlock()`会报错：
    >
    > fatal error: concurrent map writes

  - `sync.RWMutex` 是读写锁，支持：

    - 多个读者同时访问（并发读安全）

    - 但写操作必须独占（写锁会阻塞所有读写）

      > 使用`sync.RWMutex`优化读多写少的场景，允许并发读

      ```go
      var (
          rw  sync.RWMutex
          cfg = map[string]string{"version": "1.0"}
      )
      
      
      //RLock()允许并发读，阻塞写操作
      func readConfig() string {
          rw.RLock()
          defer rw.RUnlock()
          return cfg["version"]
      }
      
      //Lock()阻塞所有读写操作
      func updateConfig(v string) {
          rw.Lock()
          cfg["version"] = v
          rw.Unlock()
      }
      ```

- `Once`

  - `sync.Once` 确保某个操作在**全局范围内只执行一次**，常用于：

    - 单例初始化

    - 加载配置文件

    - 启动后台任务

      > 初始化只执行一次

      ```go
      package main
      
      import (
          "fmt"
          "sync"
      )
      
      var once sync.Once
      var config map[string]string
      
      //定义初始化配置的函数
      func loadConfig() {
          fmt.Println("Loading config...")
          config = map[string]string{"env": "prod"}
      }
      
      //确保配置初始化只执行一次，并返回配置
      func GetConfig() map[string]string {
          once.Do(loadConfig)
          return config
      }
      
      func main() {
          var wg sync.WaitGroup
          for i := 0; i < 3; i++ {
              wg.Add(1)
              go func() {
                  defer wg.Done()
                  fmt.Println(GetConfig())
              }()
          }
          wg.Wait()
      }
      ```

- `Cond`

  - `sync.Cond` 是基于锁的**等待/通知机制**，用于**协调多个 goroutine 的状态变化**。
     常用于：

    - 生产者-消费者模型

    - 等待资源就绪

    - 队列等待/唤醒机制

      > 生产者-消费者模型

      ```go
      package main
      
      import (
      	"fmt"
      	"sync"
      )
      
      func main() {
      	mu := sync.Mutex{}
      	cond := sync.NewCond(&mu)
      	queue := []int{}
      	var wg sync.WaitGroup
      
      	// 消费者
      	wg.Add(1)
      	go func() {
      		defer wg.Done()
      		for {
      			mu.Lock()
      			for len(queue) == 0 {
      				cond.Wait() // 等待生产者唤醒
      			}
      			item := queue[0]
      			queue = queue[1:]
      			mu.Unlock()
      			fmt.Println("Consumed:", item)
      
      			if item == 5 {
      				return
      			}
      		}
      	}()
      
      	// 生产者
      	for i := 1; i <= 5; i++ {
      		mu.Lock()
      		queue = append(queue, i)
      		fmt.Println("Produced:", i)
      		mu.Unlock()
      		cond.Signal() // 唤醒一个等待的消费者
      	}
      
      	wg.Wait()
      }
      ```

- `Pool`

  - `sync.Pool` 是 Go 提供的一种**短期对象缓存机制**，用于**在多个 goroutine 间高效复用临时对象**，以减少频繁的内存分配和垃圾回收负担。

  - 适用于：

    - **高频创建、短生命周期对象**；

    - **对象分配开销较大**；

    - **允许复用但不要求状态持久** 的场景。

      > 复用`bytes.Buffer`示例

      ```go
      package main
      
      import (
          "bytes"
          "fmt"
          "sync"
      )
      
      var bufPool = sync.Pool{
          New: func() interface{} {
              return new(bytes.Buffer)
          },
      }
      
      func main() {
          b := bufPool.Get().(*bytes.Buffer)
          b.Reset()
          b.WriteString("hello, world")
      
          fmt.Println(b.String())
      
          bufPool.Put(b) // 归还给池
      
          // 下次可复用
          b2 := bufPool.Get().(*bytes.Buffer)
          fmt.Println("Reused buffer len:", b2.Len()) // => 0
      }
      ```

# pond

- 旨在提供一种简单且惯用的方法来管理 Go 程序中的并发性。它允许并发运行大量任务，同时**限制**同时运行的 goroutine 数量。当需要限制并发的数量以**避免资源耗尽**或**达到速率限制**时，这非常有用。

  > 将任务提交到并发受限的池

  ```go
  package main
  
  import (
  	"fmt"
  
  	"github.com/alitto/pond/v2"
  )
  
  func main() {
  
  	// 创建一个允许同时运行的最大线程数为100的池
  	pool := pond.NewPool(100)
  
  	// 向池提交任务
  	for i := 0; i < 1000; i++ {
  		i := i
  		pool.Submit(func() { //通过闭包函数定义任务具体的执行操作
  			fmt.Printf("Running task #%d\n", i)
  		})
  	}
  
  	// 等待所有提交了的任务执行完成
  	pool.StopAndWait()
  }
  ```

- **池指标**

  每个工作器池实例都公开了有用的指标，可以通过以下方法进行查询：

  - `pool.RunningWorkers()`：当前正在运行的工作人员数量

  - `pool.SubmittedTasks()`：自创建池以来和停止之前提交的任务总数。这包括由于队列已满而被删除的任务

  - `pool.WaitingTasks()`：队列中等待执行的当前任务数

  - `pool.SuccessfulTasks()`：自创建池以来已成功完成执行的任务总数

  - `pool.FailedTasks()`：自创建池以来完成并出现崩溃的任务总数

  - `pool.CompletedTasks()`：自创建池以来已成功完成执行或出现崩溃的任务总数

  - `pool.DroppedTasks()`：自创建池以来，由于队列已满而删除的任务总数

>  并发检查大量服务器的系统状态并写入日志

```go
package main

import (
	"fmt"
	"log"
	"math/rand"
	"os"
	"sync"

	"github.com/alitto/pond/v2"
)

// 定义结构体存储服务器的检查结果
type ServerCheckResult struct {
	ServerName string
	CPU        int
	Memory     int
	Disk       int
}

func main() {
	var wg sync.WaitGroup
	servers := []string{}      //定义空切片，存储服务器名
	for i := 1; i <= 20; i++ { //生成字符串填充切片
		servers = append(servers, fmt.Sprintf("Server%d", i))
	}

	pool := pond.NewPool(5) //创建池，最多允许的并发数量为5

	results := make(chan ServerCheckResult, len(servers)) //定义channel，用于接收和发送服务器结构体

	for _, s := range servers { //遍历服务器切片
		server := s          //每遍历一个服务器就进行一次更新，防止闭包捕获
		pool.Submit(func() { //（生成结构体数据并发送给channel）将任务提交给池
			result := ServerCheckResult{
				ServerName: server,
				CPU:        rand.Intn(101),
				Memory:     rand.Intn(33),
				Disk:       rand.Intn(1025),
			}
			results <- result
		})
	}

	//启动一个新线程写日志
	wg.Add(1)
	go func() {
		defer wg.Done()
		file, err := os.OpenFile("C:\\Users\\86133\\Desktop\\学习\\Go\\concurrent processing\\pond_learn\\concurrent check.log", os.O_CREATE|os.O_APPEND|os.O_WRONLY, 0644)
		if err != nil {
			log.Println("日志创建失败")
		}
		defer file.Close()

		log.SetOutput(file)

		for data := range results {
			log.Printf("\n===%s===\nCPU: %d\nMemory: %d\nDisk: %d\n", data.ServerName, data.CPU, data.Memory, data.Disk)
		}
	}()

	//等待所有并发任务完成并关闭channel
	pool.StopAndWait()
	close(results)
	wg.Wait()

	fmt.Println("已将服务器巡检结果写入日志，请参考文件cocurrent check.log")
}
```
